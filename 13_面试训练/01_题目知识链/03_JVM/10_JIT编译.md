## 知识链提示

```
JIT 编译
  → [[03_JVM/09_JIT编译与性能优化/JIT编译与性能优化]]
    → JVM 执行方式演进
      → 解释执行：逐条读字节码 → 逐条解释成机器码（慢）
      → JIT 编译：热点代码 → 编译成本地机器码（缓存，下次直接执行）
      → AOT 编译（JDK 9+）：运行前就编译好（启动快，但峰值性能不如 JIT）
    → 热点探测（HotSpot 默认用计数器）
      → 方法调用计数器：方法被调用多少次
      → 回边计数器：循环中回跳多少次（OSR：栈上替换）
      → 阈值：-XX:CompileThreshold（默认 10000）
    → 两种 JIT 编译器
      → C1（Client Compiler）：简单优化，编译快（启动阶段用）
      → C2（Server Compiler）：深度优化，编译慢（峰值性能高）
      → 分层编译（JDK 7+ 默认开启）：先 C1 编译，热点方法再 C2 编译
    → 经典优化手段
      → 方法内联（Method Inlining）：减少方法调用开销
      → 逃逸分析（Escape Analysis）：对象不逃逸 → 栈上分配 / 锁消除
      → 锁消除（Lock Elision）：局部变量 StringBuffffer → 无竞争 → 去掉锁
      → 锁粗化（Lock Coarsening）：多个连续 synchronized → 合并成一个
```

## 核心追问

| 追问 | 考点 |

| 什么是 OSR（On-Stack Replacement）？ | 方法运行中，热点循环被替换成本地代码 |

---

## 参考答案要点

**JIT 的核心价值**：把热点代码编译成本地机器码，避免逐条解释执行，大幅提升性能。

**热点探测**：方法调用次数 > `-XX:CompileThreshold`（默认 10000）→ 触发 JIT 编译。

**经典优化**：
- 方法内联：省去方法调用开销
- 逃逸分析：对象不逃逸 → 栈上分配（减少 GC 压力）
- 锁消除：局部 StringBuffffer 的锁被去掉
- 锁粗化：多个连续 synchronized 合并
