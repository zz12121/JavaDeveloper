# 并发编程 — 全景地图

> 并发是区分中级和高级的分水岭。不是学会用线程，而是理解「为什么线程不安全」。

---

## 学什么

```
并发编程
├── 01_线程基础          — 进程vs线程、生命周期、创建方式、中断、守护线程
├── 02_JMM内存模型       — 主内存/工作内存、happens-before、volatile、内存屏障
├── 03_锁机制          — synchronized 锁升级、ReentrantLock、读写锁、StampedLock
├── 04_AQS             — state+CLH队列+模板方法、独占/共享模式、Condition
├── 05_CAS与原子类        — CAS原理、ABA问题、AtomicInteger/LongAdder
├── 06_并发容器          — ConcurrentHashMap、CopyOnWriteArrayList、ConcurrentSkipListMap
├── 07_阻塞队列          — ArrayBlockingQueue、LinkedBlockingQueue、SynchronousQueue、DelayQueue
├── 08_线程池            — ThreadPoolExecutor参数与流程、ForkJoinPool工作窃取、拒绝策略
├── 09_CompletableFuture — 异步编排、链式调用、异常处理、超时控制
├── 10_并发工具类         — CountDownLatch、CyclicBarrier、Semaphore
├── 11_死锁与活锁         — 必要条件、预防检测、jstack分析
├── 12_ThreadLocal       — 内存泄漏、InheritableThreadLocal、TransmittableThreadLocal
├── 13_虚拟线程          — JDK21、载体线程、Continuation、结构化并发
└── 14_实战应用          — 生产者消费者、并发缓存、限流、Disruptor
```

## 怎么学

**顺序**：01（基础概念）→ 02（JMM是理解一切并发问题的根基）→ 03（synchronized + Lock 两条并行路径）→ 04（AQS是锁和工具类的骨架）→ 05（CAS原子操作）→ 06/07/08（容器与线程池应用）→ 09/10（高级编排）→ 11/12（陷阱与坑）→ 13（未来方向）→ 14（融会贯通）

**关键认知**：
- JMM 是理解并发问题的**根本框架**——可见性、有序性、原子性三个问题都从这里来
- synchronized 和 Lock 是两条**并行的锁实现路径**，底层机制完全不同
- AQS 是 JUC 的**骨架**——ReentrantLock、Semaphore、CountDownLatch 都基于它
- 线程池不是"用了就行"，参数设置和拒绝策略直接影响线上稳定性
- 虚拟线程是并发编程的未来，但不能替代所有场景

## 和其他域的关系

```
02_并发 ←→ 01_Java核心   （多线程 → Thread类；集合 → 并发容器）
02_并发 ←→ 03_JVM        （JMM → 内存模型；线程 → JVM运行时数据区）
02_并发 ←→ 05_数据库      （连接池 → 线程池；事务隔离 → 并发控制）
02_并发 ←→ 06_中间件      （Redis → 分布式锁；MQ → 生产者消费者）
02_并发 ←→ 07_分布式      （分布式锁 → CAP；幂等 → 并发安全）
```
