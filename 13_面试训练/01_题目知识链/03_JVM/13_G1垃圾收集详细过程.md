## 参考要点（盲答后再看）


**G1 堆结构**：
```
堆 = 多个 Region（1MB ~ 32MB，大小是 2 的幂）
每个 Region 的角色动态变化：Eden / Survivor / Old / Humongous
```

**三种 GC 模式**：

| 模式 | 触发条件 | 范围 |
|------|---------|------|
| Young GC | Eden 区满了 | 只回收 Eden + Survivor |
| Mixed GC | IHOP 阈值触发（默认 45%） | Eden + 部分老年代 Region |
| Full GC | 空间耗尽，无法分配 | 整堆（单线程，最慢！） |

**RSet（记忆集）—— G1 的核心**：
```
每个 Region 有一个 RSet
记录「哪些其他 Region 的对象引用了本 Region 的对象」
→ 回收 Region 时，不需要扫描整个堆（只需要扫描 RSet）
```
代价：写屏障维护 RSet，有开销。

**Mixed GC 过程（重点）**：
```
1. 初始标记（STW，借 Young GC 做）
2. 并发标记（三色标记 + SATB）
3. 最终标记（STW，处理 SATB 队列）
4. 筛选回收（ST：选收益最高的 Region，复制存活对象）
```

**IHOP（Initiating Heap Occupancy Percent）**：
- 默认值：45%（JDK9+ 自适应）
- 当老年代占用达到 45% → 触发并发标记周期
- 目的：在堆满之前提前开始回收，避免 Full GC

**G1 调优核心参数**：
| 参数 | 默认值 | 说明 |
|------|--------|------|
| `-XX:MaxGCPauseMillis` | 200ms | 目标停顿时间（不是硬保证） |
| `-XX:InitiatingHeapOccupancyPercent` | 45 | 触发并发标记的阈值 |
| `-XX:G1HeapRegionSize` | 自动计算 | Region 大小 |
| `-XX:G1MixedGCCountTarget` | 8 | 触发 Full GC 前最多做几次 Mixed GC |
