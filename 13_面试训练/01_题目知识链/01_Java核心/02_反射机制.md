## 参考要点（盲答后再看）

<details>
<summary>点击展开</summary>

**反射底层流程**：
1. 获取 `Class` 对象（`forName` / `.class` / `getClass()`）
2. 获取 `Field` / `Method` / `Constructor` 对象
3. 调用 `setAccessible(true)` 关闭访问检查
4. `method.invoke(obj, args)` → 底层走 `native` 方法

**性能开销来源**：
- 访问检查（`setAccessible` 可以缓解）
- 参数装箱/拆箱（invoke 参数是 `Object[]`）
- 无法被 JIT 内联优化（JDK 8+ 有 `LambdaForm` 优化，缓解了很多）
- 每次反射调用都要检查 `Method` 对象是否还有效

**forName vs loadClass**：
```
Class.forName()         →  加载 + 链接 + 初始化（执行 static 块）
ClassLoader.loadClass() →  只加载（不初始化）
```

**动态代理本质**：`Proxy.newProxyInstance()` → 运行时生成字节码 → 继承 Proxy → InvocationHandler.invoke() → 反射调用目标方法

</details>
