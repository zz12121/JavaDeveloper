# 堆与优先队列

> 堆是解决"Top K 问题"和"优先级调度"的核心数据结构。PriorityQueue 的底层就是堆。

---

## 一、堆的基本概念

### 1.1 定义

堆（Heap）是一种特殊的完全二叉树，满足**堆性质**：

- **大顶堆（Max Heap）**：每个节点的值 ≥ 其子节点的值
- **小顶堆（Min Heap）**：每个节点的值 ≤ 其子节点的值

```
大顶堆示例：            小顶堆示例：
       10                   1
      /  \                 /  \
     9    8               2     3
    / \  / \             / \   / \
   5  6 7   4           8   5 6   7

根节点是最大值            根节点是最小值
```

### 1.2 数组表示

堆可以用数组高效存储（完全二叉树的特性）：

```
       10 (index: 0)
      /  \
     9(1)  8(2)
    / \    / \
   5(3) 6(4) 7(5) 4(6)

数组: [10, 9, 8, 5, 6, 7, 4]

父子关系：
- 父节点: parent(i) = (i - 1) / 2
- 左子节点: left(i) = 2i + 1
- 右子节点: right(i) = 2i + 2
```

## 二、堆的核心操作

### 2.1 上浮（Sift Up / Swim）

插入元素后，从底部向上调整，恢复堆性质。

```java
// 小顶堆上浮
private void siftUp(int k) {
    while (k > 0) {
        int parent = (k - 1) / 2;
        if (heap[k] >= heap[parent]) break; // 已满足堆性质
        swap(heap, k, parent);
        k = parent; // 继续向上比较
    }
}
```

```
上浮过程示例（插入 2 到小顶堆）：

       5                    5                   2
      / \                  / \                 / \
     7   10    →         7   10     →        5   10
    /                   /                   / \
   2                   2                   7   7  ← 2 上浮到根
```

### 2.2 下沉（Sift Down / Sink）

删除堆顶或建堆时，从顶部向下调整，恢复堆性质。

```java
// 小顶堆下沉
private void siftDown(int k, int n) {
    while (2 * k + 1 < n) { // 有左子节点
        int left = 2 * k + 1;
        int right = left + 1;
        int smallest = left;
        // 找到更小的子节点
        if (right < n && heap[right] < heap[left]) {
            smallest = right;
        }
        if (heap[k] <= heap[smallest]) break; // 已满足堆性质
        swap(heap, k, smallest);
        k = smallest; // 继续向下比较
    }
}
```

### 2.3 操作复杂度

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 查看堆顶 | O(1) | 数组首元素 |
| 插入 | O(log n) | 上浮，最多 log n 层 |
| 删除堆顶 | O(log n) | 交换首尾 + 删除尾部 + 下沉 |
| 建堆 | O(n) | 从最后一个非叶子节点开始依次下沉 |
| 堆排序 | O(n log n) | 建堆 O(n) + n 次取堆顶 O(n log n) |

### 2.4 建堆为什么是 O(n)？

```
直觉：n 个元素每个都要下沉，每次最多 log n → O(n log n)？
实际：O(n)！

分析（小顶堆下沉）：
- 最后一层（n/2 个节点）：不需要下沉（没有子节点）→ 0
- 倒数第二层（n/4 个节点）：最多下沉 1 次 → n/4
- 倒数第三层（n/8 个节点）：最多下沉 2 次 → n/8 × 2
- ...
- 根节点（1 个）：最多下沉 log n 次 → log n

总次数 = Σ(n/2^(i+1) × i), i=0 to log n
       ≤ n × Σ(i/2^i), i=1 to ∞
       = n × 1  （Σ(i/2^i) = 2 是常数）
       = O(n)
```

## 三、优先队列（Priority Queue）

### 3.1 Java PriorityQueue

```java
// 默认小顶堆
Queue<Integer> minHeap = new PriorityQueue<>();

// 大顶堆（两种写法）
Queue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
Queue<Integer> maxHeap2 = new PriorityQueue<>((a, b) -> b - a);

// 基本操作
minHeap.offer(5);     // 插入 O(log n)
minHeap.offer(3);
minHeap.offer(7);
minHeap.peek();       // 查看堆顶 → 3，O(1)
minHeap.poll();       // 删除并返回堆顶 → 3，O(log n)
minHeap.size();       // 元素个数
```

### 3.2 PriorityQueue 源码要点

```java
public class PriorityQueue<E> {
    private Object[] queue;  // 默认初始容量 11
    private int size;
    private final Comparator<? super E> comparator;

    // 插入
    public boolean offer(E e) {
        if (e == null) throw new NullPointerException();
        modCount++;
        int i = size;
        if (i >= queue.length) grow(i + 1);  // 扩容
        size = i + 1;
        if (i == 0) queue[0] = e;
        else siftUp(i, e);                   // 上浮
        return true;
    }

    // 取出堆顶
    public E poll() {
        final Object[] es;
        final E result;
        if ((result = (E)((es = queue)[0])) != null) {
            modCount++;
            final int n;
            final E x = (E)es[--(n = size)];
            es[n] = null;  // help GC
            if (n > 0) {
                E moved = (E)es[0];  // 最后一个元素移到堆顶
                es[0] = null;
                siftDown(0, moved, es, n);  // 下沉
            }
            return result;
        }
        return null;
    }
}
```

## 四、堆排序

### 4.1 算法流程

```
1. 建堆（原地堆化）：从最后一个非叶子节点 (n/2 - 1) 开始依次下沉
2. 排序：反复取出堆顶（交换首尾 → 缩小堆 → 下沉堆顶）
```

### 4.2 代码实现

```java
public void heapSort(int[] arr) {
    int n = arr.length;

    // 1. 建堆（大顶堆），从最后一个非叶子节点开始
    for (int i = n / 2 - 1; i >= 0; i--) {
        siftDown(arr, i, n);
    }

    // 2. 排序：将堆顶（最大值）移到末尾
    for (int i = n - 1; i > 0; i--) {
        swap(arr, 0, i);        // 堆顶与末尾交换
        siftDown(arr, 0, i);    // 缩小堆，重新下沉
    }
}

private void siftDown(int[] arr, int k, int n) {
    while (2 * k + 1 < n) {
        int left = 2 * k + 1, right = left + 1;
        int largest = left;
        if (right < n && arr[right] > arr[left]) largest = right;
        if (arr[k] >= arr[largest]) break;
        swap(arr, k, largest);
        k = largest;
    }
}
```

### 4.3 堆排序特点

| 维度 | 堆排序 | 快排 | 归并排序 |
|------|--------|------|---------|
| 时间 | O(n log n) | O(n log n) | O(n log n) |
| 空间 | O(1) | O(log n) | O(n) |
| 稳定性 | 不稳定 | 不稳定 | 稳定 |
| 最坏情况 | O(n log n) | O(n²) | O(n log n) |
| 缓存友好 | 较差（跳跃访问） | 好（顺序访问） | 好 |

> 堆排序的优势：最坏情况也是 O(n log n)，且原地排序。劣势：缓存不友好，实际运行速度通常比快排慢。

## 五、Top K 问题（面试高频）

### 5.1 Top K 最大元素 → 小顶堆

```java
public int[] topK(int[] nums, int k) {
    // 小顶堆，堆中始终保持 K 个最大元素
    PriorityQueue<Integer> minHeap = new PriorityQueue<>(k);
    for (int num : nums) {
        if (minHeap.size() < k) {
            minHeap.offer(num);
        } else if (num > minHeap.peek()) {
            minHeap.poll();    // 移除堆中最小的
            minHeap.offer(num); // 加入更大的
        }
    }
    int[] result = new int[k];
    for (int i = k - 1; i >= 0; i--) {
        result[i] = minHeap.poll();
    }
    return result;
}
// 时间: O(n log k)，空间: O(k)
```

### 5.2 为什么 Top K 用小顶堆？

```
求 Top K 最大：
- 小顶堆维护 K 个元素，堆顶是这 K 个中最小的
- 遍历时，比堆顶大的元素才能入堆
- 最终堆中就是最大的 K 个元素

求 Top K 最小：
- 大顶堆维护 K 个元素，堆顶是这 K 个中最大的
- 遍历时，比堆顶小的元素才能入堆
- 最终堆中就是最小的 K 个元素
```

### 5.3 Top K 方法对比

| 方法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|-----------|-----------|---------|
| 小顶堆 | O(n log k) | O(k) | n 很大，k 很小 |
| 大顶堆 | O(n + k log n) | O(n) | k 接近 n |
| 快速选择 | O(n) 平均 | O(1) | 不需要保持有序 |
| 堆排序 | O(n log n) | O(1) | 需要完整排序 |

### 5.4 快速选择（QuickSelect）

```java
// 平均 O(n)，最坏 O(n²)（可以通过随机化 pivot 优化）
public int findKthLargest(int[] nums, int k) {
    int target = nums.length - k; // 第 k 大 = 第 n-k 小
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int pivotIndex = partition(nums, left, right);
        if (pivotIndex == target) return nums[pivotIndex];
        else if (pivotIndex < target) left = pivotIndex + 1;
        else right = pivotIndex - 1;
    }
    return -1;
}
```

## 六、堆的实际应用

### 6.1 合并 K 个有序链表

```java
public ListNode mergeKLists(ListNode[] lists) {
    // 小顶堆，按节点值排序
    PriorityQueue<ListNode> minHeap = new PriorityQueue<>(
        (a, b) -> a.val - b.val
    );

    // 将每个链表的头节点入堆
    for (ListNode node : lists) {
        if (node != null) minHeap.offer(node);
    }

    ListNode dummy = new ListNode(0);
    ListNode tail = dummy;

    while (!minHeap.isEmpty()) {
        ListNode node = minHeap.poll();
        tail.next = node;
        tail = tail.next;
        if (node.next != null) {
            minHeap.offer(node.next); // 将该链表的下一个节点入堆
        }
    }
    return dummy.next;
}
// 时间: O(n log k)，k 为链表数量，n 为总节点数
```

### 6.2 数据流的中位数

```java
// 两个堆：大顶堆存较小的一半，小顶堆存较大的一半
class MedianFinder {
    private PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder()); // 较小的一半
    private PriorityQueue<Integer> minHeap = new PriorityQueue<>(); // 较大的一半

    public void addNum(int num) {
        if (maxHeap.isEmpty() || num <= maxHeap.peek()) {
            maxHeap.offer(num);
        } else {
            minHeap.offer(num);
        }
        // 平衡两个堆的大小，差不超过 1
        if (maxHeap.size() > minHeap.size() + 1) {
            minHeap.offer(maxHeap.poll());
        } else if (minHeap.size() > maxHeap.size()) {
            maxHeap.offer(minHeap.poll());
        }
    }

    public double findMedian() {
        if (maxHeap.size() == minHeap.size()) {
            return (maxHeap.peek() + minHeap.peek()) / 2.0;
        }
        return maxHeap.peek();
    }
}
// 插入 O(log n)，查询 O(1)
```
