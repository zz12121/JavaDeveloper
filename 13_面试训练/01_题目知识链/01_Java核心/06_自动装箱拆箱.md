## 参考要点（盲答后再看）


**自动装箱/拆箱本质**（编译期，javac 做的）：

```java
// 源码
Integer i = 100;   // 装箱
int x = i;          // 拆箱

// 编译后
Integer i = Integer.valueOf(100);
int x = i.intValue();
```

**Integer 缓存池**（`IntegerCache` 静态内部类）：

```java
// Integer.valueOf() 源码
public static Integer valueOf(int i) {
    if (i >= IntegerCache.low && i <= IntegerCache.high)
        return IntegerCache.cache[i + (-IntegerCache.low)];
    return new Integer(i);
}

// 缓存范围：low = -128，high 默认 = 127（可调）
```

**各包装类缓存情况**：

| 包装类 | 有缓存池？ | 缓存范围 |
|--------|-----------|---------|
| `Integer` | ✅ | -128 ~ 127（上限可调） |
| `Long` | ✅ | -128 ~ 127（不可调） |
| `Short` | ✅ | -128 ~ 127 |
| `Character` | ✅ | 0 ~ 127 |
| `Byte` | ✅ | 全部缓存（-128~127） |
| `Float` / `Double` | ❌ | 没有（浮点数不适合缓存） |
| `Boolean` | ✅ | `TRUE` / `FALSE` 两个常量 |

**== 与 equals**：

```java
Integer a = 127;
Integer b = 127;
a == b;        // true（同一缓存对象）
a.equals(b);  // true（值相等）

Integer c = 128;
Integer d = 128;
c == d;        // false（不同对象，超出缓存范围）
c.equals(d);  // true
```

**性能陷阱**：

```java
// 糟糕：每次循环都拆箱+装箱
Integer sum = 0;
for (int i = 0; i < 10000; i++) {
    sum += i;  // sum.intValue() + i → Integer.valueOf(结果)
}
// 创建了 10001 个Integer对象！

```
