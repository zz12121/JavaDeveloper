<summary>点击展开</summary>

**逃逸分析的核心问题**：判断对象的作用域是否「逃出」当前方法或线程。

| 逃逸类型 | 含义 | 例子 |
|----------|------|------|
| 无逃逸 | 对象只在方法内使用 | `new Object()` 不出方法 |
| 方法逃逸 | 对象被当作返回值返回 | `return new Object()` |
| 线程逃逸 | 对象被其他线程访问 | 赋值给静态变量 / 其他线程可见的字段 |

**三大 JIT 优化（基于逃逸分析）**：

1. **栈上分配（标量替换）**：
```java
// 源码
void test() {
    Point p = new Point(1, 2);  // 不逃逸
    int x = p.x;
}
// JIT 优化后（标量替换，不创建 Point 对象）：
void test() {
    int x = 1;  // 直接拆成基本类型，在栈上
}
```
→ GC 不需要回收，栈帧弹出自动释放。

2. **锁消除（Lock Elimination）**：
```java
// 源码
synchronized (new Object()) {  // 锁对象不逃逸
    // do something
}
// JIT：锁对象不会共享 → 消除 synchronized
```

3. **锁粗化（Lock Coarsening）**：
```java
// 源码
for (int i = 0; i < 100; i++) {
    synchronized (lock) { doSomething(); }
}
// JIT：合并成一次加锁
synchronized (lock) {
    for (int i = 0; i < 100; i++) { doSomething(); }
}
```

**注意**：「栈上分配」在 HotSpot 里实际是标量替换，不是真的在栈上分配对象头。

**逃逸分析失败的场景**：
- 对象被返回（方法逃逸）
- 对象赋值给静态变量或实例字段（可能被其他线程看到）
- 对象被放入集合（集合本身逃逸）
