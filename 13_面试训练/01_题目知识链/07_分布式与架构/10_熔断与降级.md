# 熔断和降级是什么？有什么区别？

> ⚠️ **先盲答**：熔断和降级是一回事吗？它们分别解决什么问题？

---

## 盲答引导

1. 熔断（Circuit Breaker）的三个状态是什么？—— 闭合 / 打开 / 半开
2. 熔断的触发条件是什么？—— 失败率达到阈值
3. 降级（Fallback）是什么意思？—— 降级到「备用方案」返回兜底数据
4. Hystrix / Sentinel / Resilience4j 分别是干什么的？

---

## 知识链提示

```
服务容错
  → [[服务容错]]
    → 熔断（Circuit Breaker）
      → 三种状态
        → CLOSED（闭合）：正常调用，统计失败率
        → OPEN（打开）：请求直接失败（快速失败），不调用下游
        → HALF_OPEN（半开）：放一个请求试探，如果成功则 CLOSED，否则 OPEN
      → 触发条件
        → 失败率 > 阈值（如 50%，10 秒内 10 次请求中 5 次失败）
        → 超时 / 异常 / 限流
      → [[Hystrix]]（Netflix，已停止维护）/ [[Sentinel]]（阿里，生产级）/ Resilience4j
    → 降级（Fallback）
      → 下游故障 → 降级返回兜底数据（缓存 / 静态数据 / 默认值）
      → 不影响主流程，让用户有响应
    → 限流 + 熔断 + 降级的关系
      → 限流：拒绝超出阈值的请求（保护自己）
      → 熔断：当下游故障时，快速失败，不再调用下游（保护下游）
      → 降级：熔断后返回兜底结果（让用户有响应）
    → 降级策略
      → 返回缓存数据
      → 返回默认值
      → 返回友好的错误提示
```

## 核心追问

| 追问 | 考点 |
|------|------|
| 熔断打开后，什么时候会尝试「半开」？ | 经过一段冷却时间后（如 30s） |
| Hystrix 和 Sentinel 选哪个？ | Sentinel（功能更全，规则动态更新）/ Hystrix（停止维护）|
| 为什么需要「半开」状态？ | 直接 CLOSED 可能假恢复，半开是试探 |
| 降级和熔断能互相替代吗？ | 不能，是两个不同层次的保护 |

---

## 参考答案要点

**熔断 vs 降级**：
- 熔断：保护下游，阻止请求打到故障服务
- 降级：保护用户体验，给一个「能用的」替代结果

**三者关系**：限流（入口保护）+ 熔断（故障隔离）+ 降级（兜底体验）。

**Hystrix vs Sentinel**：
- Hystrix：线程池隔离（资源消耗大），停止维护
- Sentinel：滑动窗口统计（资源消耗小），支持多种降级策略，生产推荐
