# JVM — 全景地图

> JVM 知识让你从「会用 Java」变成「理解 Java」。是排查疑难杂症的底层武器。

---

## 学什么

```
JVM
├── 01_JVM架构              — JVM整体架构、JVM vs JRE vs JDK、执行引擎
├── 02_字节码与类文件         — 字节码指令、javap使用、class文件结构
├── 03_类加载机制            — 双亲委派、加载/链接/初始化、自定义ClassLoader
├── 04_运行时数据区          — 堆、方法区、虚拟机栈、本地方法栈、程序计数器
├── 05_堆内存结构            — 新生代/老年代、Eden/Survivor、对象分配、TLAB
├── 06_垃圾回收算法          — GC Roots、可达性分析、标记-清除/复制/整理
├── 07_垃圾收集器            — Serial/ParNew/CMS/G1/ZGC/Shenandoah
├── 08_JVM调优与问题排查     — OOM排查、MAT分析、GC日志、Arthas/jstat/jmap
│   ├── 内存溢出与泄漏       — OOM类型、泄漏场景、排查工具
│   ├── JVM调优             — 参数设置、调优策略
│   └── JVM调优实战进阶     — 线上OOM排查、GC问题定位、CPU 100%/死锁排查
└── 09_JIT编译与性能优化     — 逃逸分析、内联、分层编译
```

## 怎么学

**顺序**：01 → 02 → 03 → 04 → 05 → 06 → 07 → 08 → 09

这条线对应的是一段代码从「源码」到「运行」到「优化」的完整生命周期：

```
源码(.java)
  → 字节码(.class)           ← 02
    → 类加载                  ← 03
      → 运行时数据区           ← 04
        → 堆内存结构           ← 05
          → 垃圾回收算法       ← 06
            → 垃圾收集器       ← 07
              → 调优与排查      ← 08
                → JIT优化      ← 09
```

**关键认知**：
- 运行时数据区是**理解一切的基础**——线程安全问题、内存泄漏、GC 都跟它相关
- 堆内存分代是**理解 GC 的前提**——不看堆结构就学 GC 等于空中楼阁
- GC 不是背参数，是理解「什么对象该回收」「怎么回收」「回收对应用有什么影响」
- 类加载机制是理解 Spring/MyBatis/Tomcat 启动过程的基础
- JVM 调优的本质是「先优化业务，再调 JVM」——Arthas + MAT + GC 日志分析是排查三板斧

## 和其他域的关系

```
03_JVM ←→ 01_Java核心   （类加载 → 反射；字符串 → 常量池）
03_JVM ←→ 02_并发       （运行时数据区 → 线程私有区；synchronized → 字节码层面）
03_JVM ←→ 04_Spring    （类加载 → Bean生命周期；JIT → Spring启动优化）
03_JVM ←→ 09_计算机基础  （内存管理 → 操作系统虚拟内存）
```
