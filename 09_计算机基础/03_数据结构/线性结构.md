# 线性结构

> 线性结构是数据元素之间存在一对一关系的数据结构，是最基础也最常用的数据结构。

---

## 一、数组（Array）

### 1.1 基本概念

数组是内存中**连续存储**的相同类型元素的集合。通过下标（索引）直接访问任意元素。

```
内存布局（以 int[5] 为例，每个 int 4 字节）：

地址:    0x1000  0x1004  0x1008  0x100C  0x1010
数据:    [  3  ] [  7  ] [  1  ] [  9  ] [  5  ]
索引:      [0]     [1]     [2]     [3]     [4]

访问公式：address(base) + index × element_size
例：a[3] = 0x1000 + 3 × 4 = 0x100C → 9
```

### 1.2 操作复杂度

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 按索引访问 | O(1) | 直接地址计算 |
| 按值查找 | O(n) | 线性扫描 |
| 尾部插入 | O(1) 均摊 | 动态数组偶尔扩容 |
| 头部插入 | O(n) | 所有元素后移 |
| 中间插入 | O(n) | 部分元素后移 |
| 删除 | O(n) | 元素前移 |
| 扩容 | O(n) | 数据拷贝到新数组 |

### 1.3 动态数组扩容机制（ArrayList）

```java
// ArrayList 扩容核心逻辑（JDK 8）
private void grow(int minCapacity) {
    int oldCapacity = elementData.length;
    int newCapacity = oldCapacity + (oldCapacity >> 1); // 1.5 倍
    if (newCapacity < minCapacity)
        newCapacity = minCapacity;
    if (newCapacity > MAX_ARRAY_SIZE)
        newCapacity = hugeCapacity(minCapacity);
    elementData = Arrays.copyOf(elementData, newCapacity);
}
```

**为什么是 1.5 倍而不是 2 倍？**
- 1.5 倍：扩容后空间利用率约 2/3，浪费较少
- 2 倍：扩容后空间利用率约 1/2，浪费较多
- 黄金比例（1.618）更优，但 1.5 倍用位运算 `(oldCapacity >> 1)` 计算更高效

### 1.4 数组的 CPU Cache 友好性

```
CPU Cache Line 通常 64 字节

数组（连续内存）：
[elem0][elem1][elem2][elem3][elem4][elem5][elem6][elem7]
└─────────── 一个 Cache Line ──────────┘
一次加载 16 个 int（64B / 4B），顺序遍历 Cache 命中率极高

链表（离散内存）：
Node0 → Node1 → Node5 → Node3 → Node8
每个节点可能在不同 Cache Line，遍历时 Cache Miss 频繁
```

> **关键结论**：即使 ArrayList 插入删除是 O(n)，由于 CPU Cache 友好，在小到中等数据量下，性能往往优于 LinkedList。

## 二、链表（Linked List）

### 2.1 三种链表类型

```
单链表（Singly Linked List）：
Head → [data|next] → [data|next] → [data|next] → null

双向链表（Doubly Linked List）：
null ← [prev|data|next] ⇄ [prev|data|next] ⇄ [prev|data|next] → null

循环链表（Circular Linked List）：
双向循环：最后一个节点的 next 指回头节点
         头节点的 prev 指向尾节点
```

### 2.2 操作复杂度

| 操作 | 单链表 | 双向链表 | 说明 |
|------|--------|---------|------|
| 头部插入 | O(1) | O(1) | 修改 head 指针 |
| 尾部插入 | O(n)* | O(1)** | 单链表需要遍历找尾 |
| 头部删除 | O(1) | O(1) | — |
| 尾部删除 | O(n) | O(1) | 双向链表有 prev 指针 |
| 按值查找 | O(n) | O(n) | 线性扫描 |
| 按索引访问 | O(n) | O(n) | 无随机访问 |

> *单链表维护尾指针可实现 O(1) 尾部插入
> **双向链表维护尾指针可实现 O(1) 尾部插入和删除

### 2.3 链表反转（面试高频）

```java
// 迭代法（O(n) 时间, O(1) 空间）
public ListNode reverseList(ListNode head) {
    ListNode prev = null;
    ListNode curr = head;
    while (curr != null) {
        ListNode next = curr.next; // 暂存下一个
        curr.next = prev;          // 反转指针
        prev = curr;               // prev 前进
        curr = next;               // curr 前进
    }
    return prev;
}
```

### 2.4 判断链表是否有环

```java
// 快慢指针法（Floyd 判圈算法）
public boolean hasCycle(ListNode head) {
    if (head == null) return false;
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;       // 慢指针走 1 步
        fast = fast.next.next;  // 快指针走 2 步
        if (slow == fast) return true;
    }
    return false;
}
```

**原理**：快慢指针在有环链表中终将相遇。设环前长度 L，环长度 C：
- 慢指针进环时，快指针已在环中走了 L 步（距入口 L % C）
- 每轮快比慢多走 1 步，最多 C 步必然相遇

### 2.5 合并两个有序链表

```java
public ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0); // 哑节点技巧
    ListNode tail = dummy;
    while (l1 != null && l2 != null) {
        if (l1.val <= l2.val) {
            tail.next = l1;
            l1 = l1.next;
        } else {
            tail.next = l2;
            l2 = l2.next;
        }
        tail = tail.next;
    }
    tail.next = (l1 != null) ? l1 : l2;
    return dummy.next;
}
```

### 2.6 LRU 缓存实现

```java
// LRU = HashMap + 双向链表，O(1) get + O(1) put
class LRUCache {
    class DLinkedNode {
        int key, value;
        DLinkedNode prev, next;
    }

    private void addNode(DLinkedNode node) {
        node.prev = head;
        node.next = head.next;
        head.next.prev = node;
        head.next = node;
    }

    private void removeNode(DLinkedNode node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }

    private void moveToHead(DLinkedNode node) {
        removeNode(node);
        addNode(node);
    }

    // get: O(1) 通过 HashMap 找到节点 + 移到头部
    // put: O(1) HashMap 写入 + 链表头部插入 + 尾部淘汰
}
```

## 三、栈（Stack）

### 3.1 核心特征

**LIFO（Last In First Out）——后进先出**

```
操作         栈状态
push(A)      [A]
push(B)      [B, A]      ← B 在栈顶
push(C)      [C, B, A]
pop() → C    [B, A]
pop() → B    [A]
peek() → A   [A]         ← 查看栈顶但不移除
```

### 3.2 典型应用场景

| 场景 | 说明 |
|------|------|
| 函数调用栈 | 方法调用链、递归 |
| 括号匹配 | `({[]})` 有效性判断 |
| 表达式求值 | 中缀 → 后缀（逆波兰表达式） |
| 浏览器前进/后退 | 访问历史栈 |
| 撤销/重做 | 编辑器操作栈 |
| DFS 深度优先搜索 | 隐式使用栈 |

### 3.3 括号匹配

```java
public boolean isValid(String s) {
    Deque<Character> stack = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (c == '(' || c == '[' || c == '{') {
            stack.push(c);
        } else {
            if (stack.isEmpty()) return false;
            char top = stack.pop();
            if (c == ')' && top != '(') return false;
            if (c == ']' && top != '[') return false;
            if (c == '}' && top != '{') return false;
        }
    }
    return stack.isEmpty();
}
```

### 3.4 单调栈

```java
// 下一个更大元素（经典单调栈问题）
public int[] nextGreaterElement(int[] nums) {
    int n = nums.length;
    int[] result = new int[n];
    Arrays.fill(result, -1);
    Deque<Integer> stack = new ArrayDeque<>(); // 存索引

    for (int i = 0; i < n; i++) {
        while (!stack.isEmpty() && nums[i] > nums[stack.peek()]) {
            result[stack.pop()] = nums[i];
        }
        stack.push(i);
    }
    return result;
}
// 时间复杂度：O(n)，每个元素最多入栈出栈各一次
```

## 四、队列（Queue）

### 4.1 核心特征

**FIFO（First In First Out）——先进先出**

```
入队（enqueue）    队列          出队（dequeue）
A → [A]                            A →
B → [A, B]                          A →
C → [A, B, C]                       A → B
                                    B → C
```

### 4.2 队列变体

| 类型 | 特点 | 典型应用 |
|------|------|---------|
| 普通队列 | FIFO | BFS、任务调度 |
| 双端队列 | 两端都可入/出 | 滑动窗口、回文判断 |
| 优先队列 | 按优先级出队 | 任务调度、Dijkstra |
| 阻塞队列 | 满时等待 / 空时等待 | 生产者-消费者 |
| 循环队列 | 固定大小，头尾相连 | 缓冲区 |

### 4.3 循环队列

```java
class CircularQueue {
    private int[] data;
    private int head, tail, size, capacity;

    public boolean enqueue(int val) {
        if (isFull()) return false;
        data[tail] = val;
        tail = (tail + 1) % capacity; // 环形
        size++;
        return true;
    }

    public int dequeue() {
        if (isEmpty()) return -1;
        int val = data[head];
        head = (head + 1) % capacity;
        size--;
        return val;
    }
}
```

### 4.4 阻塞队列（Java 并发核心）

```java
// 生产者-消费者模式
BlockingQueue<String> queue = new ArrayBlockingQueue<>(100);

// 生产者
queue.put("task");     // 队列满时阻塞等待

// 消费者
String task = queue.take(); // 队列空时阻塞等待
```

| 实现 | 有界性 | 数据结构 | 锁策略 |
|------|--------|---------|--------|
| ArrayBlockingQueue | 有界 | 数组 | 一把 ReentrantLock + 两个 Condition |
| LinkedBlockingQueue | 可选有界 | 链表 | put 和 take 分别用两把锁 |
| SynchronousQueue | 零容量 | — | 直接交接，无缓冲 |
| PriorityBlockingQueue | 无界 | 堆 | 一把 ReentrantLock |
| DelayQueue | 无界 | 堆 | 元素到期才能取出 |

### 4.5 双端队列与滑动窗口

```java
// 滑动窗口最大值（经典双端队列应用）
public int[] maxSlidingWindow(int[] nums, int k) {
    int n = nums.length;
    int[] result = new int[n - k + 1];
    Deque<Integer> deque = new ArrayDeque<>(); // 存索引，单调递减

    for (int i = 0; i < n; i++) {
        // 移除超出窗口的元素
        while (!deque.isEmpty() && deque.peekFirst() <= i - k) {
            deque.pollFirst();
        }
        // 维护单调递减：移除比当前小的元素
        while (!deque.isEmpty() && nums[deque.peekLast()] < nums[i]) {
            deque.pollLast();
        }
        deque.offerLast(i);
        // 窗口形成后记录结果
        if (i >= k - 1) {
            result[i - k + 1] = nums[deque.peekFirst()];
        }
    }
    return result;
}
```

## 五、栈与队列的互相实现

### 栈实现队列

```java
class MyQueue {
    private Deque<Integer> inStack = new ArrayDeque<>();
    private Deque<Integer> outStack = new ArrayDeque<>();

    public void push(int x) {
        inStack.push(x);
    }

    public int pop() {
        if (outStack.isEmpty()) {
            while (!inStack.isEmpty()) {
                outStack.push(inStack.pop());
            }
        }
        return outStack.pop();
    }
    // 均摊时间复杂度：O(1)
}
```

### 队列实现栈

```java
class MyStack {
    private Queue<Integer> queue = new LinkedList<>();

    public void push(int x) {
        int size = queue.size();
        queue.offer(x);
        for (int i = 0; i < size; i++) {
            queue.offer(queue.poll()); // 把之前的元素全部移到 x 后面
        }
    }

    public int pop() {
        return queue.poll();
    }
}
```
