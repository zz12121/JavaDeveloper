# 克隆与拷贝

> Java 赋值 `a = b` 只拷贝引用，不拷贝对象。`Object.clone()` 默认做浅拷贝，而开发者期望的往往是深拷贝。理解这个 gap，是避免共享引用 bug 的关键。

---

## 这个问题为什么存在？

日常开发中经常需要"复制一个对象"：

```
场景：
- 备份数据：修改副本不影响原对象（如编辑器的撤销/重做）
- 线程安全：每个线程操作自己的对象副本（ThreadLocal 的原理）
- 缓存：返回对象的防御性拷贝，防止外部修改内部状态
- DTO 转换：Entity → VO 的数据复制
```

但 Java 的赋值操作**只拷贝引用，不拷贝对象**：

```java
List<String> a = new ArrayList<>(Arrays.asList("x", "y"));
List<String> b = a;         // 引用拷贝：a 和 b 指向同一个对象
b.add("z");
System.out.println(a);       // [x, y, z] ← a 也变了！
```

需要真正的"对象拷贝"时，Java 提供了 `clone()` 机制，但它设计上有明显缺陷。

---

## 它是怎么解决的？

### 三种"拷贝"层次

```
引用拷贝          浅拷贝                     深拷贝
a ──→ [obj]       a ──→ [obj₁]              a ──→ [obj₁]
                  b ──→ [obj₁]  共享引用      b ──→ [obj₂] 独立对象
                  同一个对象                   obj₁ 和 obj₂ 完全独立
```

### Object.clone()：浅拷贝

```java
class Profile implements Cloneable {
    String name;
    int[] scores;           // 引用类型

    @Override
    public Profile clone() {
        try {
            return (Profile) super.clone();
            // super.clone() 做了什么？
            // 1. 检查 this instanceof Cloneable（否则 CloneNotSupportedException）
            // 2. 分配新内存（native 方法，按对象大小分配）
            // 3. 逐字段内存拷贝：
            //    - name: 拷贝引用（String 是不可变的，没问题）
            //    - scores: 拷贝引用（两个对象共享同一个数组！）
        } catch (CloneNotSupportedException e) {
            throw new AssertionError();
        }
    }
}
```

**浅拷贝的问题**：
```java
Profile a = new Profile();
a.scores = new int[]{90, 80};

Profile b = a.clone();
b.scores[0] = 100;
System.out.println(a.scores[0]);  // 100 ← a 也被改了！
```

### Cloneable 的设计缺陷

```java
// Cloneable 是一个空接口（标记接口）
public interface Cloneable {}
```

```
Cloneable 的设计问题：
1. 没有 clone() 方法签名 → 无法声明 Cloneable 类型变量调用 clone()
2. protected → 只有子类（且在同包）才能调用，跨包不行
3. 返回 Object → 需要强制类型转换
4. 不抛异常的约定靠文档，编译器不检查
→ Effective Java 建议：不要用 Cloneable
```

**推荐替代方案**：

```java
// 推荐方式 1：拷贝构造器
public Profile(Profile source) {
    this.name = source.name;                // String 不可变，共享安全
    this.scores = source.scores.clone();  // 手动深拷贝
}

// 推荐方式 2：拷贝工厂方法
public static Profile copyOf(Profile source) {
    return new Profile(source.name, source.scores.clone());
}
```

## 深入原理

### 深拷贝实现方式

#### 方式 1：手动递归拷贝

```java
class Profile implements Cloneable {
    String name;
    int[] scores;
    Address address;  // 引用类型，也需要深拷贝

    @Override
    public Profile clone() {
        Profile copy = (Profile) super.clone();
        copy.scores = this.scores.clone();                  // 拷贝数组
        copy.address = this.address != null ? this.address.clone() : null;
        return copy;
    }
}
```

**优点**：性能好，可控。**缺点**：每个引用类型字段都要手动处理，容易遗漏。

#### 方式 2：序列化深拷贝

```java
public static <T extends Serializable> T deepClone(T obj) {
    try {
        ByteArrayOutputStream bos = new ByteArrayOutputStream();
        ObjectOutputStream oos = new ObjectOutputStream(bos);
        oos.writeObject(obj);
        oos.close();

        ObjectInputStream ois = new ObjectInputStream(
            new ByteArrayInputStream(bos.toByteArray())
        );
        @SuppressWarnings("unchecked")
        T copy = (T) ois.readObject();
        ois.close();
        return copy;
    } catch (IOException | ClassNotFoundException e) {
        throw new RuntimeException("深拷贝失败", e);
    }
}
```

**优点**：自动处理所有引用类型，支持循环引用（ObjectOutputStream 内部维护已写入对象的引用表）。**缺点**：所有类必须实现 Serializable，性能较差（序列化/反序列化开销大）。

#### 方式 3：JSON 序列化

```java
ObjectMapper mapper = new ObjectMapper();
String json = mapper.writeValueAsString(original);
MyClass copy = mapper.readValue(json, MyClass.class);
```

**优点**：不需要实现 Serializable，可读性好。**缺点**：会丢失 transient 字段，不能处理循环引用（默认配置下），性能低于手动拷贝。

#### 方式 4：BeanUtils.copyProperties

```java
BeanUtils.copyProperties(source, target);  // Spring 提供的工具
```

**注意：这是浅拷贝！** 只是同名同类型字段的值复制。如果字段是引用类型，仍然共享同一个对象。

### ArrayList 的 clone() 是深拷贝吗？

```java
// ArrayList 源码
public Object clone() {
    try {
        ArrayList<?> v = (ArrayList<?>) super.clone();
        v.elementData = Arrays.copyOf(elementData, size);
        v.modCount = 0;
        return v;
    } catch (CloneNotSupportedException e) {
        throw new InternalError(e);
    }
}
```

**半深拷贝**：
- 新建了一个独立的数组（`Arrays.copyOf` 创建新数组）
- 但数组中的元素是引用拷贝 → 如果 `ArrayList<Object>` 里的 Object 是可变对象，修改副本中的元素仍会影响原对象

```
ArrayList<String>（String 不可变）→ clone 后完全独立
ArrayList<StringBuilder>（StringBuilder 可变）→ clone 后元素仍共享
```

### 不可变对象天然"安全拷贝"

如果一个类的所有字段都是 `final` 且引用不可变对象（String、Integer 等），那么不需要深拷贝——任何"拷贝"方式都是安全的，因为对象本身就不能被修改。

这就是为什么 `String` 设计成不可变的一大好处：**无需拷贝即可安全共享。**

## 正确使用方式

### 1. 优先用拷贝构造器，不用 Cloneable

```java
// ❌ 不好：Cloneable 设计缺陷多
class Profile implements Cloneable {
    @Override
    public Profile clone() { ... }
}

// ✅ 好：拷贝构造器，清晰、类型安全
class Profile {
    private final String name;
    private final int[] scores;

    // 拷贝构造器
    public Profile(Profile source) {
        this.name = source.name;
        this.scores = source.scores.clone();
    }
}
```

### 2. 返回对象时做防御性拷贝（入参也要）

```java
class Profile {
    private final int[] scores;

    // 入参防御
    public Profile(int[] scores) {
        this.scores = scores.clone();  // 不持有外部传入的引用
    }

    // 返回防御
    public int[] getScores() {
        return scores.clone();  // 不暴露内部数组引用
    }
}
```

详见 [[01_Java核心/02_面向对象/封装|封装]] 中关于防御性拷贝的相关内容。

### 3. 不可变对象直接共享

```java
// String、Integer 等不可变对象，不需要拷贝
String s1 = "hello";
String s2 = s1;  // 引用拷贝完全安全
```

## 边界情况和坑

### 1. 使用 clone() 前必须检查 Cloneable

```java
Object obj = new Object();
obj.clone();  // CloneNotSupportedException！
// Object 类没有实现 Cloneable
```

只有实现（或继承实现）了 `Cloneable` 的类才能调用 `clone()`。

### 2. 深拷贝的循环引用问题

```java
class Node implements Serializable {
    Node next;  // 循环引用：A → B → A

    // 序列化深拷贝：✅ 正常工作
    // ObjectOutputStream 内部维护了已写入对象表，不会无限递归

    // 手动递归深拷贝：❌ 需要额外处理，否则栈溢出
    Node deepCopy(Node source, Map<Node, Node> visited) { ... }
}
```

手动实现深拷贝时，**必须处理循环引用**，否则会无限递归直到栈溢出。用 `IdentityHashMap` 记录已拷贝对象。

### 3. transient 字段在不同深拷贝方式中的行为

```java
class Data implements Serializable {
    transient int tempValue;  // 序列化时会跳过

    Data deepCopy() throws Exception {
        // 序列化方式：tempValue 不会被拷贝（值为默认值 0）
        // 手动拷贝：可以有选择地拷贝 transient 字段
    }
}
```

### 4. 数组 clone() vs System.arraycopy()

```java
int[] a = {1, 2, 3};

int[] b = a.clone();                  // ✓ 简洁，返回同类型数组
int[] c = new int[a.length];
System.arraycopy(a, 0, c, 0, a.length);  // ✓ 更灵活，可部分拷贝
```

`System.arraycopy()` 是 native 方法，性能通常比手写循环好。但如果只是完整拷贝数组，`clone()` 更简洁。
