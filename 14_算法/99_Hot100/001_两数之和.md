## 题干

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出**和为目标值** `target` 的那两个整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，且同样的元素不能被重复利用。

### 示例

```
输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：因为 nums[0] + nums[1] == 9 ，返回 [0, 1] 。
```

#### 我的解题答案：
```java
private static int[] getIndex(int[] a, int target) {  
    if (a == null) {  
        return new int[0];  
    }  
    for (int i =0 ; i < a.length; i++) {  
        int b = target - a[i];  
        for (int j = i + 1; j< a.length; j++) {  
            if (a[j] == b) {  
                return new int[] {i, j};  
            }  
        }  
    }  
    return new int[0];  
}
```
这段代码的逻辑是：
1. 判断数组是否为空，空则返回空数组。
2. 外层循环遍历每个元素a[i]，计算差值b = target - a[i]。
3. 内层循环从i+1开始遍历后续元素，查找是否有元素等于b。
4. 找到则返回下标对(i, j)，否则最后返回空数组。
评价：
这种方法的时间复杂度为O(n^2)，空间复杂度为O(1)。
虽然正确，但在LeetCode上提交可能会因为超时而无法通过所有测试用例（特别是数据量大时）。

#### 更优解法：使用哈希表（HashMap）
思路：遍历数组，将每个元素的值和索引存入哈希表。同时，在遍历时检查是否存在target - current_value的键在哈希表中。
注意：题目说同一个元素不能重复使用，所以我们在遍历到第i个元素时，先检查哈希表中是否存在key=target-nums[i]，存在则返回，然后再把当前元素放入哈希表，这样避免自己和自己匹配（例如target=6，当前元素是3，如果先存再查，就会查到刚存的自己）。
或者也可以先不存当前元素，而是先检查之前已经存储的元素中是否有满足条件的，这样可以自然避免重复使用同一元素。
优化后的代码（时间复杂度O(n)，空间复杂度O(n)）：
```java
private static int[] getIndexHashMap(int[] a, int target) {  
    if (a == null) {  
        return new int[0];  
    }  
    HashMap<Integer,Integer> targetHashMap = new HashMap();  
    for (int i =0 ; i < a.length; i++) {  
        if (targetHashMap.containsKey(target-a[i])) {  
            return new int[] {i, targetHashMap.get(target-a[i])};  
        }  
        targetHashMap.put(a[i], i);  
    }  
    return new int[0];  
}
```
**核心思路**：牺牲 O(n)的空间换取时间，用哈希表记录“已遍历过的数值及其下标”。每遍历到一个数 `nums[i]`，就检查 `target - nums[i]`是否已在表中，若存在则直接返回结果。
