# 上下文刷新

## 这个问题为什么存在？

> Spring 容器启动不是「一步完成」的——要先加载 Bean 定义、再处理后置处理器、再实例化单例 Bean... 如果把这些步骤写死在一个方法里，无法扩展（比如 Spring Boot 要在某个步骤启动 Web 服务器）。

> **`refresh()` 方法** 是 Spring 容器启动的「总指挥」，把启动流程拆成 **12 个步骤**，每个步骤可以独立扩展。

**核心问题**：`refresh()` 里到底做了什么？

---

## 它是怎么解决问题的？

### refresh() 的 12 个步骤（完整版）

```java
// AbstractApplicationContext.refresh()
public void refresh() throws BeansException, IllegalStateException {
    synchronized (this.startupShutdownMonitor) {

        // 【1】准备刷新（校验、初始化）
        prepareRefresh();

        // 【2】创建/刷新 BeanFactory（从 XML/注解 加载 BeanDefinition）
        ConfigurableListableBeanFactory beanFactory = obtainFreshBeanFactory();

        // 【3】配置 BeanFactory（类加载器、SpEL、属性编辑器等）
        prepareBeanFactory(beanFactory);

        try {
            // 【4】留给子类扩展（Spring Boot 在这里启动 Web 服务器）
            postProcessBeanFactory(beanFactory);

            // 【5】★ 执行 BeanFactoryPostProcessor（修改 BeanDefinition）
            invokeBeanFactoryPostProcessors(beanFactory);

            // 【6】★ 注册 BeanPostProcessor（初始化前后的拦截器）
            registerBeanPostProcessors(beanFactory);

            // 【7】初始化 MessageSource（国际化）
            initMessageSource();

            // 【8】初始化事件广播器（Spring 事件机制）
            initApplicationEventMulticaster();

            // 【9】★ 留给子类扩展（Spring Boot 在这里启动 Tomcat）
            onRefresh();

            // 【10】注册监听器
            registerListeners();

            // 【11】★ 实例化所有单例 Bean（核心！）
            finishBeanFactoryInitialization(beanFactory);

            // 【12】完成刷新（发布 ContextRefreshedEvent）
            finishRefresh();
        }
        // ... catch / finally ...
    }
}
```

> **面试高频**：`refresh()` 的 12 个步骤是必考题，尤其是第 5、6、9、11 步。

---

### 第 5 步详解：invokeBeanFactoryPostProcessors()

**BeanFactoryPostProcessor** 可以**修改 BeanDefinition**（在 Bean 实例化之前）。

```java
// 执行顺序：
// 1. BeanDefinitionRegistryPostProcessor.postProcessBeanDefinitionRegistry()
//    （可以注册新的 BeanDefinition）
// 2. BeanFactoryPostProcessor.postProcessBeanFactory()
//    （可以修改已有的 BeanDefinition）

// 举例：ConfigurationClassPostProcessor
//   → 扫描 @Configuration 类
//   → 解析 @Bean、@ComponentScan、@Import
//   → 注册新的 BeanDefinition 到容器
```

**Spring Boot 自动配置就在这里发生**：

```
invokeBeanFactoryPostProcessors()
    ↓
ConfigurationClassPostProcessor 执行
    ↓
解析 @SpringBootApplication
    ↓
找到所有 @Configuration 类
    ↓
解析 @Bean 方法 → 注册 BeanDefinition
    ↓
解析 @EnableAutoConfiguration
    ↓
加载 META-INF/spring Aut oConfiguration.imports
    ↓
注册所有自动配置类的 BeanDefinition
```

---

### 第 6 步详解：registerBeanPostProcessors()

**BeanPostProcessor** 在 **Bean 初始化前后** 执行（AOP 代理就是在这里创建的）。

```java
// BeanPostProcessor 的两个方法：
public interface BeanPostProcessor {
    // 在 @PostConstruct / InitializingBean.afterPropertiesSet() 之前执行
    Object postProcessBeforeInitialization(Object bean, String beanName);

    // 在 @PostConstruct / InitializingBean.afterPropertiesSet() 之后执行
    Object postProcessAfterInitialization(Object bean, String beanName);
}
```

**AOP 代理的创建时机**：

```
创建 Bean 实例（doCreateBean）
    ↓
populateBean()（依赖注入）
    ↓
initializeBean()
    ├── 1. invokeAwareMethods()         （BeanNameAware 等）
    ├── 2. postProcessBeforeInitialization()  （@PostConstruct 在这里）
    ├── 3. afterPropertiesSet()           （InitializingBean）
    ├── 4. init-method                  （自定义 init 方法）
    └── 5. postProcessAfterInitialization()   （★ AOP 代理在这里创建！）
```

> **关键**：`AnnotationAwareAspectJAutoProxyCreator`（一个 `BeanPostProcessor`）  
> 在第 5 步（`postProcessAfterInitialization()`）中，判断 Bean 是否匹配切面，  
> 如果是 → 返回代理对象（替换原始 Bean）。

---

### 第 9 步详解：onRefresh()（Spring Boot 关键扩展点）

**问题**：Spring Boot 要在容器刷新时启动 Tomcat，但 `AbstractApplicationContext` 不知道具体要启动什么。

**解法**：模板方法模式 —— `onRefresh()` 留给子类实现。

```java
// AbstractApplicationContext.onRefresh() — 空实现
protected void onRefresh() throws BeansException {
    // 留给子类扩展
}

// Spring Boot 的 ServletWebServerApplicationContext.onRefresh()
protected void onRefresh() {
    super.onRefresh();
    // ★ 在这里创建并启动 Tomcat / Jetty / Undertow
    createWebServer();
}

// createWebServer()
private void createWebServer() {
    ServletWebServerFactory factory = getWebServerFactory();
    this.webServer = factory.getWebServer(getSelfInitializer());
    // Tomcat 在新线程中启动，监听端口
}
```

---

### 第 11 步详解：finishBeanFactoryInitialization()

**问题**：所有单例 Bean 什么时候创建？

**答案**：这一步！

```java
// DefaultListableBeanFactory.preInstantiateSingletons()
public void preInstantiateSingletons() {
    // 1. 拿到所有 BeanDefinition 的名称
    List<String> beanNames = new ArrayList<>(this.beanDefinitionNames);

    // 2. 遍历，逐个创建（非懒加载的单例）
    for (String beanName : beanNames) {
        RootBeanDefinition bd = getMergedLocalBeanDefinition(beanName);

        // 3. 只处理：非抽象 && 是单例 && 非懒加载
        if (!bd.isAbstract() && bd.isSingleton() && !bd.isLazyInit()) {
            // 4. 创建 Bean（getBean() → doCreateBean()）
            getBean(beanName);
        }
    }
}
```

> **这一步完成后，所有单例 Bean 都创建完毕**，容器就绪。

## 它和相似方案的本质区别是什么？

### XML 配置启动 vs 注解配置启动

| | `ClassPathXmlApplicationContext` | `AnnotationConfigApplicationContext` |
|---|---|---|
| 配置源 | XML 文件 | Java 类（`@Configuration`） |
| BeanDefinition 读取器 | `XmlBeanDefinitionReader` | `AnnotatedBeanDefinitionReader` |
| 扫描 | 不支持（XML 里配 `<context:component-scan>`） | `ClassPathBeanDefinitionScanner` |
| Spring Boot | ❌ 不用 | ✅ 使用 |

**共同之处**：最终都调用 `AbstractApplicationContext.refresh()`。

## 正确使用方式

### 1. 自定义 BeanFactoryPostProcessor（修改 BeanDefinition）

```java
@Component
public class MyBeanFactoryPostProcessor implements BeanFactoryPostProcessor {

    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) {
        // 拿到所有 BeanDefinition，进行修改
        String[] beanNames = beanFactory.getBeanDefinitionNames();
        for (String beanName : beanNames) {
            BeanDefinition bd = beanFactory.getBeanDefinition(beanName);
            // 例如：给所有 Bean 加一个初始化方法
            if (bd instanceof GenericBeanDefinition) {
                bd.setInitMethodName("myInit");
            }
        }
    }
}
```

---

### 2. 自定义 BeanPostProcessor（AOP 之外的代理）

```java
@Component
public class MyBeanPostProcessor implements BeanPostProcessor, Ordered {

    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) {
        // 对特定 Bean 创建代理（不通过 AOP 切面）
        if (bean instanceof UserService) {
            return Proxy.newProxyInstance(...);
        }
        return bean;  // 不需要代理，原样返回
    }

    @Override
    public int getOrder() {
        return 0;  // 越小越先执行
    }
}
```

---

### 3. 监听 ContextRefreshedEvent（容器启动完成后执行）

```java
@EventListener
public void onApplicationEvent(ContextRefreshedEvent event) {
    // 容器启动完成后，执行一些初始化逻辑
    log.info("Spring 容器启动完成！");
    // 例如：预热缓存、加载字典表...
}
```

> **注意**：如果有父子容器（如 Spring MVC），`ContextRefreshedEvent` 会发布两次。  
> 解决：判断 `event.getApplicationContext() == this.applicationContext`。

## 边界情况和坑

### 1. refresh() 只能调用一次

```java
// ❌ 重复调用 refresh()
AnnotationConfigApplicationContext ctx = new AnnotationConfigApplicationContext();
ctx.register(AppConfig.class);
ctx.refresh();  // 第一次：正常
ctx.refresh();  // 抛异常：IllegalStateException（已激活）
```

**解决**：用 `close()` 关闭后再 refresh，或者直接用 Spring Boot（自动管理生命周期）。

---

### 2. BeanFactoryPostProcessor 中不能引用 Bean

```java
// ❌ 错误：BeanFactoryPostProcessor 执行时，Bean 还没创建！
@Component
public class MyBeanFactoryPostProcessor implements BeanFactoryPostProcessor {

    @Autowired  // ❌ 这个字段注入不生效（post processor 本身也是 Bean）
    private UserService userService;

    @Override
    public void postProcessBeanFactory(...) {
        userService.doSomething();  // NPE！
    }
}
```

> **原因**：`BeanFactoryPostProcessor` 在 Bean 实例化之前执行，  
> 此时 `userService` 还没创建，注入不进去。

---

### 3. onRefresh() 和 Bean 初始化的顺序

```
onRefresh()（第 9 步）
    → 启动 Tomcat
    ↓
finishBeanFactoryInitialization()（第 11 步）
    → 创建所有单例 Bean
```

> **Tomcat 先启动，Bean 后创建** → 如果 Bean 初始化很慢，Tomcat 已经接受了请求，但 Bean 还没 ready → 报错。  
> **解决**：配置优雅停机 + 健康检查（`/actuator/health`）。

---

## 我的理解

**`refresh()` 的本质**：Spring 容器启动的「总指挥」，把启动流程拆成 12 步，每步可扩展。

**面试最高频**：

1. `refresh()` 的 12 个步骤（背诵！）
2. 第 5 步：`invokeBeanFactoryPostProcessors()`（自动配置在这里发生）
3. 第 6 步：`registerBeanPostProcessors()`（AOP 代理在这里注册）
4. 第 9 步：`onRefresh()`（Spring Boot 在这里启动 Tomcat）
5. 第 11 步：`finishBeanFactoryInitialization()`（所有单例 Bean 在这里创建）

**记忆口诀**：「准备 → 加载 → 配置 → 扩展 → 后处理 → 注册监听器 → 实例化 → 完成」
