# 百万级定时任务调度平台设计

> 支持百万级定时任务的调度平台，核心是 **时间轮（Timing Wheel）+ 分片调度 + 持久化 + 集群选举**。Netty 的 HashedWheelTimer 是经典实现，大规模场景需要分层时间轮。

## 场景 A：时间轮（HashedWheelTimer）原理

### Netty HashedWheelTimer
```
时间轮类似时钟：
- 轮盘有 N 个槽（默认 512）
- 每个槽是一个双向链表，存到期任务
- 指针每秒跳一格，跳到哪个槽就执行那个槽的任务
- 层数（round）：任务到期时间 > 一轮时间，需要多轮
```

```java
// Netty HashedWheelTimer 使用
HashedWheelTimer timer = new HashedWheelTimer(
    new DefaultThreadFactory("timer"),
    100,                           // tick 间隔（毫秒）
    TimeUnit.MILLISECONDS,
    512                            // 槽数
);

timer.newTimeout(timeout -> {
    System.out.println("5秒后执行");
}, 5, TimeUnit.SECONDS);
```

### 时间复杂度
| 操作 | 时间复杂度 |
|------|------------|
| 添加任务 | O(1) |
| 取消任务 | O(1)（从链表中删除）|
| 触发任务 | O(n)（n = 槽数，但每个槽的任务数很少）|

## 场景 B：分层时间轮（大规模场景）

### 问题
```
单层时间轮：如果要支持「1 小时后执行」→ 轮盘需要很大 → 内存浪费
解决：分层时间轮（类似时钟的 秒/分/时）
```

### Kafka 分层时间轮实现
```
Level 1：秒级时间轮（tick=1s, wheelSize=60）→ 覆盖 60 秒
Level 2：分钟级时间轮（tick=60s, wheelSize=60）→ 覆盖 1 小时
Level 3：小时级时间轮（tick=3600s, wheelSize=24）→ 覆盖 1 天

任务到期时间 > 当前轮覆盖范围 → 升级到上一级时间轮
任务到期时间 <= 当前轮覆盖范围 → 降级到下一级时间轮
最终到达 Level 1，到期执行
```

```java
// Kafka TimingWheel 核心思想（简化）
public class TimingWheel {
    private final long tickMs;
    private final int wheelSize;
    private final long interval;  // tickMs * wheelSize
    private final ArrayDeque<Task>[] slots;
    private TimingWheel overflowWheel;  // 上级时间轮
    
    public void add(Task task) {
        if (task.expireMs < currentTime + interval) {
            // 能放在当前轮 → 放到对应槽
            int slot = (int)((task.expireMs / tickMs) % wheelSize);
            slots[slot].add(task);
        } else {
            // 放不下 → 放上级时间轮
            overflowWheel.add(task);
        }
    }
}
```

## 场景 C：集群调度（防重复执行）

### 问题
```
多台调度器同时工作 → 同一任务被多次触发 → 重复执行
```

### 解决方案：集群选举 + 分片
```java
// 1. ZooKeeper 选主（只有 Leader 触发任务）
public class LeaderSelectorListener implements LeaderLatchListener {
    @Override
    public void takeLeadership() throws Exception {
        // 成为 Leader → 开始调度
        scheduler.start();
        // 阻塞直到失去领导权
        Thread.sleep(Long.MAX_VALUE);
    }
}

// 2. 分片（多台机器同时工作，但各管各的）
// 更适合大规模场景：每台机器管一部分任务
public class ShardingScheduler {
    private int shardIndex;  // 当前机器分片
    private int shardTotal; // 总分片数
    
    public void schedule() {
        List<Task> tasks = taskDao.selectAll();
        for (Task task : tasks) {
            // 任务 ID 取模分片
            if (task.getId() % shardTotal == shardIndex) {
                execute(task);
            }
        }
    }
}
```
