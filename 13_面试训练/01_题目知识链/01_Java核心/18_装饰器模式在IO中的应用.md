## 参考要点（盲答后再看）


**装饰器模式在 IO 中的体现**：
```
InputStream（抽象组件）
  └── FileInputStream（节点流，具体组件）
  └── ByteArrayInputStream（节点流）
  └── FilterInputStream（装饰器抽象类）
        └── BufferedInputStream（具体装饰器，加缓冲）
        └── DataInputStream（具体装饰器，加类型读取）
        └── PushbackInputStream（具体装饰器，可回推）
```

**为什么不用继承**：
- 继承是静态的，组合是动态的
- 如果用继承：BufferedFileInputStream、BufferedByteArrayInputStream、BufferedPipedInputStream……类爆炸
- 用装饰器：一个 `BufferedInputStream` 可以包装任意 `InputStream`

**FilterInputStream 的角色**：
```java
// 典型的装饰器结构
public class FilterInputStream extends InputStream {
    protected volatile InputStream in;  // 被装饰的对象
    protected FilterInputStream(InputStream in) { this.in = in; }
    // 默认把所有方法委托给 in
}
```

**节点流 vs 处理流**：
| 类型 | 作用 | 例子 |
|------|------|------|
| 节点流 | 直接连接数据源 | FileInputStream, ByteArrayInputStream |
| 处理流 | 包装另一个流，增加功能 | BufferedInputStream, DataInputStream |

**常见错误**：
```java
// ❌ 忘了用装饰器，直接读文件——慢
FileInputStream fis = new FileInputStream("file.txt");
int b; while ((b = fis.read()) != -1) { ... }  // 每次读1字节，超慢

// ✅ 用 BufferedInputStream 包装
BufferedInputStream bis = new BufferedInputStream(new FileInputStream("file.txt"));
```
