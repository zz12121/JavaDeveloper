# 数据导出中心设计

> 后台管理系统用户要导出大批量 Excel/CSV，直接查库写 Excel → 接口超时、OOM、Tomcat 线程池被打满。需要一个统一的导出中心：异步导出、流式查询、文件存储、下载管理、导出限流。

---

## 业务背景

```
现状痛点：
  用户在后台点「导出」→ 接口直接查库 + 生成 Excel → 返回文件流
  → 数据量 1 万条以内还行
  → 超过 5 万条：接口超时（30 秒）
  → 超过 10 万条：JVM OOM（全部数据加载到内存）
  → 多人同时导出：Tomcat 200 个线程全占满 → 其他接口也卡

真实场景：
  运营导出订单数据：50 万条，20 个字段
  财务导出对账数据：100 万条，30 个字段
  客服导出用户反馈：10 万条，含长文本字段
  HR 导出工资明细：5000 条，但涉及敏感数据需要权限控制

需求：
  1. 异步导出：点导出后立刻返回，后台执行，完成后通知
  2. 流式处理：不能把全部数据加载到内存
  3. 导出记录：谁在什么时候导出了什么，可追溯
  4. 权限控制：敏感数据需要审批或脱敏
  5. 并发限制：同时最多 N 个导出任务
  6. 大文件下载：支持分片下载和断点续传
```

```java
// ===== 1. 创建导出任务（同步接口，快速返回） =====
@RestController
@RequestMapping("/api/export")
public class ExportController {

    @Autowired
    private ExportService exportService;

    /**
     * 创建导出任务
     * 用户点「导出」调用此接口，立即返回任务编号
     */
    @PostMapping("/create")
    public Result<ExportTaskDTO> createExport(@RequestBody ExportCreateRequest request) {
        // 1. 权限校验（用户有没有导出该业务数据的权限）
        checkPermission(request.getBizType());

        // 2. 创建任务记录
        ExportTask task = exportService.createTask(
            request.getBizType(),
            getCurrentUserId(),
            request.getFileName(),
            request.getFileFormat(),
            request.getQueryParams()
        );

        // 3. 异步执行（发 MQ）
        exportService.submitTask(task.getId());

        // 4. 立刻返回
        return Result.success(ExportTaskDTO.from(task));
    }

    /**
     * 查询导出进度
     */
    @GetMapping("/progress/{taskId}")
    public Result<ExportProgressDTO> getProgress(@PathVariable Long taskId) {
        ExportTask task = exportService.getTask(taskId);
        return Result.success(ExportProgressDTO.from(task));
    }

    /**
     * 下载文件
     */
    @GetMapping("/download/{taskId}")
    public void download(@PathVariable Long taskId, HttpServletResponse response)
            throws IOException {
        ExportTask task = exportService.getTask(taskId);
        if (!"COMPLETED".equals(task.getStatus())) {
            response.sendError(400, "文件未生成");
            return;
        }

        // 302 重定向到 OSS 预签名 URL（不走应用服务器带宽）
        String signedUrl = ossService.generatePresignedUrl(task.getFileUrl(), 3600);
        response.sendRedirect(signedUrl);
    }
}
```

```java
    public void onMessage(String message) {
        Long taskId = Long.parseLong(message);
        try {
            exportEngine.execute(taskId);
        } catch (Exception e) {
            log.error("导出任务执行失败: taskId={}", taskId, e);
            exportEngine.markFailed(taskId, e.getMessage());
        }
    }
}
```

```java
// ===== 3. 导出执行引擎（核心） =====
@Service
public class ExportEngine {

    @Autowired
    private ExportTaskMapper taskMapper;

    @Autowired
    private ExportQueryProvider queryProvider;

    @Autowired
    private OssService ossService;

    @Autowired
    private StringRedisTemplate redis;

    /**
     * 流式导出核心逻辑
     * 关键：不能把所有数据加载到内存，要逐条读、逐行写
     */
    public void execute(Long taskId) throws Exception {
        // 1. 更新状态为 RUNNING
        ExportTask task = taskMapper.selectById(taskId);
        taskMapper.updateStatus(taskId, "RUNNING");
        task.setStartTime(new Date());

        // 2. 流式查询 + 流式写入
        String tempFile = null;
        try {
            tempFile = "/tmp/export_" + taskId + "." + task.getFileFormat().toLowerCase();

            // 根据业务类型获取对应的 Mapper 方法
            ExportQueryHandler handler = queryProvider.getHandler(task.getBizType());
            Map<String, Object> params = JSON.parseObject(task.getQueryParams());

            if ("XLSX".equals(task.getFileFormat())) {
                exportToExcel(taskId, handler, params, tempFile);
            } else {
                exportToCsv(taskId, handler, params, tempFile);
            }

            // 3. 上传到 OSS
            String ossUrl = ossService.upload(
                "export/" + task.getBizType() + "/" + task.getTaskNo() + "."
                    + task.getFileFormat().toLowerCase(),
                new File(tempFile)
            );

            // 4. 更新任务状态
            File file = new File(tempFile);
            taskMapper.updateCompleted(taskId, ossUrl, file.length(), new Date());

        } finally {
            // 5. 清理临时文件
            if (tempFile != null) {
                new File(tempFile).delete();
            }
            // 6. 清理进度缓存
            redis.delete("export:progress:" + taskId);
        }
    }

    /**
     * 流式导出到 Excel（EasyExcel）
     * 关键：用 EasyExcel 的流式 API，不会把全部数据加载到内存
     */
    private void exportToExcel(Long taskId, ExportQueryHandler handler,
                               Map<String, Object> params, String filePath) {
        // EasyExcel 写入
        ExcelWriter excelWriter = EasyExcel.write(filePath, handler.getExcelModelClass()).build();
        WriteSheet writeSheet = EasyExcel.writerSheet("数据").build();

        // 流式查询（MyBatis Cursor）
        try (Cursor<Object> cursor = handler.queryCursor(params)) {
            List<Object> batch = new ArrayList<>(BATCH_SIZE);
            long count = 0;

            for (Object row : cursor) {
                batch.add(row);
                count++;

                if (batch.size() >= BATCH_SIZE) {
                    excelWriter.write(batch, writeSheet);
                    batch.clear();
                }

                // 每 5000 条更新一次进度
                if (count % 5000 == 0) {
                    updateProgress(taskId, count, -1);
                }
            }

            // 写入剩余数据
            if (!batch.isEmpty()) {
                excelWriter.write(batch, writeSheet);
            }

            updateProgress(taskId, count, count);  // totalCount = exportedCount = count
        } finally {
            excelWriter.finish();
        }
    }

    /**
     * 流式导出到 CSV（更省内存）
     */
    private void exportToCsv(Long taskId, ExportQueryHandler handler,
                             Map<String, Object> params, String filePath) throws IOException {
        try (BufferedWriter writer = new BufferedWriter(
                new OutputStreamWriter(new FileOutputStream(filePath), StandardCharsets.UTF_8))) {

            // 写表头
            writer.write(UnicodeBOM + String.join(",", handler.getColumnHeaders()));
            writer.newLine();

            // 流式查询
            try (Cursor<Object> cursor = handler.queryCursor(params)) {
                long count = 0;
                for (Object row : cursor) {
                    writer.write(handler.toCsvLine(row));
                    writer.newLine();
                    count++;

                    if (count % 5000 == 0) {
                        writer.flush();
                        updateProgress(taskId, count, -1);
                    }
                }
                updateProgress(taskId, count, count);
            }
        }
    }

    // CSV 文件加 BOM 头，防止 Excel 打开乱码
    private static final byte[] UnicodeBOM = {(byte) 0xEF, (byte) 0xBB, (byte) 0xBF};

    private static final int BATCH_SIZE = 2000;

    private void updateProgress(Long taskId, long exported, long total) {
        String key = "export:progress:" + taskId;
        redis.opsForHash().put(key, "exported", String.valueOf(exported));
        if (total > 0) {
            redis.opsForHash().put(key, "total", String.valueOf(total));
        }
        redis.expire(key, 2, TimeUnit.HOURS);
    }
}
```

## 核心设计三：MyBatis 流式查询

```java
// ===== 导出查询处理器接口 =====
public interface ExportQueryHandler {
    /** 流式查询，返回 Cursor（不一次性加载全部数据） */
    Cursor<Object> queryCursor(Map<String, Object> params);

    /** Excel 实体类 */
    Class<?> getExcelModelClass();

    /** CSV 表头 */
    String[] getColumnHeaders();

    /** 对象转 CSV 行 */
    String toCsvLine(Object row);
}

// ===== 订单导出实现 =====
@Component
public class OrderExportHandler implements ExportQueryHandler {

    @Autowired
    private OrderMapper orderMapper;

    @Override
    public Cursor<Object> queryCursor(Map<String, Object> params) {
        // 关键：使用 @Options(fetchSize = Integer.MIN_VALUE)
        // MySQL 的 Cursor 必须设置 fetchSize = Integer.MIN_VALUE 才能流式读取
        // 否则还是会把全部数据加载到内存
        return orderMapper.cursorExportOrder(
            (Date) params.get("startDate"),
            (Date) params.get("endDate"),
            (String) params.get("status"),
            (Long) params.get("merchantId")
        );
    }

    @Override
    public Class<?> getExcelModelClass() { return OrderExportDTO.class; }

    @Override
    public String[] getColumnHeaders() {
        return new String[]{"订单号", "用户ID", "商品名称", "金额", "状态", "下单时间"};
    }

    @Override
    public String toCsvLine(Object row) {
        OrderExportDTO o = (OrderExportDTO) row;
        return String.join(",",
            escapeCsv(o.getOrderNo()),
            escapeCsv(o.getUserId().toString()),
            escapeCsv(o.getProductName()),
            escapeCsv(o.getAmount().toString()),
            escapeCsv(o.getStatus()),
            escapeCsv(DateUtil.formatDateTime(o.getCreateTime()))
        );
    }

    private String escapeCsv(String field) {
        if (field == null) return "";
        if (field.contains(",") || field.contains("\"") || field.contains("\n")) {
            return "\"" + field.replace("\"", "\"\"") + "\"";
        }
        return field;
    }
}
```

```java
// ===== MyBatis Mapper =====
@Mapper
public interface OrderMapper {

    /**
     * 流式查询导出数据
     * 关键注解：fetchSize = Integer.MIN_VALUE
     * 这个值告诉 MySQL JDBC Driver 逐行读取，不缓存全部结果
     */
    @Select("SELECT id, order_no, user_id, product_name, amount, status, create_time " +
            "FROM `order` " +
            "WHERE create_time BETWEEN #{startDate} AND #{endDate} " +
            "  AND status = #{status} " +
            "ORDER BY id ASC")
    @Options(fetchSize = Integer.MIN_VALUE)
    Cursor<OrderExportDTO> cursorExportOrder(
        @Param("startDate") Date startDate,
        @Param("endDate") Date endDate,
        @Param("status") String status,
        @Param("merchantId") Long merchantId
    );
}
```

```
MyBatis Cursor 流式查询的关键：
  1. Mapper 方法返回类型必须是 Cursor<T>
  2. @Options(fetchSize = Integer.MIN_VALUE) 必须加
  3. 使用 try-with-resources 确保关闭（Cursor 实现了 Closeable）
  4. Cursor 必须在同一个 SqlSession 内使用，不能跨线程
  5. 查询期间数据库连接一直被占用，所以导出要控制并发数
```

## 核心设计四：并发控制与限流

```java
@Component
public class ExportConcurrencyLimiter {

    @Autowired
    private StringRedisTemplate redis;

    /**
     * 全局并发控制：最多同时执行 N 个导出任务
     * 用 Redis 信号量实现（分布式环境安全）
     */
    private static final int MAX_CONCURRENT = 5;
    private static final String SEMAPHORE_KEY = "export:semaphore";

    public boolean tryAcquire(String taskNo) {
        Long current = redis.opsForValue().increment(SEMAPHORE_KEY);
        redis.expire(SEMAPHORE_KEY, 1, TimeUnit.HOURS);  // 防止 key 永不过期

        if (current != null && current <= MAX_CONCURRENT) {
            // 记录占位
            redis.opsForSet().add("export:running", taskNo);
            return true;
        } else {
            // 超过并发限制，回退计数
            redis.opsForValue().decrement(SEMAPHORE_KEY);
            return false;
        }
    }

    public void release(String taskNo) {
        redis.opsForSet().remove("export:running", taskNo);
        redis.opsForValue().decrement(SEMAPHORE_KEY);
    }
}

/**
 * 用户级限流：单用户每天最多 N 次导出
 */
@Component
public class ExportUserRateLimit {

    @Autowired
    private StringRedisTemplate redis;

    private static final int MAX_EXPORTS_PER_DAY = 20;

    public boolean tryAcquire(Long userId) {
        String key = "export:user:daily:" + userId;
        Long count = redis.opsForValue().increment(key);
        if (count != null && count == 1) {
            redis.expire(key, 1, TimeUnit.DAYS);
        }
        return count != null && count <= MAX_EXPORTS_PER_DAY;
    }
}
```

## 核心设计五：敏感数据脱敏

```java
/**
 * 导出时的数据脱敏处理
 */
public class DataMaskUtil {

    /** 手机号脱敏：138****5678 */
    public static String maskPhone(String phone) {
        if (phone == null || phone.length() != 11) return phone;
        return phone.substring(0, 3) + "****" + phone.substring(7);
    }

    /** 身份证脱敏：110***********1234 */
    public static String maskIdCard(String idCard) {
        if (idCard == null || idCard.length() < 8) return idCard;
        return idCard.substring(0, 3) + "***********" + idCard.substring(idCard.length() - 4);
    }

    /** 银行卡脱敏：**** **** **** 5678 */
    public static String maskBankCard(String cardNo) {
        if (cardNo == null || cardNo.length() < 4) return cardNo;
        return "**** **** **** " + cardNo.substring(cardNo.length() - 4);
    }

    /** 姓名脱敏：张* / 张*明 */
    public static String maskName(String name) {
        if (name == null || name.length() <= 1) return name;
        if (name.length() == 2) return name.charAt(0) + "*";
        return name.charAt(0) + "*".repeat(name.length() - 2) + name.charAt(name.length() - 1);
    }
}

// 在导出 DTO 中使用 EasyExcel 注解脱敏

    }
```

## 生产踩坑记录

### 坑 1：MyBatis Cursor 查询时数据库连接泄漏

```
场景：导出过程中出现异常 → Cursor 没有关闭
     → 数据库连接一直被占用 → 连接池耗尽 → 所有接口超时

解决：
1. 必须用 try-with-resources 包裹 Cursor
2. 导出执行引擎加 finally 块，确保 Cursor 关闭
3. 监控数据库连接数，超过阈值告警
4. 设置 Cursor 超时时间：MyBatis 全局配置 defaultStatementTimeout
```

### 坑 2：CSV 导出 Excel 打开乱码

```
原因：Excel 默认用 ANSI 编码打开 CSV，Java 输出的是 UTF-8
     → 中文全部变成乱码

解决：在 CSV 文件开头写 BOM 头（EF BB BF）
     → Excel 看到 BOM 就知道是 UTF-8 编码
     → private static final byte[] UnicodeBOM = {(byte)0xEF, (byte)0xBB, (byte)0xBF};
```

### 坑 3：大文件上传 OSS 超时

```
场景：导出 500MB 的文件 → 上传 OSS 超时
     → 任务一直处于 RUNNING 状态

解决：
1. 分片上传：大文件拆成 5MB 的分片并行上传
2. 本地先压缩：CSV 压缩成 zip，体积减少 70%+
3. 设置合理的超时：单次上传 5 分钟超时，分片上传 30 分钟超时
```

### 坑 4：导出 SQL 慢查询

```
场景：导出条件没有索引 → 全表扫描 → DB CPU 100%
     → 导出跑了 20 分钟还在跑 → 其他业务受影响

解决：
1. 导出查询必须走索引（日期范围 + 状态）
2. 强制限制查询范围：最多导出最近 3 个月的数据
3. 超大数据量（> 100 万条）走异步跑批，不直接导出
4. DB 层面用读写分离，导出走从库
```

## 追问链

### 追问 1：EasyExcel 和 Apache POI 有什么区别？为什么选 EasyExcel？

> "核心区别在内存占用。POI 的 XSSFWorkbook 会把整个 Excel 加载到内存（100 万行大概占 2GB）。EasyExcel 基于 SAX 事件驱动，逐行解析和写入，内存占用只有几十 MB。导出用 EasyExcel 的流式 API（ExcelWriter），导入用 ReadListener，都是流式的。生产环境导出大数据量必须用 EasyExcel 或 POI 的 SXSSF（流式 API），绝不能用 XSSF。"

### 追问 2：导出 1000 万条数据怎么优化？

> "1000 万条直接导 Excel 不现实（文件太大，Excel 最大 104 万行）。方案：1. 按天拆分成多个文件（每天一个 CSV）。2. 提供筛选条件缩小范围（用户一般不需要全量数据）。3. 提供数据仓库/BI 工具直接导出（不走应用服务器）。4. 如果必须导出，用 Spark 分布式计算生成文件，应用层只负责提供下载链接。"

### 追问 3：导出文件需要保留多久？

> "根据业务需求定：普通导出文件保留 7 天（自动清理），财务类导出保留 30 天（合规要求），审计类导出保留 1 年。清理策略：定时任务每天凌晨扫描 export_task 表，删除过期记录和对应 OSS 文件。OSS 生命周期规则也可以自动清理。"

### 追问 4：多个用户同时导出同一批数据，怎么避免重复计算？

> "缓存机制：相同查询条件（MD5 哈希）的导出任务，如果有一个正在执行或已完成，直接复用结果。实现：创建任务时计算查询参数的 hash，检查是否有相同 hash 的已完成任务，有则直接返回已有文件。可以节省大量 DB 查询和导出时间。"

### 追问 5：导出过程中用户取消了怎么办？

> "1. 前端发取消请求 → 更新任务状态为 CANCELLED。2. 导出消费者轮询检查任务状态（每 1000 条检查一次），发现 CANCELLED 就中断 Cursor 关闭资源。3. 清理临时文件。4. 注意：取消操作不是事务性的，可能存在竞态（正在写最后一行时取消），需要幂等处理。"

## 排查 Checklist

```
□ Cursor 查询有 fetchSize = Integer.MIN_VALUE 吗？ → 没有就是全量加载
□ Cursor 有 try-with-resources 关闭吗？ → 没关闭就泄漏连接
□ CSV 文件有 BOM 头吗？ → 没有 Excel 打开就乱码
□ 导出并发有控制吗？ → 最多同时 5 个导出任务
□ 大文件上传有分片吗？ → 超过 100MB 必须分片上传
□ 导出下载走 OSS 吗？ → 不走应用服务器带宽
□ 敏感数据有脱敏吗？ → 手机号/身份证/银行卡
□ 导出 SQL 有索引吗？ → 日期 + 状态复合索引
□ 用户有导出权限吗？ → 越权导出 = 数据泄露
□ 过期文件有清理吗？ → OSS 生命周期 + 定时任务
```
