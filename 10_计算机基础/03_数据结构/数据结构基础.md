# 数据结构基础

> 数据结构是程序的骨架，算法是程序的灵魂。选择合适的数据结构，往往比选择高效的算法更能决定程序性能。

## 二、时间复杂度速查

### 渐进符号

| 符号 | 含义 | 数学定义 |
|------|------|---------|
| O (Big O) | 渐进上界（最坏情况） | ∃c,n₀, f(n) ≤ c·g(n), ∀n ≥ n₀ |
| Ω (Big Omega) | 渐进下界（最好情况） | ∃c,n₀, f(n) ≥ c·g(n), ∀n ≥ n₀ |
| Θ (Big Theta) | 渐近紧界（平均情况） | O ∩ Ω |
| o (Little o) | 严格上界 | lim f(n)/g(n) = 0 |
| ω (Little omega) | 严格下界 | lim f(n)/g(n) = ∞ |

### 常见复杂度排序

```
O(1) < O(log n) < O(√n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
常数     对数       线性      线性对数   平方       立方       指数       阶乘
```

### 主定理（Master Theorem）

用于分析分治算法的时间复杂度：T(n) = aT(n/b) + O(nᵈ)

| 条件 | 时间复杂度 | 典型例子 |
|------|-----------|---------|
| d < log_b(a) | O(n^log_b(a)) | 归并排序（d=1, a=2, b=2 → O(n log n)） |
| d = log_b(a) | O(nᵈ log n) | 二分搜索的变体 |
| d > log_b(a) | O(nᵈ) | 二叉树遍历 |

## 三、各数据结构核心特征对比

| 数据结构 | 访问 | 查找 | 插入 | 删除 | 空间 | 特点 |
|---------|------|------|------|------|------|------|
| 数组 | O(1) | O(n) | O(n) | O(n) | O(n) | 连续内存，缓存友好 |
| 链表 | O(n) | O(n) | O(1)* | O(1)* | O(n) | 动态大小，碎片化 |
| 栈 | O(n) | O(n) | O(1) | O(1) | O(n) | LIFO，后进先出 |
| 队列 | O(n) | O(n) | O(1) | O(1) | O(n) | FIFO，先进先出 |
| 哈希表 | — | O(1)* | O(1)* | O(1)* | O(n) | 最快的查找结构 |
| 二叉搜索树 | O(log n)* | O(log n)* | O(log n)* | O(log n)* | O(n) | 有序，支持范围查询 |
| AVL 树 | O(log n) | O(log n) | O(log n) | O(log n) | O(n) | 严格平衡 |
| 红黑树 | O(log n) | O(log n) | O(log n) | O(log n) | O(n) | 近似平衡，插入更快 |
| B+ 树 | O(log n) | O(log n) | O(log n) | O(log n) | O(n) | 磁盘友好，数据库索引 |
| 堆 | O(1)* | O(n) | O(log n) | O(log n) | O(n) | 快速取极值 |
| 图 | — | V+E | — | — | V+E | 拓扑排序、最短路径 |

> *带星号表示平均情况，最坏情况可能退化

## 四、Java 集合框架中的数据结构映射

```
Java Collections Framework
│
├── List（线性序列）
│   ├── ArrayList    → 动态数组
│   ├── LinkedList   → 双向链表（同时实现了 Deque）
│   └── Vector       → 线程安全的动态数组（已过时）
│
├── Set（不重复集合）
│   ├── HashSet      → HashMap（哈希表）
│   ├── LinkedHashSet → HashMap + 双向链表（保持插入顺序）
│   └── TreeSet      → 红黑树（有序）
│
├── Map（键值对）
│   ├── HashMap      → 数组 + 链表/红黑树（JDK 8+）
│   ├── LinkedHashMap → HashMap + 双向链表
│   ├── TreeMap      → 红黑树（按键排序）
│   ├── ConcurrentHashMap → 分段锁/ CAS + synchronized
│   └── Hashtable    → 全表锁（已过时）
│
├── Queue（队列）
│   ├── PriorityQueue → 二叉小顶堆
│   ├── ArrayDeque    → 循环数组
│   └── LinkedList    → 链表实现
│
└── 工具类
    ├── Arrays       → 数组操作（sort, binarySearch, asList）
    └── Collections  → 集合操作（sort, binarySearch, unmodifiableList）
```

## 五、如何选择数据结构

### 决策树

```
需要查找特定元素？
├── 需要键值映射 → HashMap
├── 需要保持有序 → TreeMap
├── 只需要去重 → HashSet
└── 需要范围查询 → TreeMap / B+ 树

需要按顺序遍历？
├── 频繁随机访问 → ArrayList
├── 频繁头尾操作 → LinkedList / ArrayDeque
└── 需要排序遍历 → TreeSet

需要频繁取最大/最小值？
└── 优先队列 → PriorityQueue / 堆

需要处理层级关系？
├── 一对多父子关系 → 树（Tree）
├── 前缀匹配 → Trie
└── 多对多关系 → 图（Graph）

需要先进先出？
├── 单端 → Queue（LinkedList / ArrayDeque）
└── 双端 → Deque（ArrayDeque）

需要后进先出？
└── Stack / Deque（ArrayDeque）
```

### 实际场景示例

| 场景 | 数据结构选择 | 理由 |
|------|-------------|------|
| LRU 缓存 | HashMap + 双向链表 | O(1) 查找 + O(1) 淘汰 |
| 表达式求值 | 栈（操作数栈 + 运算符栈） | 后缀表达式天然适配 |
| 任务调度 | 优先队列（堆） | 按优先级取任务 |
| 社交网络好友推荐 | 图（BFS） | 多跳关系遍历 |
| 自动补全 | Trie（前缀树） | 前缀匹配 |
| 数据库索引 | B+ 树 | 磁盘 IO 最小化 |
| 消息队列 | 链表/环形数组 | FIFO + 动态扩展 |
