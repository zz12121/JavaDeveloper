## 参考要点（盲答后再看）


**枚举编译后的真相**：
```java
// 源码
enum Color { RED, GREEN, BLUE }

// 编译后等价于
final class Color extends Enum<Color> {
    public static final Color RED = new Color("RED", 0);
    public static final Color GREEN = new Color("GREEN", 1);
    public static final Color BLUE = new Color("BLUE", 2);
}
```

**线程安全单例（枚举方式）**：
```java
public enum Singleton {
    INSTANCE;
    public void doSomething() { ... }
}
// 使用：Singleton.INSTANCE.doSomething();
```
- 类加载阶段初始化枚举实例 → **天生线程安全**（JVM 保证）
- 反射无法破坏：反射创建枚举实例会抛 `IllegalArgumentException`
- 序列化安全：反序列化时不会创建新实例，直接返回已有的枚举常量

**为什么比双重检查锁好**：
| 维度 | 双重检查锁 | 枚举单例 |
|------|-----------|---------|
| 线程安全 | 需要 volatile + synchronized | JVM 天然保证 |
| 序列化安全 | 需要 `readResolve()` | 天然安全 |
| 反射攻击 | 可以被破坏 | 天然防破坏 |
| 代码简洁度 | 较繁琐 | 极简 |

**枚举的局限**：
- 不能继承其他类（已继承 `Enum`）
- 不能懒加载（类加载时初始化，但 enum 常量是 `static final`，这是饿汉式）

**`ordinal()` 的坑**：
```java
enum Status { OK, ERROR }
// ordinal() 返回 0 和 1
// 但如果在 OK 和 ERROR 之间插入新值，所有 ordinal 都变了！
// 正确做法：用自己的字段
enum Status { OK(200), ERROR(500); private final int code; }
```
