# 常用网络工具

> **核心认知**：线上问题排查，网络层面是最基本的一环。掌握 netstat/ss/tcpdump/Wireshark/curl 等工具，能快速定位连接异常、丢包、延迟等问题。

---

## 1. 连接状态查看

### 1.1 netstat（经典但逐渐被 ss 替代）

```bash
# 查看所有 TCP 连接
netstat -ant

# 查看监听端口
netstat -tlnp

# 查看各状态连接数统计
netstat -ant | awk '{print $6}' | sort | uniq -c | sort -rn

# 查看 TIME_WAIT 连接数
netstat -ant | grep TIME_WAIT | wc -l

# 查看某个进程的所有连接
netstat -antp | grep java

# 查看网络统计信息（SYN 队列溢出、连接重置等）
netstat -s | grep -E "overflowed|SYNs to LISTEN"
```

**netstat 输出字段解读**：

| 字段 | 含义 |
|------|------|
| Proto | 协议（tcp/udp） |
| Recv-Q | 接收队列（ESTABLISHED=0 正常；LISTEN=当前等待 accept 的连接数） |
| Send-Q | 发送队列 |
| Local Address | 本地地址:端口 |
| Foreign Address | 远程地址:端口 |
| State | 连接状态 |

### 1.2 ss（Socket Statistics，推荐）

ss 比 netstat 更快（直接读取内核数据，不需要解析 /proc），功能更强。

```bash
# 查看所有 TCP 连接（等同于 netstat -ant）
ss -ant

# 查看监听端口（等同于 netstat -tlnp）
ss -tlnp

# 查看 socket 统计信息
ss -s

# 查看 TCP 各状态连接数
ss -ant | awk 'NR>1 {print $1}' | sort | uniq -c | sort -rn

# 查看 TIME_WAIT 连接
ss -ant state time-wait

# 查看 ESTABLISHED 连接数
ss -ant state established

# 查看监听队列（Recv-Q 是当前等待 accept 的连接数，Send-Q 是最大队列长度）
ss -lnt

# 按进程名过滤
ss -antp | grep java

# 查看 TCP 连接的详细信息（定时器、RTT、拥塞窗口等）
ss -ti dst :8080
```

**ss vs netstat**：

| 维度 | ss | netstat |
|------|-----|---------|
| 速度 | 快（直接读内核 netlink） | 慢（解析 /proc） |
| 连接数多时 | 稳定 | 可能卡顿 |
| TCP 详情 | 支持定时器/RTT/拥塞窗口 | 不支持 |
| 趋势 | **推荐使用** | 逐步淘汰 |

## 2. 抓包工具

### 2.1 tcpdump（命令行抓包）

```bash
# 抓取 eth0 网卡上 8080 端口的所有流量
tcpdump -i eth0 port 8080

# 抓取指定 IP 的流量
tcpdump -i eth0 host 192.168.1.100

# 抓取两个 IP 之间的流量
tcpdump -i eth0 host 192.168.1.1 and host 192.168.1.2

# 抓取 SYN 包（三次握手第一步）
tcpdump -i eth0 'tcp[tcpflags] & (tcp-syn) != 0'

# 抓取 HTTP 请求（过滤 80 端口的 GET 请求）
tcpdump -i eth0 -A -s 0 'tcp port 80 and tcp[((tcp[12:1] & 0xf0) >> 2):4] = 0x47455420'

# 抓包保存到文件（后续用 Wireshark 分析）
tcpdump -i eth0 port 8080 -w /tmp/tcpdump.pcap

# 读取抓包文件
tcpdump -r /tmp/tcpdump.pcap

# 抓取前 100 个包
tcpdump -i eth0 -c 100

# 显示内容（不显示十六进制）
tcpdump -i eth0 -A port 8080

# 显示十六进制和 ASCII
tcpdump -i eth0 -XX port 8080
```

**tcpdump 输出解读**：

```
20:15:30.123456 IP 192.168.1.100.54321 > 10.0.0.1.8080: Flags [S], seq 123456, win 65535, options [mss 1460,nop,wscale 6]
│            │  源IP.端口              目标IP.端口    │        │           │
│            │                                      │        │           └── TCP 选项
│            │                                      │        └── 窗口大小
│            │                                      └── 序列号
│            └── 时间戳
└── Flags: [S]=SYN, [.]=ACK, [P]=PSH+ACK, [F]=FIN, [R]=RST
```

### 2.2 Wireshark（图形化抓包分析）

**使用场景**：tcpdump 在服务器上抓包 → 下载 .pcap 文件 → 本地用 Wireshark 分析。

**常用功能**：

| 功能 | 操作 |
|------|------|
| 过滤 IP | `ip.addr == 192.168.1.100` |
| 过滤端口 | `tcp.port == 8080` |
| 过滤 HTTP | `http` |
| 跟踪 TCP 流 | 右键 → Follow → TCP Stream |
| 过滤 DNS | `dns` |
| 过滤 SYN | `tcp.flags.syn == 1` |
| 过滤 RST | `tcp.flags.reset == 1` |
| 过滤重传 | `tcp.analysis.retransmission` |

**排查技巧**：
1. **连接建立失败**：过滤 SYN 包，看是否有 SYN+ACK 响应
2. **连接被重置**：过滤 RST 包，看是服务端还是客户端发的
3. **大量重传**：`tcp.analysis.retransmission`，可能是网络质量差或 MTU 问题
4. **慢请求**：按时间排序，找到耗时最长的包

## 3. 网络连通性测试

### 3.1 ping

```bash
# 基础连通性测试
ping www.baidu.com

# 指定次数
ping -c 4 www.baidu.com

# 指定间隔（秒）
ping -i 0.5 www.baidu.com

# 指定包大小
ping -s 1400 www.baidu.com
```

### 3.2 traceroute / tracepath

```bash
# 追踪到目标的每一跳路由
traceroute www.baidu.com

# 使用 ICMP（不指定协议时默认 UDP）
traceroute -I www.baidu.com

# Linux 下的替代工具（不需要 root 权限）
tracepath www.baidu.com

# Windows 下
tracert www.baidu.com
```

### 3.3 telnet / nc（端口连通性测试）

```bash
# 测试端口是否可达
telnet 192.168.1.100 8080

# nc (netcat) 更灵活
nc -zv 192.168.1.100 8080        # 测试端口连通性
nc -zv 192.168.1.100 8080-8090   # 测试端口范围
nc -l 8080                        # 监听 8080 端口
echo "hello" | nc 192.168.1.100 8080  # 发送数据
```

### 3.4 curl（HTTP 请求工具）

```bash
# 基础 GET 请求
curl http://localhost:8080/api/users

# 显示详细信息（请求头、响应头、SSL 信息）
curl -v http://localhost:8080/api/users

# 只显示响应头
curl -I http://localhost:8080/api/users

# POST 请求（JSON）
curl -X POST -H "Content-Type: application/json" \
  -d '{"name":"zhangsan"}' \
  http://localhost:8080/api/users

# 设置超时时间（连接超时 5s，总超时 30s）
curl --connect-timeout 5 --max-time 30 http://localhost:8080/api/users

# 跟随重定向
curl -L http://example.com

# 测量响应时间（各阶段耗时）
curl -o /dev/null -s -w "DNS: %{time_namelookup}s\nConnect: %{time_connect}s\nTTFB: %{time_starttransfer}s\nTotal: %{time_total}s\n" http://localhost:8080/api/users

# 测试 HTTPS 证书
curl -vI https://example.com 2>&1 | grep -E "expire|subject|issuer|SSL"

# 压缩测试
curl -H "Accept-Encoding: gzip" -o /dev/null -s -w "Size: %{size_download}\n" http://localhost:8080/api/users
```

## 4. 网络性能测试

### 4.1 iperf3（带宽测试）

```bash
# 服务端
iperf3 -s

# 客户端（测试 TCP 带宽）
iperf3 -c server_ip

# 测试 UDP 带宽
iperf3 -c server_ip -u -b 1G

# 指定并发连接数
iperf3 -c server_ip -P 4

# 反向测试（服务端发送，客户端接收）
iperf3 -c server_ip -R
```

### 4.2 ab / wrk（HTTP 压测）

```bash
# ab（Apache Bench）
ab -n 1000 -c 50 http://localhost:8080/api/users
# -n: 总请求数  -c: 并发数

# wrk（更强大的 HTTP 压测）
wrk -t 4 -c 100 -d 30s http://localhost:8080/api/users
# -t: 线程数  -c: 连接数  -d: 持续时间
```

## 5. 网络问题排查流程

### 5.1 排查连接超时

```
1. ping 目标 IP → 网络是否通？
   ├── 不通 → 检查路由、防火墙、安全组
   └── 通 ↓

2. telnet 目标端口 → 端口是否可达？
   ├── 不可达 → 检查服务是否启动、防火墙是否放行端口
   └── 可达 ↓

3. curl -v 测试 HTTP → 返回什么？
   ├── Connection timed out → 网络层面有问题
   ├── Connection refused → 服务未监听
   ├── 502/503 → 服务异常
   └── 正常返回 → 应用逻辑问题
```

### 5.2 排查连接被重置（RST）

```bash
# 1. 检查是否有大量 RST
netstat -s | grep "reset"

# 2. 抓包确认 RST 来源
tcpdump -i eth0 'tcp[tcpflags] & (tcp-rst) != 0' -w /tmp/rst.pcap

# 3. 常见 RST 原因：
#    - 服务端进程崩溃
#    - 防火墙/WAF 拦截
#    - 内核参数 tcp_tw_recycle（NAT 环境下导致）
#    - 连接队列溢出（全连接队列满 → 发 RST）
#    - 中间设备（如 LVS、Nginx）超时断开
```

### 5.3 排查 DNS 解析失败

```bash
# 1. dig 验证
dig www.example.com @8.8.8.8

# 2. dig +trace 看解析链路
dig www.example.com +trace

# 3. 检查 DNS 服务器配置
cat /etc/resolv.conf

# 4. 清除缓存
systemd-resolve --flush-caches    # Linux
sudo dscacheutil -flushcache      # macOS
```

### 5.4 排查网络延迟高

```bash
# 1. ping 检查 RTT
ping -c 10 www.example.com

# 2. traceroute 检查哪一跳延迟高
traceroute www.example.com

# 3. curl 测量各阶段耗时
curl -o /dev/null -s -w "DNS: %{time_namelookup}\nConnect: %{time_connect}\nTTFB: %{time_starttransfer}\nTotal: %{time_total}\n" http://www.example.com

# 4. mtr（traceroute + ping 结合，持续监测每跳延迟）
mtr www.example.com
```

## 6. 常用网络配置命令

```bash
# 查看网卡信息
ip addr show              # 或 ifconfig（已淘汰）
ip link show              # 查看二层信息

# 查看路由表
ip route show             # 或 route -n

# 查看 ARP 表
ip neigh show             # 或 arp -a

# 查看防火墙规则
iptables -L -n            # IPv4
iptables -L -n -t nat     # NAT 规则

# 查看内核网络参数
sysctl net.ipv4.tcp_*     # TCP 相关参数
sysctl net.core.somaxconn # 全连接队列大小

# 临时修改内核参数
sysctl -w net.ipv4.tcp_tw_reuse=1

# 查看网络连接的 socket 选项
ss -ti dst :8080          # 显示 TCP 信息（RTT、拥塞窗口等）
```
