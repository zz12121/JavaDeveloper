# Spring 生态 — 全景地图

> 企业级开发的核心武器。不只是「会用注解」，而是理解 IoC/AOP 的本质。

---

## 学什么

```
Spring 生态
├── 01_Spring核心
│   ├── IoC容器            — BeanFactory/ApplicationContext、Bean生命周期、依赖注入方式
│   ├── AOP                — 代理模式、切面、通知类型、实现原理（JDK动态代理 vs CGLIB）
│   ├── 事务管理           — @Transactional原理、传播行为、失效场景
│   └── Spring设计模式      — 模板方法、工厂、观察者、代理在Spring中的应用
│
├── 02_SpringBoot
│   ├── 启动原理           — SpringApplication.run流程、自动配置机制
│   ├── 配置管理           — properties/yaml、Profile、配置优先级
│   ├── Web开发            — MVC流程、参数绑定、全局异常处理、拦截器
│   ├── 数据访问           — Spring Data JPA、MyBatis集成
│   └── 高级特性           — Actuator、自定义Starter、事件机制
│
├── 03_SpringCloud
│   ├── 服务注册发现       — Nacos/Eureka、负载均衡
│   ├── 网关              — Spring Cloud Gateway、过滤器链
│   ├── 远程调用           — OpenFeign、熔断降级（Sentinel）
│   ├── 配置中心           — Nacos Config、动态刷新
│   └── 分布式事务         — Seata（AT/TCC/Saga模式）
│
└── 04_Spring原理深入
    ├── BeanDefinition     — 解析、注册、合并
    ├── 上下文刷新         — refresh() 12步、BeanPostProcessor
    ├── 循环依赖           — 三级缓存、构造器注入vs字段注入
    └── 自动配置源码       — @EnableAutoConfiguration、spring.factories/imports
```

## 怎么学

**顺序**：01_Spring核心（先理解IoC和AOP的本质）→ 02_SpringBoot（快速开发能力）→ 04_Spring原理深入（读源码，理解底层）→ 03_SpringCloud（分布式场景，需要07_分布式的基础）

**关键认知**：
- IoC 不是「反转了什么」，是「把对象创建和依赖管理的控制权交给容器」
- AOP 的本质是**动态代理**，理解 JDK 代理和 CGLIB 的区别才能避免坑
- @Transactional 失效是最常见的线上问题，必须理解传播行为和自调用
- SpringBoot 自动配置的原理（condition + spring.factories）是面试高频
- SpringCloud 各组件不是孤立的，理解 CAP 理论才能做正确的技术选型

## 和其他域的关系

```
04_Spring ←→ 01_Java核心   （注解+反射 → IoC/AOP底层；泛型 → 泛型注入）
04_Spring ←→ 02_并发       （@Async → 线程池；@Transactional → 并发控制）
04_Spring ←→ 03_JVM        （类加载 → Bean加载；JIT → 启动优化）
04_Spring ←→ 05_数据库      （JPA/MyBatis → ORM；事务 → 数据库事务）
04_Spring ←→ 07_分布式      （SpringCloud → 微服务；Seata → 分布式事务）
```
