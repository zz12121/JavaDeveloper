# 高并发库存扣减系统设计

> 库存扣减是秒杀/电商核心场景，核心矛盾是：**高并发读 + 精确扣减不超卖 + 高性能**。设计要点：无锁化设计、Redis 预减、数据库兜底、分桶抗热 Key。

---

## 架构设计

```
        用户请求
           │
           ▼
   ┌───────────────────┐
   │   Redis 预减库存    │  ← 第一道防线（原子操作）
   │   DECR 或 Lua 脚本  │
   └─────────┬─────────┘
             │ 预减成功
             ▼
   ┌───────────────────┐
   │   消息队列          │  ← 异步削峰
   │   （有序，防止超卖） │
   └─────────┬─────────┘
             │
             ▼
   ┌───────────────────┐
   │   DB 最终扣减        │  ← 第二道兜底
   │   UPDATE ... WHERE  │
   └───────────────────┘
```

## 场景 A：Redis 原子扣减（Lua 脚本保证多操作原子性）

```lua
-- 库存扣减 Lua 脚本（保证原子性）
-- KEYS[1]: 库存 key
-- ARGV[1]: 扣减数量
-- ARGV[2]: 用户 ID（防重复）

local stock = tonumber(redis.call('get', KEYS[1]))
if not stock or stock < tonumber(ARGV[1]) then
    return -1  -- 库存不足
end

-- 防重复（用户已购买过？）
local userKey = 'stock:user:' .. ARGV[2] .. ':' .. KEYS[1]
if redis.call('exists', userKey) == 1 then
    return -2  -- 重复购买
end

-- 扣减库存 + 记录用户
redis.call('decrby', KEYS[1], ARGV[1])
redis.call('setex', userKey, 3600, '1')
return 1  -- 成功
```

```java
// Java 调用 Lua 脚本
@Service
public class StockService {
    private final String luaScript = "..."  // 上面的 Lua 脚本
    
    public boolean deductStock(String productId, int amount, Long userId) {
        Long result = (Long) redisTemplate.execute(
            new DefaultRedisScript<>(luaScript, Long.class),
            Collections.singletonList("stock:" + productId),
            String.valueOf(amount),
            String.valueOf(userId)
        );
        return result != null && result > 0;
    }
}
```

## 场景 B：分桶抗热 Key

### 问题
```
热门商品（如 iPhone 15）—— 所有请求打同一个 Redis key
Redis 单线程 → 热点 key 成为瓶颈
```

### 分桶方案
```java
// 库存 10000 → 分成 10 个桶，每桶 1000
public class BucketStockService {
    private final int BUCKET_COUNT = 10;
    
    public boolean deduct(Long userId, String productId, int amount) {
        // 1. 随机选择一个桶尝试
        int bucketId = ThreadLocalRandom.current().nextInt(BUCKET_COUNT);
        String bucketKey = "stock:" + productId + ":bucket:" + bucketId;
        
        Long remain = redisTemplate.opsForValue().decrement(bucketKey, amount);
        
        if (remain >= 0) return true;  // 扣减成功
        
        // 2. 当前桶不够，尝试其他桶
        redisTemplate.opsForValue().increment(bucketKey, amount);  // 回补
        
        for (int i = 0; i < BUCKET_COUNT; i++) {
            String otherBucket = "stock:" + productId + ":bucket:" + i;
            remain = redisTemplate.opsForValue().decrement(otherBucket, amount);
            if (remain >= 0) return true;
            redisTemplate.opsForValue().increment(otherBucket, amount);
        }
        
        return false;  // 所有桶都不够
    }
}
```

## 场景 C：数据库兜底方案

```sql
-- 必须加 WHERE stock >= amount，不能先 SELECT 再 UPDATE
UPDATE product_stock 
SET stock = stock - #{amount},
    sold = sold + #{amount}
WHERE product_id = #{productId}
  AND stock >= #{amount};

-- MyBatis 判断影响行数
if (affectedRows == 0) {
    throw new StockNotEnoughException();
}
```

### 一致性对账

```java
// 定时任务：Redis 库存 vs 数据库实际销量
@Scheduled(cron = "0 0 2 * * ?")  // 每天凌晨 2 点
public void reconcileStock() {
    List<Product> products = productMapper.selectAll();
    for (Product p : products) {
        int redisStock = getRedisStock(p.getId());
        int dbSold = p.getSold();
        int dbStock = p.getStock();
        
        // 预期：redisStock == dbStock
        // 如果不一致，告警 + 人工介入
        if (redisStock != dbStock) {
            alert("库存不一致: productId=" + p.getId());
        }
    }
}
```
