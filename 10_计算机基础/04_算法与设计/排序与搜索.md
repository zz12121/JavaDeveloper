# 排序与搜索

> 排序和搜索是算法的基础。面试中排序算法常考思路推导，搜索算法常考边界条件。

## 一、排序算法总览

### 1.1 十大排序算法对比

| 算法 | 平均 | 最好 | 最坏 | 空间 | 稳定性 | 思想 |
|------|------|------|------|------|--------|------|
| 冒泡排序 | O(n²) | O(n) | O(n²) | O(1) | 稳定 | 交换 |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 | 选择 |
| 插入排序 | O(n²) | O(n) | O(n²) | O(1) | 稳定 | 插入 |
| 希尔排序 | O(n^1.3) | O(n log n) | O(n²) | O(1) | 不稳定 | 插入 |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | 稳定 | 分治 |
| 快速排序 | O(n log n) | O(n log n) | O(n²) | O(log n) | 不稳定 | 分治 |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | 不稳定 | 选择 |
| 计数排序 | O(n + k) | O(n + k) | O(n + k) | O(k) | 稳定 | 计数 |
| 桶排序 | O(n + k) | O(n + k) | O(n²) | O(n + k) | 稳定 | 计数 |
| 基数排序 | O(n × k) | O(n × k) | O(n × k) | O(n + k) | 稳定 | 计数 |

> 稳定性：排序后相等元素的相对顺序不变。例如先按姓名排序，再按年龄排序，稳定性保证同年龄的人按姓名有序。

## 二、面试重点排序算法

### 2.1 快速排序（Quick Sort）

**分治思想**：选一个基准（pivot），将数组分为小于和大于 pivot 的两部分，递归排序。

```java
public void quickSort(int[] arr, int low, int high) {
    if (low >= high) return;
    int pivotIndex = partition(arr, low, high);
    quickSort(arr, low, pivotIndex - 1);
    quickSort(arr, pivotIndex + 1, high);
}

// Lomuto 分区（以最后一个元素为 pivot）
private int partition(int[] arr, int low, int high) {
    int pivot = arr[high];
    int i = low; // i 是小于 pivot 区域的边界
    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            swap(arr, i, j);
            i++;
        }
    }
    swap(arr, i, high); // pivot 放到正确位置
    return i;
}
```

```
分区过程示例：
arr = [3, 6, 8, 10, 1, 2, 1], pivot = 1(最后)

i=0, j=0: 3 > 1  → 不动
i=0, j=1: 6 > 1  → 不动
...
i=0, j=4: 1 ≤ 1  → swap(arr,0,4) → [1,6,8,10,3,2,1], i=1
i=1, j=5: 2 > 1  → 不动
i=1, j=6: 1 ≤ 1  → swap(arr,1,6) → [1,1,8,10,3,2,6], i=2
最后: swap(arr,2,6) → [1,1,6,10,3,2,8], pivot 在 index 2

左半: [1,1]（已有序）
右半: [10,3,2,8]（递归排序）
```

**快速排序优化**：

| 优化策略 | 说明 |
|---------|------|
| 随机 pivot | 避免最坏情况（已排序数组） |
| 三数取中 | 取首、中、末的中位数作为 pivot |
| 小数组切换插入排序 | n < 10 时用插入排序更快 |
| 三路快排 | 处理大量重复元素，分为 < = > 三部分 |

```java
// 三路快排（处理大量重复元素）
private void quickSort3Way(int[] arr, int low, int high) {
    if (low >= high) return;
    int pivot = arr[low];
    int lt = low, gt = high, i = low + 1;
    // [low..lt-1] < pivot, [lt..i-1] = pivot, [i..gt] 待处理, [gt+1..high] > pivot
    while (i <= gt) {
        if (arr[i] < pivot) swap(arr, lt++, i++);
        else if (arr[i] > pivot) swap(arr, i, gt--);
        else i++;
    }
    quickSort3Way(arr, low, lt - 1);
    quickSort3Way(arr, gt + 1, high);
}
```

### 2.2 归并排序（Merge Sort）

**分治思想**：将数组不断二分，直到单个元素，然后两两合并。

```java
public void mergeSort(int[] arr, int[] temp, int left, int right) {
    if (left >= right) return;
    int mid = left + (right - left) / 2;
    mergeSort(arr, temp, left, mid);
    mergeSort(arr, temp, mid + 1, right);
    merge(arr, temp, left, mid, right);
}

private void merge(int[] arr, int[] temp, int left, int mid, int right) {
    // 拷贝到临时数组
    for (int i = left; i <= right; i++) temp[i] = arr[i];

    int i = left, j = mid + 1, k = left;
    while (i <= mid && j <= right) {
        if (temp[i] <= temp[j]) arr[k++] = temp[i++];
        else arr[k++] = temp[j++];
    }
    while (i <= mid) arr[k++] = temp[i++]; // 左半剩余
    // 右半不需要拷贝，已在正确位置
}
```

**归并排序特点**：
- 稳定排序
- 时间稳定 O(n log n)，不依赖数据分布
- 空间 O(n)（需要临时数组）
- 适合链表排序、外部排序（大文件排序）

### 2.3 插入排序（Insertion Sort）

```java
public void insertionSort(int[] arr) {
    for (int i = 1; i < arr.length; i++) {
        int key = arr[i];
        int j = i - 1;
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j]; // 元素后移
            j--;
        }
        arr[j + 1] = key; // 插入正确位置
    }
}
```

> **为什么小数据量时插入排序更快？** 没有递归开销，常数因子小，对几乎有序的数据接近 O(n)。Java 的 `Arrays.sort()` 对小数组（< 47 个元素）使用插入排序。

## 三、Java 中的排序

### 3.1 Arrays.sort()

```java
// 基本类型：双轴快速排序（Dual-Pivot Quicksort）
Arrays.sort(int[] arr);  // O(n log n) 平均

// 对象类型：TimSort（归并排序 + 插入排序）
Arrays.sort(Integer[] arr);  // O(n log n)，稳定

// 自定义排序
Arrays.sort(arr, (a, b) -> b - a); // 降序
```

**TimSort 特点**：
- 找到数组中已有的有序子序列（run），利用已有顺序
- 对短 run 使用二分插入排序扩展到最小长度（minRun = 32~64）
- 对长 run 使用归并排序合并
- 最好情况 O(n)（已排序），最坏 O(n log n)

### 3.2 Collections.sort()

```java
// 底层调用 list.sort()，默认使用 TimSort
Collections.sort(list);
Collections.sort(list, Comparator.reverseOrder());
```

## 四、搜索算法

### 4.1 二分搜索（Binary Search）

```java
// 标准模板（找精确值）
int binarySearch(int[] arr, int target) {
    int left = 0, right = arr.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2; // 防止溢出
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1; // 未找到
}
```

### 4.2 二分搜索变体

```java
// 1. 查找第一个 >= target 的位置（lower_bound）
int lowerBound(int[] arr, int target) {
    int left = 0, right = arr.length;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] < target) left = mid + 1;
        else right = mid;
    }
    return left;
}

// 2. 查找第一个 > target 的位置（upper_bound）
int upperBound(int[] arr, int target) {
    int left = 0, right = arr.length;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] <= target) left = mid + 1;
        else right = mid;
    }
    return left;
}

// 3. 搜索旋转排序数组
int searchRotated(int[] arr, int target) {
    int left = 0, right = arr.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid] == target) return mid;
        // 判断哪半边有序
        if (arr[left] <= arr[mid]) { // 左半有序
            if (arr[left] <= target && target < arr[mid]) right = mid - 1;
            else left = mid + 1;
        } else { // 右半有序
            if (arr[mid] < target && target <= arr[right]) left = mid + 1;
            else right = mid - 1;
        }
    }
    return -1;
}
```

### 4.3 二分搜索做题框架

```
1. 确定搜索范围：left=0, right=length 或 right=length-1
2. 循环条件：while(left < right) 还是 while(left <= right)？
   - 找精确值：left <= right（最后 left > right 才退出）
   - 找边界/插入位置：left < right（最后 left == right）
3. mid 计算：mid = left + (right - left) / 2
4. 收缩范围：
   - 缩小下界：left = mid + 1
   - 缩小上界：right = mid - 1 或 right = mid（看循环条件）
5. 检查边界条件
```

## 六、排序算法选择指南

```
数据量小（< 50）          → 插入排序
数据基本有序              → 插入排序 / TimSort
需要稳定排序              → 归并排序 / TimSort
内存有限                  → 堆排序
平均最快                  → 快速排序
数据范围小（如 0-100）    → 计数排序
大数据量外部排序          → 外部归并排序
Java 默认                 → 基本类型快排，对象类型 TimSort
```
