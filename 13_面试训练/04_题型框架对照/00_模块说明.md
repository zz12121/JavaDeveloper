# 13_面试训练 — 题型框架对照模块

> 用「出题者思维」反向训练——知道每个知识点会怎么考，用什么框架回答。

---

## 模块概览

本模块按面试题型重新组织知识库，把「知识点」和「题型框架」对应起来，
让你知道：学了这个知识点之后，面试会怎么问、用什么框架回答。

## 面试题型与框架速查

| 题型 | 框架 | 核心问题 |
|------|------|---------|
| 原理/概念题 | 四维法 | 是什么？解决什么问题？原理？优缺点？ |
| 系统设计题 | 四步设计法 | 需求 → 顶层设计 → 核心模块 → 权衡取舍 |
| 场景题 | 五步法 | 澄清需求 → 技术选型 → 数据设计 → 接口设计 → 异常处理 |
| 项目经历题 | STAR法则 | 背景 → 任务 → 行动 → 结果 |
| 问题排查题 | 四步排查法 | 确认现象 → 缩小范围 → 定位根因 → 解决+预防 |
| 编程/算法题 | 三步法 | 确认题意 → 思路+代码 → 复杂度分析 |
| 行为题 | PARE法则 | 问题 → 行动 → 结果 → 证据 |

## 使用方法

### 第一步：知道要考什么

```
面试题型分为6类：
1. 原理/概念题（P0）- Java核心、JVM、数据库、中间件
2. 系统设计题（P1）- 高并发、分布式、微服务
3. 场景题（P0）- 项目经验、业务设计
4. 项目经历题（P0）- 你的真实项目
5. 问题排查题（P1）- JVM、MySQL、Redis、高并发
6. 编程/算法题（P1）- 算法与数据结构
7. 行为题（P2）- 软技能

你目标30-40万年薪：
- P0是必拿的分数（原理+项目）
- P1是拉开差距的分数（系统设计+排查+算法）
- P2是锦上添花
```

### 第二步：按框架准备

```
每个题型都有标准框架：
1. 先把框架背下来
2. 用框架练习对应的知识点
3. 练习的时候：先说/写，再对照答案
4. 不需要一字不差，理解框架本质即可
```

### 第三步：对着知识库练

```
用「01_万变不离其宗」里的知识点清单：
1. 每天选3个知识点
2. 用对应的框架回答
3. 对照知识库检查遗漏
4. 录下来回放，检查表达

坚持21天，面试不再慌。
```

## 学习路径建议

```
第1周：原理题四维法
- 背框架
- 用框架练习10道高频原理题
- 重点：并发编程、JVM、MySQL索引

第2周：项目经历STAR + 场景题五步法
- 准备2-3个项目经历
- 练习5道高频场景题
- 重点：高并发优化、缓存设计、分库分表

第3周：问题排查四步法 + 系统设计四步法
- 练习5道高频排查题
- 练习3道系统设计题
- 重点：CPU/内存/接口超时/MySQL慢查询

第4周：算法三步法 + 行为题PARE
- 刷高频算法题（数组、链表、二叉树、动态规划）
- 准备6道高频行为题
- 模拟面试
```

## 核心原则

```
1. 先框架，后内容
   框架是骨架，内容是血肉。
   先把框架练熟，再填充细节。

2. 先说，后写
   面试是口头的，先练习"能说出来"。

3. 先精准，后丰富
   先把一个知识点讲清楚，再追求广度。

4. 每天15分钟
   不追求一次学很多，追求持续不断。
```
