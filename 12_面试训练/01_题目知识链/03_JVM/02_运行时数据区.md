## 知识链提示

```
运行时数据区
  → [[运行时数据区]]
    → 线程私有
      → 程序计数器：当前线程字节码行号，OOM 不会（固定大小）
      → 虚拟机栈：方法栈帧，OOM 场景：递归调用无出口 → StackOverflowError
      → 本地方法栈：Native 方法，OOM 场景同上
    → 线程共享
      → 堆（最大）：对象实例 + 数组
        → 分代：Eden / Survivor×2 / Old Gen
        → OOM：不断创建对象不回收 / 内存泄漏
      → 元空间（方法区）：类信息 + 常量 + 静态变量（JDK 8+）
        → OOM：不断动态生成类（大量框架/CGLib/反射）
        → [[内存溢出与泄漏]] → 常见 OOM 类型
    → 直接内存（不属于 JVM 规范，但实际存在）
      → NIO 的 ByteBuffer.allocateDirect() 使用堆外内存
      → 受 -XX:MaxDirectMemorySize 控制
```

## 核心追问

| 追问 | 考点 |
|------|------|
| 虚拟机栈溢出和堆溢出，有什么区别？ | StackOverflow（方法调用太深）vs OOM（对象太多） |
| 为什么 JDK 8 把 PermGen 换成 Metaspace？ | 永久代固定大小容易 OOM，元空间用本地内存动态扩展 |
| 元空间 OOM 和堆 OOM，排查思路有什么不同？ | 堆 OOM 看对象，Metaspace OOM 看类加载 |
| 逃逸分析是什么？它有什么用？ | 标量替换 / 栈上分配 → 减少堆分配 |

---

## 参考答案要点

**OOM 场景速查**：

| 区域 | 常见 OOM 场景 | 错误类型 |
|------|-------------|---------|
| 堆 | 大对象 / 内存泄漏 | OutOfMemoryError: Java heap space |
| 虚拟机栈 | 递归无出口 | StackOverflowError |
| 元空间 | 大量动态类加载 | OutOfMemoryError: Metaspace |
| 直接内存 | NIO 过度使用 | OutOfMemoryError: Direct buffer memory |
