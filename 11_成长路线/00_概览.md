# 成长路线 — 概览

> 技术成长不只是学知识点，还有思维方式和影响力的构建。

---

## 四个阶段

```
中级（能干活）        高级（能排雷）         架构师（能设计）        顶尖（能引领）
     │                    │                    │                    │
  API会用              知道为什么             能做trade-off         能定义方向
  框架 CRUD            源码能读               架构能画              方案能讲
  Bug能修              疑难能查               技术能选              团队能带
```

## 核心理念

**技术深度**不是"知道得多"，是"理解得深"。一个HashMap的源码理解深度，比背100个API更有价值。

**架构能力**不是"画大图"，是"在约束条件下做正确的trade-off"。没有银弹，每个决策都有代价。

**影响力**是区分高级和顶尖的关键。技术好的人很多，能影响团队和社区的人很少。
