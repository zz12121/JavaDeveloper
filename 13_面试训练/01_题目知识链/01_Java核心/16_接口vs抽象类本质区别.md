## 参考要点（盲答后再看）


**语法层面对比**：

| 维度 | 抽象类 | 接口 |
|------|--------|------|
| 构造方法 | ✅ 有 | ❌ 没有 |
| 成员变量 | 任意 | `public static final`（常量） |
| 方法 | 抽象+具体 | Java8+ 可以有 default/static |
| 多继承 | ❌ 不支持 | ✅ 支持多实现 |
| 访问修饰符 | 任意 | 方法默认 public |

**设计层面（本质区别）**：
- 抽象类：**"是不是"** 的关系（is-a），代码复用
- 接口：**"能不能"** 的关系（can-do），能力契约

**default 方法冲突规则**（重要）：
```java
interface A { default void foo() { print("A"); } }
interface B { default void foo() { print("B"); } }
class C implements A, B {
    // 必须重写 foo()，否则编译错误
    @Override
    public void foo() { A.super.foo(); }  // 显式选择
}
```

**什么场景用哪个**：
- 用接口：定义能力契约、需要多继承、面向接口编程解耦
- 用抽象类：共享代码（模板方法模式）、控制子类构造过程、非 public 方法

**Java 版本演进**：
- Java8：加入 default（为了 Lambda 和 Stream 向后兼容）
- Java9：加入 private 方法（让 default 方法可以复用代码）
- Java 至今不允许多继承类 → 菱形继承问题（D 不知道继承 B 还是 C 的实现）
