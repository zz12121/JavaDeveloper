## 参考要点（盲答后再看）

<details>
<summary>点击展开</summary>

**擦除的本质**：编译期做类型检查，编译后 `List<String>` → `List`（raw type），类型参数被替换为边界（`extends` 的用上界，没有的用 `Object`）。

**局限**：
- 不能实例化泛型类型：`new T()` ❌
- 不能创建泛型数组：`new T[10]` ❌
- 不能对泛型做 `instanceof`：`obj instanceof T` ❌
- 静态方法/字段不能用泛型类型参数

**绕过方式**：
- 传递 `Class<T>` 对象，用反射 `clazz.newInstance()`
- 通配符 `? extends` / `? super`（PECS 原则）

**桥方法**（重点）：
```java
// 源码
interface Comparable<T> {
    int compareTo(T other);
}

// 擦除后
interface Comparable {
    int compareTo(Object other);  // 桥方法，编译器自动生成
}
```
