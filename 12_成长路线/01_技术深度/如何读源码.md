# 如何读源码

> 读源码不是为了背诵，而是理解"为什么这样设计"。带着问题去读，比通读效率高 10 倍。

---

## 一、读源码的四个层次

```
Level 1：能跑起来（知道怎么用）
→ 看示例、跑 Demo、看单元测试

Level 2：能看懂主流程（知道大概怎么实现的）
→ 看核心方法的调用链，画时序图

Level 3：能理解设计思想（知道为什么这样设计）
→ 看接口抽象、设计模式、扩展点

Level 4：能改进/扩展（能改、能加功能）
→ 看懂扩展机制，能自己写扩展
```

---

## 二、读源码的正确姿势

### 2.1 准备工作

```
1. 先把"怎么用"搞明白（跑通 Demo，写几个单元测试）
2. 弄清楚模块划分（看包结构、看架构图）
3. 准备好 Debug 环境（源码要能断点调试）
4. 准备一个问题清单（带着问题去读）
```

### 2.2 阅读顺序（以 HashMap 为例）

```
错误顺序：从第一行开始往下读 ❌
正确顺序：从使用入口开始，跟着调用链读 ✅

阅读顺序：
1. 构造函数（理解初始化）
   → HashMap() / HashMap(int initialCapacity)
2. put() 方法（最核心的写入流程）
   → 跟着 Debug 进 hash() → putVal()
3. get() 方法（理解读取流程）
4. resize() 方法（理解扩容）
5. treeifyBin() 方法（理解树化）
6. 回头看整体设计（为什么用 2 的幂次方容量？）

每一层都问：这一段解决了什么问题？
```

### 2.3 借助单元测试读源码（⭐ 强烈推荐）

```
为什么读测试比读实现更有效？
→ 测试用例展示了"怎么用"和"边界情况"

方法：
1. 找到该类的测试类（HashMap → HashMapTest）
2. 看测试用例覆盖了哪些场景
3. 跟着测试用例 Debug，理解每个分支
4. 看测试的名命名：testPutNullKey / testResize 等

示例（读 Spring 的 AbstractApplicationContext）：
→ 先看 AbstractApplicationContextTests
→ 每个测试方法对应一个功能点
```

## 三、各类型源码的阅读策略

### 3.1 框架类（Spring Boot）

```
切入点：@SpringBootApplication → run()

调用链（简化）：
SpringApplication.run()
  → prepareEnvironment()      // 准备环境
  → createApplicationContext()  // 创建 Context
  → prepareContext()          // 准备 Context
  → refreshContext()          // 核心！刷新 Context
    → invokeBeanFactoryPostProcessors()  // 处理 @Configuration
    → registerBeanPostProcessors()        // 注册 BeanPostProcessor
    → initMessageSource()
    → onRefresh()                        // 创建 Web 容器
    → registerListeners()
    → finishBeanFactoryInitialization()   // 初始化所有单例 Bean
    → finishRefresh()

重点类：
- SpringApplication（启动入口）
- AbstractApplicationContext（容器刷新核心）
- DefaultListableBeanFactory（Bean 工厂）
- AutowiredAnnotationBeanPostProcessor（@Autowired 处理）
```

### 3.2 中间件类（Redis/Lettuce 客户端）

```
Redis 客户端源码阅读顺序：
1. 从 RedisTemplate 入手（Spring 封装）
2. 看 LettuceConnection（实际执行命令）
3. 看 RedisCommands 接口（命令定义）
4. 看 RedisCommandBuilder（命令构建）
5. 看 StatefulRedisConnection（连接管理）
6. 看 EpollProvider / NioProvider（网络层）

关键问题：
- 连接怎么管理的？（连接池 vs 单连接）
- 命令怎么序列化的？（RESP 协议）
- 异步是怎么实现的？（CompletableFuture）
```

### 3.3 并发类（J.U.C）

```
阅读顺序（以 ThreadPoolExecutor 为例）：
1. 构造函数（理解 7 个参数）
2. execute() 方法（核心执行流程）
3. addWorker() 方法（创建工作线程）
4. Worker 类（工作线程的实现，继承了 AQS）
5. getTask() 方法（从队列取任务）
6. processWorkerExit() 方法（线程退出处理）

AQS（AbstractQueuedSynchronizer）阅读顺序：
1. 看 acquire() / release()（独占模式模板方法）
2. 看 acquireShared() / releaseShared()（共享模式）
3. 看 ReentrantLock 如何利用 AQS（Sync 内部类）
4. 看 Semaphore / CountDownLatch 的实现
```

## 四、读源码的工具和技巧

### 4.1 IDEA 快捷键（读源码神器）

| 快捷键 | 功能 | 使用场景 |
|--------|------|---------|
| Ctrl + B / Cmd + B | 跳转到定义 | 跟进方法调用 |
| Ctrl + Alt + B / Cmd + Alt + B | 跳转到实现 | 接口找实现类 |
| Ctrl + H / Cmd + H | 查看类继承树 | 理解抽象层次 |
| Ctrl + Alt + U / - | 查看类/方法调用层级 | 理解调用关系 |
| Ctrl + E / Cmd + E | 最近打开的文件 | 快速切换 |
| Alt + F7 / Option + F7 | 查找使用处 | 找谁调用了这个方法 |
| Ctrl + Shift + T / Cmd + Shift + T | 跳转到测试 | 找对应测试类 |

### 4.2 画图工具

```
时序图（Sequence Diagram）：
→ 用 IDEA 插件（SequenceDiagram）自动生成
→ 或手画（推荐，印象更深）

类图（Class Diagram）：
→ IDEA 右键类 → Diagrams → Show Diagrams

流程图：
→ 手画（笔 + 纸最快）
→ 或 ProcessOn / draw.io
```

### 4.3 做笔记的方法

```java
// 在源码里写注释（最推荐！）
// 读完一段源码，把自己的理解写成注释，提交到自己的「源码阅读分支」

public V put(K key, V value) {
    // 【张老师注释】put 的核心流程：
    // 1. 计算 hash（扰动函数，高 16 位参与运算）
    // 2. 定位桶（hash & (n-1)）
    // 3. 桶为空 → 直接放
    // 4. 桶非空 → 遍历链表 / 红黑树
    // 5. 容量不够 → resize()
    return putVal(hash(key), key, value, false, true);
}
```

## 五、常见开源项目的阅读路线图

### 5.1 Spring Boot

```
阶段 1（入门）：
- @SpringBootApplication 注解分解
- SpringApplication.run() 主流程
- 自动配置原理（@EnableAutoConfiguration）

阶段 2（核心容器）：
- Bean 的生命周期
- @Autowired 注入原理
- AOP 代理创建过程

阶段 3（Web 层）：
- DispatcherServlet 请求处理流程
- HandlerMapping / HandlerAdapter
- 过滤器链（Filter / Interceptor）

源码位置：
- spring-boot-project/spring-boot/src/main/java/org/springframework/boot/
- spring-framework/spring-context/src/main/java/org/springframework/context/
```

### 5.2 MyBatis

```
阶段 1：
- SqlSessionFactoryBuilder → build() → SqlSessionFactory
- SqlSession 的 getMapper() 怎么返回代理对象？

阶段 2：
- MapperProxy（JDK 动态代理）
- MappedStatement（SQL 解析结果）
- Executor（执行器：Simple / Reuse / Batch）

阶段 3：
- 插件机制（Interceptor 链）
- 一级缓存 / 二级缓存

源码位置：
- mybatis/src/main/java/org/apache/ibatis/
```

### 5.3 Netty

```
阶段 1：
- EventLoopGroup 是什么？
- Channel 和 ChannelPipeline 的关系？

阶段 2：
- 服务端启动流程（ServerBootstrap）
- 客户端连接接入（NioMessageUnsafe）
- 数据读取（NioByteUnsafe）

阶段 3：
- ByteBuf 内存管理（PooledByteBufAllocator）
- 零拷贝（CompositeByteBuf / FileRegion）

源码位置：
- netty-transport/src/main/java/io/netty/channel/
- netty-buffer/src/main/java/io/netty/buffer/
```

## 六、读源码的误区

| 误区 | 正确做法 |
|------|---------|
| 试图一次性全部读懂 | 先懂主流程，分支细节用到再查 |
| 遇到不懂的死磕 | 先标记，继续往下，回来再理解 |
| 只过没有输出 | 画时序图 + 写注释 + 写博客 |
| 从 main() 开始一步步跟 | 从核心方法（如 put/get）开始 |
| 不跑测试直接看实现 | 先看测试，理解使用场景 |

## 七、检验"读懂了"的标准

```
初级（知道流程）：
□ 能画出核心方法的调用时序图
□ 能回答"这个方法大概做了什么"

中级（理解设计）：
□ 能讲清楚"为什么这样设计"（trade-off）
□ 能回答面试中关于源码的问题

高级（能改能扩）：
□ 能自己写一个简化版
□ 能给开源项目提 PR
□ 能在项目中借鉴其设计思想
```
