### 一、入口：`@SpringBootApplication` 做了什么？

```java
// @SpringBootApplication 是一个组合注解
@SpringBootConfiguration    // 等同于 @Configuration，标记为配置类
@EnableAutoConfiguration    // 核心：启用自动装配
@ComponentScan              // 扫描当前包及子包下的 @Component
public @interface SpringBootApplication {
    // exclude / excludeName 可以排除特定的自动装配类
    Class<?>[] exclude() default {};
    String[] excludeName() default {};
}
```

**一句话**：`@SpringBootApplication` = 配置类 + 自动装配 + 组件扫描。

### 二、自动装配的核心流程

```
 SpringApplication.run()
        │
        ▼
  1. new SpringApplication()
     → 从 spring.factories 加载 ApplicationContextInitializer（上下文初始化器）
     → 从 spring.factories 加载 ApplicationListener（监听器）
        │
        ▼
  2. run()
     → prepareContext()
        → 调用所有的 ApplicationContextInitializer
        → 发布 ApplicationContextInitializedEvent
        → 加载 Bean Definitions（包括 @ComponentScan 扫到的）
        │
        ▼
  3. refreshContext()  ← 关键入口
     → AbstractApplicationContext.refresh()
     → invokeBeanFactoryPostProcessors()
        │
        ▼
  4. ConfigurationClassPostProcessor（BeanFactory 后置处理器）
     → 解析所有 @Configuration 类
     → 处理 @Import 注解
        │
        ▼
  5. @EnableAutoConfiguration 内部的 @Import(AutoConfigurationImportSelector.class)
     → AutoConfigurationImportSelector 被触发
        │
        ▼
  6. AutoConfigurationImportSelector.selectImports()
     → SpringFactoriesLoader.loadFactoryNames()
        → 从 META-INF/spring.factories（Spring Boot 2.x）
        → 或 META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports（Spring Boot 3.x）
        → 读取所有自动装配类的全限定名（默认 ~130 个）
        │
        ▼
  7. 过滤（核心！不是全部加载）
     → 根据 @Conditional 条件注解逐一判断
     → 只加载满足条件的装配类
        │
        ▼
  8. 满足条件的装配类被注册为 Bean Definition
     → 完成 Bean 实例化
```

### 三、条件装配注解（@Conditional 家族）

自动装配的精髓在于 **"按条件加载"**，不是一股脑全加载。

| 条件注解 | 作用 | 典型场景 |
|---------|------|---------|
| `@ConditionalOnClass` | classpath 中存在某个类才生效 | 引入了 MySQL 驱动才加载 DataSource 自动装配 |
| `@ConditionalOnMissingClass` | classpath 中不存在某个类才生效 | 排除某些依赖 |
| `@ConditionalOnBean` | 容器中已有某个 Bean 才生效 | 用户已自定义了 DataSource，就不装配默认的 |
| `@ConditionalOnMissingBean` | 容器中没有某个 Bean 才生效（最常用） | 用户没配就给默认的，用户配了就用用户的 |
| `@ConditionalOnProperty` | 配置文件中有某个属性才生效 | `spring.datasource.url` 存在才装配 |
| `@ConditionalOnWebApplication` | 是 Web 应用才生效 | WebMvc 自动配置 |
| `@ConditionalOnNotWebApplication` | 非 Web 应用才生效 | |
| `@ConditionalOnExpression` | SpEL 表达式为 true 才生效 | 复杂条件判断 |

**最重要的原则**：`@ConditionalOnMissingBean` —— 用户优先。

```
自动装配 = 如果用户没有自定义 → 给一个合理的默认值
用户一旦自定义 → 自动装配让位

例：用户自己配了一个 DataSource Bean
  → DataSourceAutoConfiguration 中的 @Bean 方法有 @ConditionalOnMissingBean
  → 检测到容器中已有 DataSource → 自动装配的 DataSource 不创建
  → 用户的配置生效
```

### 四、自动装配类的典型结构

```java
// 以 RedisAutoConfiguration 为例（简化）
@AutoConfiguration  // Spring Boot 3.x 使用 @AutoConfiguration 替代 @Configuration
@ConditionalOnClass(RedisConnectionFactory.class)  // classpath 有 Redis 依赖才生效
@EnableConfigurationProperties(RedisProperties.class) // 绑定 spring.redis.* 配置
public class RedisAutoConfiguration {

    @Bean
    @ConditionalOnMissingBean(name = "redisTemplate")  // 用户没配就用默认的
    public RedisTemplate<String, Object> redisTemplate(
            RedisConnectionFactory redisConnectionFactory) {
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(redisConnectionFactory);
        template.setKeySerializer(new StringRedisSerializer());
        template.setValueSerializer(new GenericJackson2JsonRedisSerializer());
        return template;
    }

    @Bean
    @ConditionalOnMissingBean  // 用户没配就用默认的
    public StringRedisTemplate stringRedisTemplate(
            RedisConnectionFactory redisConnectionFactory) {
        return new StringRedisTemplate(redisConnectionFactory);
    }
}
```

### 五、自定义 Starter 的自动装配

```
创建一个自定义 Starter 的步骤：

1. 创建项目：my-spring-boot-starter
2. 编写配置属性类：
   @ConfigurationProperties(prefix = "my.starter")
   public class MyStarterProperties { ... }
3. 编写自动装配类：
   @AutoConfiguration
   @ConditionalOnClass(MyService.class)
   @EnableConfigurationProperties(MyStarterProperties.class)
   public class MyServiceAutoConfiguration {
       @Bean
       @ConditionalOnMissingBean
       public MyService myService(MyStarterProperties props) { ... }
   }
4. 注册自动装配类：
   Spring Boot 2.x → META-INF/spring.factories
     org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
       com.example.MyServiceAutoConfiguration
   Spring Boot 3.x → META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports
     com.example.MyServiceAutoConfiguration
5. 引入 Starter 即可使用（无需额外配置）
```
