# 常见的限流算法有哪些？各自的原理是什么？

> ⚠️ **先盲答**：什么是限流？为什么需要限流？

---

## 盲答引导

1. 固定窗口 / 滑动窗口 / 漏桶 / 令牌桶，四个算法分别是什么？
2. 固定窗口限流有什么缺陷？—— 临界时间窗口双倍问题
3. 令牌桶和漏桶的核心区别是什么？—— 令牌桶允许突发，漏桶不允许
4. 限流在应用层怎么做？—— Guava RateLimiter / Sentinel / Nginx

---

## 知识链提示

```
限流
  → [[限流]]
    → 固定窗口计数器
      → 用一个计数器：时间窗口内计数，超过阈值拒绝
      → 缺陷：临界时刻（如 12:00:59 和 13:00:01，各剩半量）→ 2 倍 QPS 通过
    → 滑动窗口计数器
      → 把时间窗口切成多个小窗口，滚动求和
      → 解决固定窗口的临界双倍问题
      → Redis ZSet 实现
    → 漏桶（Leaky Bucket）
      → 水（请求）→ 固定速率从桶底漏出
      → 优点：流量完全平滑
      → 缺点：无法处理突发流量（来多少漏多少）
    → 令牌桶（Token Bucket）
      → 桶以固定速率产令牌 → 请求来 → 拿令牌 → 无令牌则拒绝
      → 优点：允许突发（桶里有令牌就处理）
      → [[Guava RateLimiter]] → 基于令牌桶实现
    → 限流维度
      → QPS 限流（令牌桶/漏桶）
      → 并发数限流（Semaphore / 线程池）
      → 节点限流 vs 全局限流（分布式限流 → Redis + Lua）
```

## 核心追问

| 追问 | 考点 |
|------|------|
| 为什么 Guava RateLimiter 能允许突发流量？ | 预消费（拿多个令牌），用未来时间换当前处理 |
| 固定窗口的临界双倍问题，具体是怎么发生的？ | 窗口边界附近，请求量是平时的 2 倍 |
| 分布式环境下怎么实现限流？ | Redis + Lua 脚本（保证原子性）/ 计数器写 Redis |
| Sentinel 和 Guava RateLimiter 选哪个？ | Sentinel（生产级，熔断降级一体）/ RateLimiter（单机）|

---

## 参考答案要点

**四算法对比**：

| 算法 | 平滑度 | 突发处理 | 实现难度 |
|------|--------|---------|---------|
| 固定窗口 | 低（临界双倍） | 差 | 简单 |
| 滑动窗口 | 高 | 差 | 中 |
| 漏桶 | 最高（完全平滑） | 不允许 | 中 |
| 令牌桶 | 高 | 允许 | 中 |

**令牌桶最常用**：允许一定的突发流量（用户体验好），同时保证长期速率不超过阈值。
