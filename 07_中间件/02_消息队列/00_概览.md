# 02_消息队列 模块概览

## 本模块目标

理解消息队列的核心概念，以及 Kafka / RabbitMQ / RocketMQ 三大主流 MQ 的架构、特性与选型。

## 进度

- [x] 消息队列核心概念（为什么存在、消息模型、Topic/Partition/Consumer Group/Offset）
- [x] Kafka架构与原理（日志存储、零拷贝、副本机制、ISR）
- [x] Kafka消费语义（At most once / At least once / Exactly once、幂等方案）
- [x] Kafka高可靠与高吞吐（三层保障、四种高吞吐手段、acks权衡）
- [x] RabbitMQ架构与交换机（四种Exchange、Queue特性、路由规则）
- [x] RabbitMQ消息可靠性（Confirm机制、Return机制、持久化、仲裁队列、手动Ack）
- [x] RocketMQ架构与核心概念（NameServer、CommitLog存储、Tag二级分类、队列数）
- [x] RocketMQ消息模型与高可靠（延迟消息/时间轮、事务消息/半消息、顺序消息）

**进度：8/8 完成 ✅**

## 学习重点

- 消息模型：点对点 vs 发布订阅，Consumer Group 的本质
- Kafka：高吞吐原理（顺序写 + 零拷贝 + 批处理 + 分区并行）、副本与 ISR
- Kafka 消费语义：Offset 提交时机决定 At most / At least / Exactly once
- RabbitMQ：四种 Exchange 路由规则、Confirm 机制、仲裁队列
- RocketMQ：延迟消息（时间轮）、事务消息（半消息 + 回查）、顺序消息、CommitLog 混合存储
- 三大 MQ 选型：日志流用 Kafka，业务消息用 RocketMQ，灵活路由用 RabbitMQ
