# Java Developer 知识体系

> **定位**：不是面试速成手册，是支撑你从中级到顶尖的系统性知识库。
> 持续积累，持续重构，持续深化。

---

## 📐 知识库结构说明

本仓库按 **「技术域」×「内容类型」** 双维度组织：

### 技术域（纵向深度）

| 编号 | 域 | 说明 |
|------|-----|------|
| 01 | Java 核心 | 语言基础、OOP、集合、IO、泛型、新特性 |
| 02 | 并发编程 | JMM、锁、并发容器、线程池、虚拟线程 |
| 03 | JVM | 内存模型、GC、类加载、字节码、调优 |
| 04 | Spring 生态 | Spring、SpringBoot、SpringCloud |
| 05 | 数据库 | MySQL、SQL优化、事务、锁、分库分表 |
| 06 | 中间件 | Redis、Kafka/RabbitMQ、Elasticsearch |
| 07 | 分布式与架构 | CAP、分布式事务、微服务、高可用、系统设计 |
| 08 | 工程化 | Git、Maven、Docker、K8s、CI-CD、Linux |
| 09 | 计算机基础 | 网络、操作系统、数据结构与算法 |

### 内容类型（横向切面）

每个技术域下的目录结构统一为：

```
XX_技术域/
├── 00_概览.md              ← 这个域的全景地图（学之前先看）
├── 01_主题A/
│   └── 主题A.md            ← 深度理解文档（为什么这样设计）
├── 02_主题B/
│   └── 主题B.md
└── ...
```

### 专题串联（跨域）

```
10_实战专题/                ← 跨技术域的场景串联
11_成长路线/                ← 能力进阶、技术视野、影响力
```

## 🎯 各层目标

| 层级 | 目标 | 标志 |
|------|------|------|
| 夯实基础 | 能讲清楚「为什么」而不只是「是什么」 | 任何知识点能说出设计动机 |
| 框架与数据 | 熟练运用，知道底层原理 | 能读框架源码，能排查疑难 |
| 架构视野 | 能设计系统，做技术选型 | 能画架构图，能做 trade-off 分析 |
| 内功心法 | 计算机基础扎实 | 网络问题能抓包定位，算法题能手写 |
| 融会贯通 | 知识没有边界 | 遇到新问题能快速建立分析框架 |

## 📝 文档规范

### 主题文档（XX_主题.md）— 讲「为什么」
- 设计动机和背景
- 原理深度解析（不是API罗列）
- 与其他概念的关系
- ASCII图解
- 源码关键片段（精炼，不是全文粘贴）
- 易错点

### 专题文档（10_实战专题/）— 讲「怎么用」
- 真实场景/问题
- 分析思路（工具+方法论）
- 涉及的知识点链接
- 解决方案和trade-off
- 追问应对
