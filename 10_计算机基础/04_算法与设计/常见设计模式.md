# 常见设计模式

> 设计模式不是死记 UML 图，而是理解"什么场景下、为什么用这个模式、trade-off 是什么"。面试常问模式的应用场景，而不是代码实现。

## 二、创建型模式

### 2.1 单例模式（Singleton）

**确保一个类只有一个实例，并提供全局访问点。**

```java
// 1. 饿汉式（类加载时就创建，线程安全，但不能延迟加载）
public class HungrySingleton {
    private static final HungrySingleton INSTANCE = new HungrySingleton();
    private HungrySingleton() {}
    public static HungrySingleton getInstance() { return INSTANCE; }
}

// 2. 懒汉式 + synchronized（延迟加载，但锁粒度太大）
public class LazySingleton {
    private static LazySingleton instance;
    private LazySingleton() {}
    public static synchronized LazySingleton getInstance() {
        if (instance == null) instance = new LazySingleton();
        return instance;
    }
}

// 3. 双重检查锁（DCL）⭐ 面试重点
public class DCLSingleton {
    // volatile 禁止指令重排序（防止"分配内存→引用赋值→初始化"重排为"分配→赋值→初始化"）
    private static volatile DCLSingleton instance;
    private DCLSingleton() {}

    public static DCLSingleton getInstance() {
        if (instance == null) {                    // 第一次检查（无锁）
            synchronized (DCLSingleton.class) {
                if (instance == null) {            // 第二次检查（有锁）
                    instance = new DCLSingleton(); // 非原子操作！
                }
            }
        }
        return instance;
    }
}

// 4. 静态内部类（推荐）⭐ 最优雅
public class InnerClassSingleton {
    private InnerClassSingleton() {}
    private static class Holder {
        static final InnerClassSingleton INSTANCE = new InnerClassSingleton();
    }
    public static InnerClassSingleton getInstance() {
        return Holder.INSTANCE; // 利用类加载机制保证线程安全
    }
}

// 5. 枚举单例（Effective Java 推荐）⭐ 最安全
public enum EnumSingleton {
    INSTANCE;
    public void doSomething() { /* ... */ }
    // 天然防反射、防序列化
}
```

**单例模式破坏方式及防御**：

| 破坏方式 | 原理 | 防御 |
|---------|------|------|
| 反射 | setAccessible(true) 绕过 private | 构造器中判断 instance != null 则抛异常 |
| 序列化 | 反序列化创建新对象 | 实现 readResolve() 返回 instance |
| 克隆 | 克隆创建新对象 | 不实现 Cloneable |

**面试问题**："Spring Bean 是单例的吗？"
> Spring 默认 scope=singleton，但它是容器级单例，不是 JVM 级单例。每个 ApplicationContext 内一个实例。如果有多 个 ClassLoader，就不是单例。

### 2.2 工厂模式（Factory）

```java
// 简单工厂（不属于 GoF 23 种，但最常用）
public class ShapeFactory {
    public static Shape create(String type) {
        switch (type) {
            case "circle": return new Circle();
            case "square": return new Square();
            default: throw new IllegalArgumentException();
        }
    }
}

// 工厂方法（每个产品一个工厂）
public interface ShapeFactory {
    Shape create();
}
public class CircleFactory implements ShapeFactory {
    public Shape create() { return new Circle(); }
}

// 抽象工厂（一组相关产品）
public interface UIFactory {
    Button createButton();
    TextField createTextField();
}
public class MacUIFactory implements UIFactory {
    public Button createButton() { return new MacButton(); }
    public TextField createTextField() { return new MacTextField(); }
}
```

**Spring 中的工厂模式**：
- `BeanFactory`：IoC 容器的根接口，通过 `getBean()` 获取 Bean
- `FactoryBean`：自定义 Bean 的创建逻辑（如 `SqlSessionFactoryBean`）

### 2.3 建造者模式（Builder）

**适用场景：对象有很多参数，其中很多是可选的。**

```java
// 传统写法： telescoping constructor 反模式
User user = new User("张三", 25, "男", "北京", null, null, null, null);

// Builder 模式：链式调用，可读性强
User user = User.builder()
    .name("张三")
    .age(25)
    .gender("男")
    .city("北京")
    .build();
```

> **Lombok @Builder** 就是建造者模式的代码生成。面试常问 Builder 的实现原理。

## 三、结构型模式

### 3.1 代理模式（Proxy）

**在不修改原对象的前提下，通过代理对象控制对原对象的访问。**

```java
// 静态代理
public interface UserDao { void save(); }
public class UserDaoImpl implements UserDao {
    public void save() { System.out.println("保存用户"); }
}
public class UserDaoProxy implements UserDao {
    private UserDao target;
    public UserDaoProxy(UserDao target) { this.target = target; }
    public void save() {
        System.out.println("开启事务");  // 前置增强
        target.save();
        System.out.println("提交事务");  // 后置增强
    }
}

// JDK 动态代理（基于接口）⭐ 面试重点
UserDao proxy = (UserDao) Proxy.newProxyInstance(
    UserDao.class.getClassLoader(),
    new Class[]{ UserDao.class },
    (obj, method, args) -> {
        System.out.println("开启事务");
        Object result = method.invoke(target, args);
        System.out.println("提交事务");
        return result;
    }
);
```

**JDK 动态代理 vs CGLIB 代理**：

| 维度 | JDK 动态代理 | CGLIB |
|------|-------------|-------|
| 原理 | 基于接口，生成实现类 | 基于继承，生成子类 |
| 要求 | 目标类必须实现接口 | 目标类不能是 final |
| 性能 | JDK 8+ 已优化 | 创建慢，调用快 |
| Spring 默认 | 有接口时使用 | 无接口时使用 |

> **Spring AOP** 底层就是动态代理。`@Transactional` 注解的实现原理就是通过代理对象在方法执行前后添加事务管理逻辑。

### 3.2 适配器模式（Adapter）

**将一个类的接口转换为客户端期望的另一个接口。**

```
Spring MVC 中的适配器：
HandlerAdapter 接口（适配不同的 Controller 类型）
├── SimpleControllerHandlerAdapter → 适配 Controller 接口
├── HttpRequestHandlerAdapter    → 适配 HttpRequestHandler
└── AnnotationMethodHandlerAdapter → 适配 @RequestMapping 方法
```

### 3.3 装饰器模式（Decorator）

**动态地给对象添加职责，比继承更灵活。**

```
Java IO 中的装饰器：
InputStream
├── FileInputStream        （被装饰对象）
├── BufferedInputStream    （装饰器：添加缓冲）
├── DataInputStream        （装饰器：添加数据类型读取）
└── ObjectInputStream      （装饰器：添加对象反序列化）

用法：
InputStream in = new BufferedInputStream(new FileInputStream("file.txt"));
//      装饰器              被装饰对象
```

**装饰器 vs 代理**：

| 维度 | 装饰器 | 代理 |
|------|--------|------|
| 目的 | 增强功能 | 控制访问 |
| 关注点 | 添加新行为 | 不改变行为 |
| 关系 | 客户端知道装饰器的存在 | 客户端可能不知道代理存在 |

### 3.4 外观模式（Facade）

**为复杂子系统提供统一的高层接口。**

```
例子：Slf4j 就是日志门面（Facade）

SLF4J（门面）
├── Logback（实现）
├── Log4j2（实现）
└── JUL（实现）

好处：切换日志框架只需更换依赖，不用改业务代码
```

## 四、行为型模式

### 4.1 策略模式（Strategy）⭐ 高频

**定义一系列算法，将每个算法封装起来，使它们可以互相替换。**

```java
// 场景：支付方式选择
public interface PaymentStrategy {
    void pay(int amount);
}

public class AlipayStrategy implements PaymentStrategy {
    public void pay(int amount) { System.out.println("支付宝支付: " + amount); }
}

public class WechatStrategy implements PaymentStrategy {
    public void pay(int amount) { System.out.println("微信支付: " + amount); }
}

// 使用
PaymentStrategy strategy = new AlipayStrategy();
strategy.pay(100);
```

**消除 if-else**：
```java
// 不好：大量 if-else
if (type.equals("alipay")) { /* ... */ }
else if (type.equals("wechat")) { /* ... */ }
else if (type.equals("card")) { /* ... */ }

// 好：策略模式 + 工厂
Map<String, PaymentStrategy> strategies = Map.of(
    "alipay", new AlipayStrategy(),
    "wechat", new WechatStrategy(),
    "card", new CardStrategy()
);
strategies.get(type).pay(amount);
```

> **Spring 中的策略模式**：`Resource` 接口的多种实现（`ClassPathResource`、`FileSystemResource`、`UrlResource`）。

### 4.2 观察者模式（Observer）

**定义对象间一对多的依赖关系，当一个对象改变状态时，所有依赖它的对象都会收到通知。**

```java
// JDK 内置观察者模式
public class WeatherData extends Observable {
    public void setTemperature(float temp) {
        setChanged();
        notifyObservers(temp); // 通知所有观察者
    }
}

public class Display implements Observer {
    public void update(Observable o, Object arg) {
        System.out.println("温度更新: " + arg);
    }
}
```

**实际应用**：
- **事件监听**：`ApplicationContextEvent`、`@EventListener`
- **消息队列**：发布-订阅模式
- **Vue/React 响应式**：状态变化自动触发视图更新

### 4.3 模板方法模式（Template Method）

**定义算法的骨架，将某些步骤延迟到子类实现。**

```java
public abstract class AbstractDataParser {
    // 模板方法（final 防止子类覆盖骨架）
    public final void parse() {
        readData();
        processData();
        writeData();
    }

    protected abstract void readData();      // 子类实现
    protected abstract void processData();   // 子类实现

    private void writeData() {               // 固定步骤
        System.out.println("写入结果");
    }
}

// Spring 中的模板方法：
// JdbcTemplate.query() — 固定了获取连接、执行查询、关闭连接的流程
// RedisTemplate.execute() — 固定了获取连接、执行命令、关闭连接的流程
```

### 4.4 责任链模式（Chain of Responsibility）

**将请求沿着处理者链传递，直到某个处理者处理它。**

```java
// 场景：审批流程
public abstract class Handler {
    protected Handler next;
    public void setNext(Handler next) { this.next = next; }

    public abstract boolean handle(int amount);
}

public class ManagerHandler extends Handler {
    public boolean handle(int amount) {
        if (amount <= 1000) return true;
        return next != null && next.handle(amount);
    }
}

public class DirectorHandler extends Handler {
    public boolean handle(int amount) {
        if (amount <= 5000) return true;
        return next != null && next.handle(amount);
    }
}
```

**实际应用**：
- **Servlet Filter**：请求经过一系列 Filter 链
- **Spring Interceptor**：HandlerInterceptor 链
- **Netty Pipeline**：ChannelHandler 链
- **Logback/Log4j2**：Appender 链

### 4.5 状态模式（State）

**允许对象在内部状态改变时改变它的行为。**

```java
// 状态机示例：订单状态
public interface OrderState {
    void pay(OrderContext ctx);
    void ship(OrderContext ctx);
    void confirm(OrderContext ctx);
}

// 待支付 → 已支付 → 已发货 → 已确认
// 每个状态对相同操作有不同的响应
```

## 五、设计原则（SOLID）

| 原则 | 全称 | 一句话 |
|------|------|--------|
| S | Single Responsibility | 一个类只做一件事 |
| O | Open/Closed | 对扩展开放，对修改关闭 |
| L | Liskov Substitution | 子类可以替换父类 |
| I | Interface Segregation | 接口要小而专 |
| D | Dependency Inversion | 依赖抽象，不依赖具体 |

**其他原则**：
- **DRY（Don't Repeat Yourself）**：不重复
- **KISS（Keep It Simple, Stupid）**：保持简单
- **YAGNI（You Aren't Gonna Need It）**：不要过度设计
