# JVM — 全景地图

> JVM 知识让你从「会用 Java」变成「理解 Java」。是排查疑难杂症的底层武器。

---

## 学什么

```
JVM
├── 01_体系结构           — JVM架构、JVM vs JRE vs JDK
├── 02_运行时数据区        — 堆、方法区、虚拟机栈、本地方法栈、程序计数器
├── 03_内存模型            — 堆内存结构（新生代/老年代）、直接内存、元空间
├── 04_垃圾回收            — GC Roots、可达性分析、标记-清除/复制/整理
├── 05_垃圾收集器          — Serial/ParNew/CMS/G1/ZGC/Shenandoah
├── 06_类加载机制          — 双亲委派、加载/链接/初始化、自定义ClassLoader
├── 07_字节码              — 字节码指令、javap使用、字节码与并发的关系
├── 08_内存溢出与泄漏       — OOM排查、MAT分析、常见泄漏场景
├── 09_JVM调优            — 参数设置、GC日志、Arthas/jstat/jmap
│   └── JVM调优实战进阶   — 线上OOM排查、MAT分析、GC问题定位、CPU 100%/死锁排查
└── 10_JIT编译            — 逃逸分析、内联、分层编译
```

## 怎么学

**顺序**：01 → 02（运行时数据区是基础）→ 03（内存结构）→ 04/05（GC，重点）→ 06（类加载）→ 07（字节码，帮助理解底层）→ 08/09（实战排障）→ 10（进阶）

**关键认知**：
- 运行时数据区是**理解一切的基础**——线程安全问题、内存泄漏、GC 都跟它相关
- GC 不是背参数，是理解「什么对象该回收」「怎么回收」「回收对应用有什么影响」
- 类加载机制是理解 Spring/MyBatis/Tomcat 启动过程的基础
- 字节码看起来冷门，但它是理解 synchronized、Lambda、泛型擦除的底层窗口
- JVM 调优的本质是「先优化业务，再调 JVM」——Arthas + MAT + GC 日志分析是排查三板斧

## 和其他域的关系

```
03_JVM ←→ 01_Java核心   （类加载 → 反射；字符串 → 常量池）
03_JVM ←→ 02_并发       （JMM → 运行时数据区；线程 → 虚拟机栈）
03_JVM ←→ 04_Spring    （类加载 → Bean生命周期；JIT → Spring启动优化）
03_JVM ←→ 09_计算机基础  （内存管理 → 操作系统虚拟内存）
```
