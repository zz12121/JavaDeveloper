# 继承

> 继承解决的是"公共行为复用"问题，但它也是 Java 中**耦合度最高的关系**——父类一改，所有子类都可能悄悄出 bug。理解什么时候该用继承、什么时候该用组合，比记住 extends 语法重要一百倍。

---

## 这个问题为什么存在？

没有继承时，相似类型的公共行为需要重复写：

```java
// 没有继承：狗和猫都有「吃」「睡」的行为，代码重复
class Dog {
    String name;
    void eat() { System.out.println(name + " 在吃东西"); }
    void sleep() { System.out.println(name + " 在睡觉"); }
    void bark() { System.out.println("汪汪！"); }
}

class Cat {
    String name;
    void eat() { System.out.println(name + " 在吃东西"); }   // 完全重复！
    void sleep() { System.out.println(name + " 在睡觉"); }   // 完全重复！
    void meow() { System.out.println("喵喵~"); }
}
```

继承的目标：**提取公共行为到父类，子类只写自己特有的部分**。

---

## 它是怎么解决问题的？

### 基本用法

```java
class Animal {
    String name;
    void eat() { System.out.println(name + " 在吃东西"); }
    void sleep() { System.out.println(name + " 在睡觉"); }
}

class Dog extends Animal {
    void bark() { System.out.println("汪汪！"); }   // 只写特有的
}

class Cat extends Animal {
    void meow() { System.out.println("喵喵~"); }   // 只写特有的
}
```

### 方法覆写（Override）

```java
class Animal {
    void makeSound() {
        System.out.println("...");
    }
}

class Dog extends Animal {
    @Override  // 强烈建议加：让编译器帮你确认是覆写而非重载
    void makeSound() {
        System.out.println("汪汪！");
    }
}
```

**覆写规则**（编译期检查）：
```
返回值类型：可以是父类返回类型的子类（协变返回类型）
访问修饰符：只能放宽，不能收紧
  （子类 public 覆写父类 protected ✓，反之 ✗）
异常声明：只能抛出父类方法声明的检查型异常的子类
```

### super 关键字

```java
class Animal {
    void eat() {
        System.out.println("动物吃东西");
    }
}

class Dog extends Animal {
    @Override
    void eat() {
        super.eat();                        // 调用父类实现（扩展而非替换）
        System.out.println("狗更喜欢骨头");
    }
}
```

## 深入原理

### 继承的代价：脆弱的基类问题（Fragile Base Class）

**继承是 Java 中耦合度最高的关系**——子类和父类之间牵一发动全身。

```java
// 父类修改了方法签名，子类可能悄悄就出 bug 了
class Animal {
    // 原来是：void eat() { ... }
    boolean eat(int calories) { ... }  // 现在改成有返回值的
}

class Dog extends Animal {
    @Override
    void eat() { ... }  // 编译错误！但如果是重载，编译器不会报错
}
```

更隐蔽的问题：子类覆写的方法，父类压根不知道它的存在：

```java
class Parent {
    void process() {
        doStep1();   // 调用的是子类的覆写版本！
        doStep2();
    }

    void doStep1() { /* 父类实现 */ }
    void doStep2() { /* 父类实现 */ }
}

class Child extends Parent {
    @Override
    void doStep1() {
        // 如果这里抛异常，Parent.process() 就炸了
        // 但 Parent 的作者完全不知道 Child 覆写了 doStep1！
    }
}
```

### 继承 vs 组合

| 维度 | 继承（is-a） | 组合（has-a） |
|------|-------------|---------------|
| 耦合度 | 极强（父类实现变化直接影响子类） | 弱（只依赖接口） |
| 扩展性 | 父类改，子类全受影响 | 换实现只改一处 |
| 内存 | 单对象 | 两个对象 |
| 典型场景 | Dog is-a Animal | Car has-a Engine |

**选择原则**：
```
用继承：is-a 关系清晰，且父类的变化概率低
  （如：String → Object，Integer → Number）

用组合：只是为了复用代码，没有 is-a 关系
  （如：ArrayList 作为字段，而非 extends ArrayList）

Effective Java 第 18 条：**优先使用组合，其次才是继承。**
```

### 构造函数中的覆写陷阱

```java
class Parent {
    Parent() {
        doSomething();  // ❌ 危险！如果被子类覆写，子类字段还没初始化
    }
    void doSomething() { }
}

class Child extends Parent {
    int value = 10;

    Child() {
        super();  // 先调用 Parent()，此时 value 还是默认值 0
    }

    @Override
    void doSomething() {
        System.out.println(value);  // 打印 0，不是 10！
    }
}
```

**原因**：`super()` 在子类字段初始化之前执行。构造函数中**不要调用可覆写的方法**。模板方法模式用 `final` 保护核心步骤来解决这个问題，详见 [[01_Java核心/01_基础语法/final关键字|final 方法]]。

## 正确使用方式

### 1. 模板方法模式：继承的正确用法

```java
abstract class DataProcessor {
    // 模板方法：定义算法骨架，用 final 防止子类破坏
    public final void process() {
        readData();       // 固定步骤
        transformData();  // 抽象，子类实现
        writeData();      // 固定步骤
    }

    abstract void transformData();  // 子类必须实现

    void readData() { /* 通用实现 */ }
    void writeData() { /* 通用实现 */ }
}
```

模板方法模式是继承的**最佳使用场景**：父类控制流程，子类只填充变化的部分。

### 2. 用组合替代继承

```java
// 不好：通过继承复用 ArrayList 的行为
class CountingList<E> extends ArrayList<E> {
    int addCount = 0;

    @Override
    public boolean add(E e) {
        addCount++;
        return super.add(e);
    }

    @Override
    public boolean addAll(Collection<? extends E> c) {
        addCount += c.size();
        return super.addAll(c);   // ❌ addAll 内部会调用 add()，addCount 被双计了
    }
}

// 好：通过组合持有 ArrayList
class CountingList<E> {
    private final List<E> list = new ArrayList<>();
    private int addCount = 0;

    public boolean add(E e) {
        addCount++;
        return list.add(e);  // ✓ 精确控制，不受父类内部实现影响
    }
}
```

### 3. 覆写时永远加 @Override

```java
class Parent {
    void doWork() { }
}

class Child extends Parent {
    @Override
    void doWork() { }      // ✓ 编译器确认是覆写

    // @Override
    // void doWork(int x) { }  // 如果没有 @Override，这是重载，不是覆写，编译器不报错！
}
```

## 边界情况和坑

### 1. 构造函数链：默认 super() 插入

```java
class Parent {
    Parent() { System.out.println("parent"); }
}

class Child extends Parent {
    Child() {
        // 编译器自动插入 super(); 在最前面
        System.out.println("child");
    }
}

new Child();
// 输出：
// parent
// child
```

如果父类没有无参构造函数，子类必须**显式**调用 `super(args)`：

```java
class Parent {
    Parent(int x) { }  // 没有无参构造器
}

class Child extends Parent {
    Child() {
        super(10);  // ✓ 必须显式调用，否则编译错误
    }
}
```

### 2. 字段隐藏（不是覆写）

```java
class Animal { String name = "Animal"; }
class Dog extends Animal { String name = "Dog"; }  // 字段隐藏，不是覆写

Animal a = new Dog();
System.out.println(a.name);   // "Animal"（编译时类型决定）
System.out.println(((Dog) a).name);  // "Dog"
```

**字段不参与多态**！`Dog.name` 隐藏了 `Animal.name`，但访问时看的是引用类型，不是实际类型。这也是为什么建议字段 private——通过方法访问才能利用多态。

### 3. 父子类同名方法的返回类型协变

```java
class Parent {
    Number getValue() { return 42; }
}

class Child extends Parent {
    @Override
    Integer getValue() { return 42; }  // ✓ 协变返回类型：Integer 是 Number 的子类
}
```

这是 Java 5 引入的特性。注意：这只适用于覆写，不适用于重载。

### 4. 继承层次过深

```java
class A { }
class B extends A { }
class C extends B { }
class D extends C { }
class E extends D { }  // ← 继承链已经太深了
```

**经验法则**：继承层次超过 2~3 层，就要重新考虑设计。过深的继承链导致：
- 理解成本指数级上升
- 修改父类影响的范围不可控
- 脆弱的基类问题被放大

考虑用**策略模式**或**组合**替代深继承链。
