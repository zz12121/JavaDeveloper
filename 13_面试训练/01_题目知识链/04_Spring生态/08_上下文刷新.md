# ApplicationContext.refresh() 做了哪些事？

> ⚠️ **先盲答**：`AbstractApplicationContext.refresh()` 是 Spring 容器初始化的核心，你知道它有几步吗？

---

## 盲答引导

1. `refresh()` 的九大步骤，你能按顺序说出来吗？
2. `prepareBeanFactory()` 和 `finishBeanFactoryInitialization()` 分别做了什么？
3. `invokeBeanFactoryPostProcessors()` 做了什么？—— 执行所有 BeanFactoryPostProcessor
4. `onRefresh()` 和 `finishRefresh()` 分别在什么时候调用？

---

## 知识链提示

```
ApplicationContext.refresh()
  → ApplicationContext刷新
    → AbstractApplicationContext.refresh() 九大步骤
      → 1. prepareBeanFactory：给 BeanFactory 设置上下文参数（ClassLoader / ELResolver）
      → 2. postProcessBeanFactory：空实现（留给子类，如 Tomcat 的 ServletContextFactoryBean）
      → 3. invokeBeanFactoryPostProcessors：执行所有 BeanFactoryPostProcessor（包括 ConfigurationClassPostProcessor）
        → 处理 @Configuration 类，注册 BeanDefinition
      → 4. registerBeanPostProcessors：注册 BeanPostProcessor（注意：只是注册，还没执行）
        → 重要：AnnotationAwareAspectJAutoProxyCreator 在这里注册（AOP 代理的根基）
      → 5. initMessageSource：国际化（MessageSource）
      → 6. initApplicationEventMulticaster：事件广播器
      → 7. onRefresh：留给子类（Web 实现：创建内嵌 Tomcat/Jetty）
      → 8. registerListeners：注册 ApplicationListener
      → 9. finishBeanFactoryInitialization：实例化所有单例 Bean（懒加载的除外）
        → [[04_Spring生态/04_Spring原理深入/循环依赖解决方案]] → 这里触发
      → 10. finishRefresh：发布 ContextRefreshedEvent，LifecycleProcessor.start()
```

---

## 核心追问

| 追问 | 考点 |
|------|------|
| 为什么 registerBeanPostProcessors 只是「注册」，不「执行」？ | 执行 BeanPostProcessor 需要 Bean 已实例化，先注册后实例化 |
| `onRefresh()` 在 Web 场景下做了什么？ | 创建内嵌 Web 服务器（Tomcat/Jetty/Undertow）|
| finishBeanFactoryInitialization 为什么重要？ | [[04_Spring生态/04_Spring原理深入/循环依赖解决方案]] 在这里触发，单例 Bean 在这里真正创建 |
| 为什么 refresh() 要加 synchronized？ | 防止重复刷新容器（多线程安全）|

---

## 参考答案要点

**九大步骤简化记忆**：
```
准备工厂 → 注册后处理器 → 初始化广播器 → 创建内嵌容器
→ 注册监听器 → 初始化所有单例 Bean → 完成刷新
```

**最容易考的点**：
- 第 4 步：注册 AOP 代理的根基（AnnotationAwareAspectJAutoProxyCreator）
- 第 9 步：循环依赖在这里触发
