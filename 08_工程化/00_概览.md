## 怎么学

**顺序**：01（每天用的Git）→ 02（Maven/Gradle）→ 03（Docker，基础要扎实）→ 05（CI-CD理解流程）→ 04（K8s，按需深入）→ 06（Linux命令和脚本，随用随学）

**关键认知**：
- Git 不只是 `add/commit/push`，理解分支模型和rebase的本质才能避免灾难
- Docker 的核心价值是「环境一致性」，不是轻量级虚拟机
- K8s 解决了「容器编排」问题，但要理解它解决的问题域再深入
- CI/CD 不只是自动化部署，是「质量内建」的文化
