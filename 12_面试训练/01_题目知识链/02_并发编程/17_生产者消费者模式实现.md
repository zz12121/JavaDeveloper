## 参考要点（盲答后再看）

<details>
<summary>点击展开</summary>

**三种实现方式对比**：

| 方式 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| `wait/notify` | 无依赖，纯 JDK | 容易写错（while vs if），只支持单条件 | ⭐⭐ |
| `ReentrantLock + Condition` | 多条件队列，精细控制 | 代码量较多 | ⭐⭐⭐ |
| `BlockingQueue` | 最简单，内置支持 | 功能固定 | ⭐⭐⭐⭐⭐ |

**wait/notify 经典陷阱（必须知道）**：
```java
// ❌ 错误写法：if 判断
synchronized (queue) {
    if (queue.isEmpty()) queue.wait();  // 虚假唤醒后不重新检查！
}

// ✅ 正确写法：while 循环
synchronized (queue) {
    while (queue.isEmpty()) queue.wait();  // 虚假唤醒后重新检查
}
```
→ 虚假唤醒（spurious wakeup）：系统可能在没有 notify 的情况下唤醒线程！

**BlockingQueue 最简实现**：
```java
BlockingQueue<Integer> queue = new ArrayBlockingQueue<>(10);

// 生产者
new Thread(() -> {
    try { queue.put(produce()); } catch (InterruptedException e) {}
}).start();

// 消费者
new Thread(() -> {
    try { consume(queue.take()); } catch (InterruptedException e) {}
}).start();
```

**优雅停止**：
```java
// 生产者发「毒丸」（Poison Pill）
queue.put(POISON_PILL);
// 消费者收到 POISON_PILL 后退出
```

**线程池就是生产者-消费者**：
- 生产者：调用 `execute()` 的线程
- 队列：`workQueue`（BlockingQueue）
- 消费者：Worker 线程
