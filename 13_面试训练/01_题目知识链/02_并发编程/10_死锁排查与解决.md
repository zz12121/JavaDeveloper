## 知识链提示

```
死锁与活锁
  → [[死锁与活锁]]
    → 四个必要条件（同时满足才会死锁）
      → 互斥：资源同一时刻只能被一个线程持有
      → 占有并等待：线程持有资源的同时等待其他资源
      → 不可抢占：资源不能被强制释放，只能线程主动释放
      → 循环等待：线程A等线程B持有的资源，线程B等线程A持有的资源
    → 排查方法
      → jstack <pid> → 打印线程栈，搜索 "deadlock" 或 Blocked 状态的线程
      → jconsole / VisualVM → 可视化线程状态
      → [[线上问题排查手册]] → 死锁章节
    → 解决思路
      → 破坏任一条件即可
        → 按固定顺序获取锁（破坏循环等待）
        → 设置锁超时（Lock.tryLock(timeout)，破坏不可抢占）
        → [[ReentrantLock与显式锁]] → tryLock() 可中断获取
```

## 参考答案要点

**四个必要条件**：互斥 / 占有并等待 / 不可抢占 / 循环等待。

**最实用的解决方式**：
- 按固定顺序获取多个锁（最常用）
- `ReentrantLock.tryLock(timeout)` 替代 `lock()`（超时自动释放）
- 减少锁粒度，避免一个线程同时持有多把锁

**排查命令**：`jstack <pid>` → 找 `Found one deadlock` 或两个线程互相 Blocked。
