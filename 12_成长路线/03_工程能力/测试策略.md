# 测试策略

> 测试不是"写完功能再补"，而是"没有测试就不算完成"。好的测试策略能让你放心重构。

---

## 一、测试金字塔

```
        /\
       /  \      UI 自动化测试（少量）
      /____\      E2E（Selenium/Cypress）
     /      \    
    /  ______\   集成测试（中量）
   /  /      \   接口测试（Postman/JMeter）
  /  /  ______\ 单元测试（大量）
 /  /  /      \ 
/__/__/________\  → 越多越底层，越快越稳定
```

| 层级 | 速度 | 成本 | 稳定性 | 占比 |
|------|------|------|--------|------|
| 单元测试 | 快（ms 级） | 低 | 高 | 70% |
| 集成测试 | 中（s 级） | 中 | 中 | 20% |
| E2E 测试 | 慢（10s 级） | 高 | 低 | 10% |

## 二、单元测试

### 2.1 JUnit 5 最佳实践

```java
class OrderServiceTest {
    private OrderService service;
    @BeforeEach void setUp() { service = new OrderService(); }

    @Test
    void shouldCalculateTotalPrice() {
        // Given（准备）
        List<OrderItem> items = List.of(new OrderItem("apple", 5.0, 2));
        // When（执行）
        double total = service.calculateTotal(items);
        // Then（断言）
        assertEquals(10.0, total, 0.001);
    }

    @Test
    void shouldThrowWhenQuantityNegative() {
        List<OrderItem> items = List.of(new OrderItem("apple", 5.0, -1));
        assertThrows(IllegalArgumentException.class,
            () -> service.calculateTotal(items));
    }
}
// 命名：should_ExpectedBehavior_When_Condition
```

### 2.2 嘲弄（Mock）策略

```java
@ExtendWith(MockitoExtension.class)
class UserServiceTest {
    @Mock UserDao userDao;
    @InjectMocks UserService userService;

    @Test
    void shouldReturnUserWhenExists() {
        // 1. 打桩（Stub）
        when(userDao.findById(1L)).thenReturn(new User(1L, "张三"));
        // 2. 执行
        User result = userService.getUser(1L);
        // 3. 验证
        assertEquals("张三", result.getName());
        verify(userDao, times(1)).findById(1L); // 验证交互
    }
}
```

**Mock 原则**：
```
✅ 只 Mock 你拥有的接口（不 Mock 第三方）
✅ 验证状态（返回值）优先于验证行为（交互）
❌ 不要 Mock 过多的类（说明类的耦合太高）
❌ 不要使过于复杂的 Mock 链
```

## 三、集成测试

### 3.1 Spring Boot 集成测试

```java
@SpringBootTest(webEnvironment = WebEnvironment.RANDOM_PORT)
class OrderControllerIntegrationTest {
    @Autowired TestRestTemplate restTemplate;

    @Test
    void shouldCreateOrder() {
        CreateOrderRequest req = new CreateOrderRequest(1L, 2);
        ResponseEntity<OrderDto> resp = restTemplate.postForEntity(
            "/api/orders", req, OrderDto.class);
        assertEquals(200, resp.getStatusCode().value());
        assertNotNull(resp.getBody().getId());
    }
}
// 会启动真实 Spring 容器，连接真实数据库
```

### 3.2 Testcontainers（⭐ 强烈推荐）

```java
@SpringBootTest
@Testcontainers
class UserServiceIntegrationTest {
    @Container
    static PostgreSQLContainer<?> postgres = new PostgreSQLContainer<>("postgres:15")
        .withDatabaseName("test")
        .withUsername("test")
        .withPassword("test");

    @DynamicPropertySource
    static void configure(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", postgres::getJdbcUrl);
    }

    @Test
    void shouldSaveUser() { /* 真实的 PostgreSQL 环境 */ }
}
// 每次测试启动一个 Docker 容器，测试完自动销毁
// 解决了"集成测试依赖本地数据库"的问题
```

## 四、契约测试（Contract Testing）

```
问题： Provider 改了 API，Consumer 不知道，上线就挂

解决方案：契约测试（Pact）

Consumer 端：
1. 定义期望的 API 响应格式（契约）
2. 生成 Pact 文件

Provider 端：
1. 读取 Pact 文件
2. 验证 API 响应是否满足契约

工具：Pact、Spring Cloud Contract
```

## 五、压力测试

### 5.1 压测指标

| 指标 | 说明 | 目标 |
|------|------|------|
| QPS（每秒请求数） | 吞吐量 | 越高越好 |
| RT（响应时间） | P50/P95/P99 | P99 < 200ms |
| 错误率 | 失败请求 / 总请求 | < 0.1% |
| 并发数 | 同时处理的请求数 | 逐步增加找拐点 |

### 5.2 压测工具对比

| 工具 | 协议 | 优点 | 缺点 |
|------|------|------|------|
| JMeter | HTTP/TCP 等 | GUI 操作，插件多 | 资源消耗大 |
| wrk | HTTP | 轻量，性能极高 | 只支持 HTTP |
| ab | HTTP | 简单 | 功能弱 |
| Gatling | HTTP | Scala DSL，报告美观 | 学习曲线陡 |
| k6 | HTTP/WebSocket | 代码化，CI 友好 | 相对较新 |

## 六、测试覆盖率策略

```
单元测试覆盖率目标：
- Service 层：≥ 80%
- Controller 层：≥ 60%（重点测参数校验）
- DAO 层：由集成测试覆盖
- 工具类：≥ 90%

增量覆盖率（推荐）：
- 新增代码行覆盖率 ≥ 90%
- 新增代码分支覆盖率 ≥ 80%
```

## 七、CI 中的测试流水线

```yaml
# .github/workflows/ci.yml
jobs:
  test:
    steps:
      - name: Unit Test
        run: mvn test

      - name: Integration Test
        run: mvn verify -P integration-test

      - name: Coverage Check
        run: |
          COVERAGE=$(awk -F',' '/Total/{print $4}' target/site/jacoco/jacoco.csv)
          if (( $(echo "$COVERAGE < 80" | bc -l) )); then
            echo "❌ Coverage $COVERAGE% < 80%"
            exit 1
          fi
```
