## 为什么需要 MyBatis？

JDBC 直接使用有以下痛点：
- 大量样板代码（加载驱动、创建连接、创建 Statement、处理结果集、关闭资源）
- SQL 散落在代码里，改 SQL 要重新编译
- 结果集映射手动处理，字段多时非常繁琐
- 没有缓存机制

ORM 框架（如 Hibernate）又过度封装，复杂 SQL 难以优化。

**MyBatis 的定位**：半自动化 ORM —— SQL 由开发者编写（可控），但结果映射和参数绑定自动化（省事）。

## 核心概念速查

```
MyBatis 核心组件
├── Configuration（全局配置，单例）
├── SqlSessionFactory（工厂，创建 SqlSession）
├── SqlSession（会话，类似 Connection）
├── Executor（执行器，真正执行 SQL）
│   ├── SimpleExecutor（默认）
│   ├── ReuseExecutor（复用 Statement）
│   └── BatchExecutor（批量）
├── MappedStatement（每条 SQL 的封装）
├── MapperProxy（Mapper 接口的 JDK 动态代理）
└── Interceptor（插件，拦截 Executor/ParameterHandler/ResultSetHandler/StatementHandler）
```
