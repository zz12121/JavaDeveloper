# 线程池的核心参数有哪些？拒绝策略呢？

> ⚠️ **先盲答**：线程池是什么？它为什么能提升性能？核心参数有哪些？

## 盲答引导

1. 线程池解决的是什么问题？—— 为什么不能无限制创建线程？
2. 线程池有哪 **7 个核心参数**？每个参数的作用是什么？
3. 拒绝策略是什么？JDK 自带了哪几种？
4. 线程池里的线程在什么情况下会被回收？核心线程会吗？

## 知识链提示

```
线程池
  → [[线程池]]
    → 核心参数（7个）
      → corePoolSize：常驻核心线程数
        → allowCoreThreadTimeOut → 核心线程也能被回收
      → maximumPoolSize：最大线程数
        → 任务队列满了 + 核心线程都在用 → 扩到 max
      → keepAliveTime / unit：多余线程的存活时间
      → workQueue：任务队列
        → [[阻塞队列]] → 有界 vs 无界
          → 有界（ArrayBlockingQueue）：拒绝策略更可能触发
          → 无界（LinkedBlockingQueue）：线程不会超过 corePoolSize，但可能 OOM
      → threadFactory：线程工厂（自定义线程名、守护线程）
      → handler：拒绝策略
        → AbortPolicy（抛异常）→ 默认
        → DiscardPolicy（静默丢弃）
        → DiscardOldestPolicy（丢弃最老的）
        → CallerRunsPolicy（调用者执行）
    → 执行流程
      → 任务来 → core 未满 → 核心线程执行
      → core 满了 → 队列未满 → 入队
      → 队列满了 + max 未满 → 创建新线程
      → max 满了 → 触发拒绝策略
```

## 核心追问

| 追问 | 考点 |
|------|------|
| 为什么不推荐用 Executors 创建线程池？ | OOM 风险（无界队列） |
| 如何合理设置线程池参数？ | CPU 密集 vs IO 密集 |
| 线程池的线程是怎么被复用的？ | worker 不断从队列取任务 |
| 线程池 shutdown() 和 shutdownNow() 的区别？ | 是否中断正在执行的任务 |

---

## 参考答案要点

**7 个参数**：corePoolSize / maximumPoolSize / keepAliveTime / unit / workQueue / threadFactory / handler

**为什么不推荐 Executors**：`newCachedThreadPool`（无界 Integer.MAX）+ `newFixedThreadPool`（无界 LinkedBlockingQueue）都可能 OOM。

**合理配置**：
- CPU 密集型：core = CPU核心数 + 1，max = core
- IO 密集型：core = CPU核心数 × 2（或根据 IO 等待比例调大），max = core
