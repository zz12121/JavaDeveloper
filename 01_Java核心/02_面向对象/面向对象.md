# 面向对象

> 面向对象不是"三大特性背诵题"，是 Java 解决"如何管理复杂度"这个工程问题的系统性答案。

---

## 这个问题为什么存在？

面向过程编程（C 语言时代）的核心困境：程序规模一大，全局变量和函数之间的依赖关系就乱成一锅粥——任何人修改任何函数，都可能影响到完全不相关的地方。

面向对象（OOP）解决的是同一个根本问题：**如何给代码划定边界，让大规模软件可以被独立开发、独立修改、独立测试。**

三大特性（封装、继承、多态）不是独立的概念——它们相互配合，服务于这个同一个目标。

---

## 核心脉络

### 封装：划定边界

封装的本质是**把"可以改变的内部实现"藏起来，只暴露"稳定的接口"**。它是一切 OOP 设计的基础——没有封装，继承和多态都无从谈起。

访问控制（private / protected / public）是封装的工具，`[[01_Java核心/01_基础语法/final关键字|final]]` 是封装的强化手段。真正的封装不是"给字段加 getter/setter"，而是让字段的修改通过有业务语义的方法进行。

详见 [[01_Java核心/02_面向对象/封装|封装]]。

### 继承：代码复用的利与弊

继承提取公共行为到父类，子类只写自己特有的部分。但它也是 Java 中**耦合度最高的关系**——父类一改，子类可能悄悄出 bug。

**组合优于继承**是 OOP 的核心设计原则。继承只适合 is-a 关系（狗 is-a 动物），如果只是为了复用代码，组合（has-a）更安全。

详见 [[01_Java核心/02_面向对象/继承|继承]]。

### 多态：灵活性的来源

多态让你写出"对接口编程"的代码，而不是"对实现编程"。新增类型不需要修改调用方代码——这就是**开闭原则**的体现。

底层靠虚方法表（vtable）实现运行时动态分派。静态方法、`[[01_Java核心/01_基础语法/final关键字|final]]` 方法、private 方法不参与多态。

详见 [[01_Java核心/02_面向对象/多态|多态]]。

### 抽象类与接口：两种抽象机制

抽象类用于 is-a 关系和模板方法模式，接口用于 can-do 关系和能力契约。Java 8 的 default 方法缩小了两者的差距，但设计意图仍然不同：抽象类共享状态和实现，接口只约定行为。

详见 [[01_Java核心/02_面向对象/抽象类与接口|抽象类与接口]]。

### Object 类：所有类的根

Java 中所有类都隐式继承 Object。最重要的三个方法：equals() / hashCode() 的契约（违背会导致 [[01_Java核心/05_集合框架/集合框架|HashMap / HashSet]] 行为异常）、toString()、wait() / notify()。

详见 [[01_Java核心/02_面向对象/Object类|Object类]]。

### 内部类：共享封装边界

当一个类型只在另一个类型内部才有意义时，内部类让两个紧密耦合的类型共享封装边界。四种形态（静态内部类、成员内部类、局部内部类、匿名内部类）各有适用场景。匿名内部类是 [[01_Java核心/08_Lambda与Stream/Lambda与函数式|Lambda]] 的前身，理解 `this$0` 机制是避免内存泄漏的关键。

详见 [[01_Java核心/02_面向对象/内部类|内部类]]。

### 枚举：类型安全的常量集合

枚举的底层是一个继承了 `Enum<T>` 的 final 类，每个枚举值是一个 `public static final` 实例。它是实现单例模式的最佳方式——天然线程安全、防反射攻击、防序列化破坏。枚举的每个值可以独立实现抽象方法，是策略模式的内建实现。

详见 [[01_Java核心/02_面向对象/枚举|枚举]]。

### 克隆与拷贝：对象复制的三个层次

Java 赋值只拷贝引用。`Object.clone()` 默认做浅拷贝（共享引用类型字段），[[01_Java核心/02_面向对象/克隆与拷贝|克隆与拷贝]] 详解了浅拷贝与深拷贝的区别，以及 Cloneable 的设计缺陷和替代方案。

## 与其他维度的关系

- [[01_Java核心/01_基础语法/基础语法|基础语法]] 中的值传递和自动装箱是理解 OOP 行为的基础
- [[01_Java核心/01_基础语法/final关键字|final关键字]] 的"禁止继承/重写/赋值"直接服务于 OOP 的封装和多态
- [[01_Java核心/04_泛型/泛型|泛型]] 的类型擦除和边界设计影响了多态在集合中的表现
- [[01_Java核心/05_集合框架/集合框架|集合框架]] 的 equals/hashCode 契约依赖于 Object 类的正确覆写
- [[01_Java核心/07_注解与反射/反射|反射]] 是多态的"运行时镜像"——能在运行时调用任意方法，绕过编译期类型检查
- [[01_Java核心/08_Lambda与Stream/Lambda与函数式|Lambda]] 是匿名内部类的进化形态，理解内部类是理解 Lambda 的前提
