# 字节码是怎么回事？你能看懂吗？

> ⚠️ **先盲答**：.class 文件里存的是什么？JVM 怎么执行它的？

---

## 盲答引导

1. `javac` 编译后生成的 .class 文件是什么格式？—— 魔数 + 常量池 + ...
2. 字节码里有哪些核心结构？—— 常量池 / 方法表 / 属性表
3. 操作数栈和局部变量表分别是什么？—— 在栈帧里
4. `invokespecial / invokevirtual / invokeinterface / invokedynamic` 分别调用什么？

---

## 知识链提示

```
字节码
  → [[字节码]]
    → .class 文件结构
      → 魔数：0xCAFEBABE（咖啡宝贝）
      → 版本号：minor_version + major_version
      → 常量池：字面量（字符串/数字）/ 符号引用（类/方法/字段）
      → 访问标志：public / final / interface / abstract
      → 类索引 / 父类索引 / 接口索引
      → 字段表 / 方法表 / 属性表
    → 字节码执行模型（栈帧）
      → 局部变量表：方法参数 + 局部变量
      → 操作数栈：字节码指令的操作数存放地
        → `iconst_1`：把 1 压入操作数栈
        → `istore_0`：把栈顶 int 存入局部变量表 slot 0
      → 动态链接：指向运行时常量池的方法引用
      → 方法返回地址
    → 方法调用指令
      → invokespecial：私有方法 / 构造方法 / super()
      → invokevirtual：虚方法（多态，运行时绑定）
      → invokeinterface：接口方法
      → invokedynamic：lambda / 反射（JDK 7+）
```

## 核心追问

| 追问 | 考点 |
|------|------|
| 为什么 JVM 要用「栈」而不是「寄存器」？ | 平台无关性（寄存器数量因 CPU 而异） |
| 方法重载（overload）和重写（override）在字节码层面有区别吗？ | 重载是静态分派（编译期），重写是动态分派（运行期） |
| `invokedynamic` 是为了解决什么问题？ | Lambda 表达式 / 动态语言支持 |
| `javap -c` 的输出你能看懂吗？ | 能看懂简单的（面试题可能让你手写字节码反编译结果） |

---

## 参考答案要点

**.class 文件核心结构**：魔数 + 版本 + 常量池 + 类/父类/接口 + 字段 + 方法 + 属性。

**栈帧操作示例**：
```java
int a = 1;
int b = 2;
int c = a + b;
```
字节码：
```
iconst_1     // 1 入操作数栈
istore_0     // 存入 slot 0（a）
iconst_2     // 2 入栈
istore_1     // 存入 slot 1（b）
iload_0      // 加载 a 到栈
iload_1      // 加载 b 到栈
iadd         // 弹出两个 int，相加，结果入栈
istore_2     // 存入 slot 2（c）
```
