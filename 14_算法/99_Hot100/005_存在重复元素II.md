## 题干

给定一个整数数组 `nums` 和一个整数 `k`，判断是否存在两个**不同的下标** `i` 和 `j`，满足 `nums[i] == nums[j]` 且 `abs(i - j) <= k`。

### 示例

```
输入：nums = [1,2,3,1], k = 3
输出：true

输入：nums = [1,0,1,1], k = 1
输出：true
```

我的解题答案：
```java
private static boolean getResult(int[] a, int k) {  
    HashMap<Integer, Integer> paramHashMap = new HashMap<>();  
    for(int i =0; i< a.length; i++) {  
        if (paramHashMap.containsKey(a[i])) {  
            if (Math.abs(i - paramHashMap.get(a[i])) <= k) {  
                return true;  
            }  
        } else {  
            paramHashMap.put(a[i], i);  
        }  
    }  
    return false;  
}
```
1.不能在判断是否满足条件后，不成立后再赋值最新的下标，这里隐含了如果正在判断的下标都不符合条件，那么比他更大的下标也不可能符合，此时需要将下标更新为当前下标。
```java
private static boolean getResult(int[] nums, int k) {
    if (nums == null || k < 0) return false; // 边界更健壮
    
    Map<Integer, Integer> lastIndexMap = new HashMap<>(); // key:数, val:最近下标
    
    for (int i = 0; i < nums.length; i++) {
        int num = nums[i];
        
        if (lastIndexMap.containsKey(num)) {
            int prevIndex = lastIndexMap.get(num);
            // 只要当前下标与上一次出现的下标差 ≤ k 就满足
            if (Math.abs(i - prevIndex) <= k) {
                return true;
            }
        }
        
        lastIndexMap.put(num, i); // 无论是否找到，都要更新为最近下标
    }
    return false;
}
```

## 🚀 进阶：滑动窗口（空间更省）

维护一个长度为 `k+1`的窗口（Set），看窗口内是否有重复：

```java
public boolean containsNearbyDuplicate(int[] nums, int k) {
    Set<Integer> window = new HashSet<>();
    for (int i = 0; i < nums.length; i++) {
        if (window.contains(nums[i])) return true;
        window.add(nums[i]);
        if (i >= k) window.remove(nums[i - k]); // 保持窗口大小≤k+1
    }
    return false;
}
```

**优点**：空间 O(k)，当 k远小于 n时更省内存。
