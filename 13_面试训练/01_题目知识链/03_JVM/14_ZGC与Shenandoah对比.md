## 盲答引导

1. ZGC 的「着色指针」是什么？指针上能存多少位信息？
2. Shenandoah 用的是「Brooks 指针」，它和 ZGC 的着色指针有什么本质区别？
3. ZGC 的「读屏障」和 G1 的「写屏障」哪个开销更大？为什么？
4. 为什么 ZGC 和 Shenandoah 的停顿时间和堆大小基本无关？

## 知识链提示

这道题应该让你联想到：

- `[[着色指针 Colored Pointers]]` → ZGC 在指针上染色，不需要写屏障
- `[[Brooks 指针]]` → Shenandoah 在每个对象头里加一个 forwarding 指针
- `[[读屏障 vs 写屏障]]` → ZGC 用读屏障，Shenandoah 用读写双屏障
- `[[转移（Relocation）]]` → 两者都支持并发转移，是低停顿的关键
- `[[与 G1 对比]]` → G1 的写屏障维护 RSet，开销大

## 参考要点（盲答后再看）

<details>
<summary>点击展开</summary>

**ZGC 核心原理**：

| 特性 | ZGC | Shenandoah |
|------|------|--------------|
| 染色位置 | 指针（64位地址） | 对象头（Brooks 指针） |
| 屏障类型 | 读屏障（Load Barrier） | 读+写屏障 |
| 分代 | JDK21+ 支持分代 | 不分代 |
| 停顿时间 | <1ms（通常） | <10ms（通常） |
| 适用堆 | 几百 MB ~ 16TB | 同上 |

**着色指针（ZGC）**：
```
64位指针布局（Linux x86）：
[63..42] 保留位（22位）
[41..40] 染色位（2位：Marked0/Marked1/Remapped/Finalizable）
[39..0]  对象地址（44位，最大 16TB）
```
→ 读指针时发现未标记 → 触发标记/转移，对应用透明。

**Brooks 指针（Shenandoah）**：
```java
// 每个对象头里多一个指针
class Object {
    Object brooksPointer;  // 指向自己（未转移）或新位置（已转移）
    // ... 对象字段
}
// 读对象时：如果 brooksPointer != this → 重定向到新位置
```

**两者低停顿的根本原因**：
- 标记阶段：并发（三色标记 + 屏障）
- 转移阶段：并发（转移时读屏障重定向引用）
- 不需要 STW 做全堆转移

**生产选型建议**：
| 场景 | 推荐 |
|------|------|
| 堆 < 4GB，延迟要求不极端 | G1（成熟稳定） |
| 堆 > 8GB，要求亚毫秒停顿 | ZGC（JDK17+ 推荐） |
| 想要比 G1 更低停顿，但不想用 ZGC | Shenandoah（JDK12+） |

**ZGC 分代（JDK21 新特性）**：
- 不分代的缺点：全堆扫描，对象存活多时标记开销大
- 分代 ZGC：年轻代单独回收，减少标记范围，吞吐更高
