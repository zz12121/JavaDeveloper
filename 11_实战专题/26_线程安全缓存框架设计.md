# 线程安全缓存框架设计

> 设计一个类似 Guava Cache 的线程安全缓存框架，核心是：**ConcurrentHashMap 分段锁 + 淘汰策略（LRU/LFU）+ 容量控制 + 过期清理**。重点理解无锁读操作和并发淘汰的实现。

---

## 架构设计

```
┌──────────────────────────────────────┐
│           缓存框架                   │
│  ┌──────────────────────────────┐  │
│  │    ConcurrentHashMap          │  │
│  │  (存储缓存条目)                │  │
│  └───────────┬──────────────────┘  │
│              │                       │
│  ┌───────────▼──────────────────┐  │
│  │    淘汰算法                   │  │
│  │  (LRU 队列 / LFU 计数)       │  │
│  └───────────┬──────────────────┘  │
│              │                       │
│  ┌───────────▼──────────────────┐  │
│  │    过期清理线程                │  │
│  │  (懒清理 + 定时清理)          │  │
│  └──────────────────────────────┘  │
└──────────────────────────────────────┘
```

## 场景 A：ConcurrentHashMap 实现缓存（简化版）

```java
public class SimpleCache<K, V> {
    private final Map<K, V> cache = new ConcurrentHashMap<>();
    private final Map<K, Long> expireAt = new ConcurrentHashMap<>();
    private final int maxSize;
    
    public SimpleCache(int maxSize) {
        this.maxSize = maxSize;
    }
    
    public V get(K key) {
        Long exp = expireAt.get(key);
        if (exp != null && exp < System.currentTimeMillis()) {
            remove(key);  // 懒清理
            return null;
        }
        return cache.get(key);
    }
    
    public void put(K key, V value, long ttlSeconds) {
        if (cache.size() >= maxSize) {
            // 简化版淘汰：随机删除一个（生产用 LRU）
            K randomKey = cache.keySet().iterator().next();
            remove(randomKey);
        }
        cache.put(key, value);
        expireAt.put(key, System.currentTimeMillis() + ttlSeconds * 1000);
    }
    
    private void remove(K key) {
        cache.remove(key);
        expireAt.remove(key);
    }
}
```

**问题**：淘汰策略太简单，没有 LRU/LFU。实际生产用 Guava Cache 或 Caffeine。

## 场景 B：LRU 淘汰算法实现

```java
// 线程安全的 LRU 缓存（用 ReadWriteLock）
public class LruCache<K, V> {
    private final int maxSize;
    private final Map<K, Node<K, V>> map;
    private final Node<K, V> head, tail;  // 双向链表
    private final ReadWriteLock lock = new ReentrantReadWriteLock();
    
    // 访问时移到队尾（最久未使用在队头）
    public V get(K key) {
        lock.readLock().lock();
        try {
            Node<K, V> node = map.get(key);
            if (node != null) {
                moveToTail(node);  // 移到队尾（最近使用）
                return node.value;
            }
            return null;
        } finally {
            lock.readLock().unlock();
        }
    }
    
    // 插入时检查容量，超过则删除队头
    public void put(K key, V value) {
        lock.writeLock().lock();
        try {
            Node<K, V> node = map.get(key);
            if (node != null) {
                node.value = value;
                moveToTail(node);
            } else {
                node = new Node<>(key, value);
                map.put(key, node);
                addToTail(node);
                
                if (map.size() > maxSize) {
                    Node<K, V> removed = removeHead();  // 淘汰最久未使用
                    map.remove(removed.key);
                }
            }
        } finally {
            lock.writeLock().unlock();
        }
    }
}
```

**Caffeine 方案**：用 `ConcurrentHashMap` + W-TinyLFU 算法，无锁读，性能碾压上面的方案。面试直接说「生产用 Caffeine」。

## 场景 C：缓存穿透/击穿防护

```java
// 方案1：布隆过滤器（防止缓存穿透）
public class BloomFilterCache {
    private final BloomFilter<String> bloomFilter = 
        BloomFilter.create(Funnels.stringFunnel(), 10_000_000, 0.01);
    
    public User getUser(Long id) {
        // 1. 布隆过滤器快速判断（不存在的一定不存在）
        if (!bloomFilter.mightContain("user:" + id)) {
            return null;  // 一定不存在，不查缓存和 DB
        }
        
        // 2. 查缓存
        User user = cache.get("user:" + id);
        if (user != null) return user;
        
        // 3. 查 DB
        user = userMapper.selectById(id);
        if (user != null) {
            cache.put("user:" + id, user);
        } else {
            // 空值缓存（防止穿透，TTL 设短一点）
            cache.put("user:" + id, null, 60);
        }
        return user;
    }
}
```
