# 并发编程 — 全景地图

> 并发是区分中级和高级的分水岭。不是学会用线程，而是理解「为什么线程不安全」。

---

## 学什么

```
并发编程
├── 01_线程基础          — 进程vs线程、生命周期、创建方式、中断
├── 02_JMM内存模型       — 主内存/工作内存、Happens-Before、指令重排序、内存屏障
├── 03_volatile          — 可见性+有序性、不保证原子性、应用场景
├── 04_synchronized      — 锁升级、对象头、ObjectMonitor、锁消除/粗化
├── 05_CAS与原子类        — CAS原理、ABA问题、AtomicInteger/LongAdder
├── 06_AQS               — state+FIFO+模板方法、独占/共享模式、Condition
├── 07_Lock              — ReentrantLock、ReadWriteLock、StampedLock
├── 08_并发容器          — ConcurrentHashMap、CopyOnWrite、跳表
├── 09_阻塞队列          — ArrayBlockingQueue、SynchronousQueue、DelayQueue
├── 10_线程池            — ThreadPoolExecutor参数与流程、拒绝策略、调优
├── 11_ForkJoin          — 分治、工作窃取、RecursiveTask
├── 12_CompletableFuture — 异步编排、链式调用、异常处理、超时控制
├── 13_并发工具类         — CountDownLatch、CyclicBarrier、Semaphore
├── 14_死锁与活锁         — 必要条件、预防检测、jstack分析
├── 15_ThreadLocal       — 内存泄漏、InheritableThreadLocal、TTL
├── 16_虚拟线程          — JDK21、载体线程、Continuation、结构化并发
└── 17_实战应用          — 生产者消费者、并发缓存、限流、Disruptor
```

## 怎么学

**顺序**：01（基础概念）→ 02（JMM是理解一切并发问题的根基）→ 03/04/05（三大基础机制）→ 06（AQS是锁的骨架）→ 07/08/09/10（工具应用）→ 11/12/13（高级编排）→ 14/15（陷阱与陷阱）→ 16（未来方向）→ 17（融会贯通）

**关键认知**：
- JMM 是理解并发问题的**根本框架**——可见性、有序性、原子性三个问题都从这里来
- synchronized 和 Lock 是两条**并行的锁实现路径**，底层机制完全不同
- AQS 是 JUC 的**骨架**——ReentrantLock、Semaphore、CountDownLatch 都基于它
- 线程池不是"用了就行"，参数设置和拒绝策略直接影响线上稳定性
- 虚拟线程是并发编程的未来，但不能替代所有场景

## 和其他域的关系

```
02_并发 ←→ 01_Java核心   （多线程 → Thread类；集合 → 并发容器）
02_并发 ←→ 03_JVM        （JMM → 内存模型；线程 → JVM运行时数据区）
02_并发 ←→ 05_数据库      （连接池 → 线程池；事务隔离 → 并发控制）
02_并发 ←→ 06_中间件      （Redis → 分布式锁；MQ → 生产者消费者）
02_并发 ←→ 07_分布式      （分布式锁 → CAP；幂等 → 并发安全）
```
