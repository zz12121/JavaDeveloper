## 参考要点（盲答后再看）

<details>
<summary>点击展开</summary>

**注解的本质**：**注解是接口**，编译后变成 `Annotation` 的子接口。
```java
// 源码
@interface MyAnno { String value(); }

// 编译后等价于
public interface MyAnno extends Annotation {
    String value();
}
```

**三种保留策略（必须记住）**：

| 策略 | 保留到 | 典型用途 |
|------|--------|---------|
| `SOURCE` | 源码 only | `@Override`（编译器检查）、Lombok |
| `CLASS` | 字节码（默认） | 字节码增强工具（ASM/Javassist） |
| `RUNTIME` | 运行时 | Spring `@Autowired`、JUnit `@Test` |

**运行时读取注解（反射）**：
```java
// 读取方法上的注解
Method m = obj.getClass().getMethod("foo");
if (m.isAnnotationPresent(MyAnno.class)) {
    MyAnno anno = m.getAnnotation(MyAnno.class);
    System.out.println(anno.value());
}
```

**编译期 APT（Annotation Processing Tool）**：
- 执行时机：编译期间，在 `parse→analyze→generate` 的分析阶段之后
- 可以：生成新文件（`.java`）、修改 AST（JDK8 及之前有限制）
- 典型应用：`@Builder`（Lombok）、`@Entity`（JPA 元模型生成）
- 不能：修改现有方法的字节码（这是 Javac 之后的事）

**Spring 注解驱动的核心流程**（简述）：
1. 启动 → 扫描类路径（ASM 读字节码，不加载类）
2. 找到 `@Component` 及其派生注解（`@Service`、`@Repository`）
3. 注册为 `BeanDefinition` → 放入 IoC 容器
4. 处理 `@Autowired` → 后置处理器 `AutowiredAnnotationBeanPostProcessor`

**`@Inherited` 的陷阱**：
- 只对**类继承**有效，对**接口实现**无效
- `@Service` 没有 `@Inherited`，但 Spring 通过类路径扫描绕过了这个限制
