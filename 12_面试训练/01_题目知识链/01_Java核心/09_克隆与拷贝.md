**Object.clone() 默认行为（浅拷贝）**：

```java
class A implements Cloneable {
    int x;
    List<String> list;

    @Override
    public A clone() {
        return (A) super.clone();
        // 等价于：
        // 1. 分配新内存（类型大小 = A对象大小）
        // 2. 逐字段拷贝（基本类型拷贝值，引用类型拷贝引用）
        // 结果：x不同，但list引用相同（共享同一个ArrayList对象）
    }
}
```

```
浅拷贝：
A@1001 { x=1, list=@2001 }    ← 原对象
A@3001 { x=1, list=@2001 }    ← 克隆对象（list引用相同！）
         ↑
      同一个ArrayList对象
```

**深拷贝实现方式**：

```java
// 方式1：手动递归拷贝
@Override
public A clone() {
    A result = (A) super.clone();
    result.list = new ArrayList<>(this.list);  // 新建一个ArrayList
    return result;
}

// 方式2：序列化（支持循环引用）
public static <T> T deepClone(T obj) throws Exception {
    ByteArrayOutputStream bos = new ByteArrayOutputStream();
    ObjectOutputStream oos = new ObjectOutputStream(bos);
    oos.writeObject(obj);
    ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
    ObjectInputStream ois = new ObjectInputStream(bis);
    return (T) ois.readObject();
}

// 方式3：JSON序列化（Jackson/Gson）
String json = new ObjectMapper().writeValueAsString(obj);
A cloned = new ObjectMapper().readValue(json, A.class);
```

**Arrays.asList() / ArrayList<>(existingList) 是浅拷贝**：
```java
List<String> original = new ArrayList<>();
original.add("a");

List<String> copy = new ArrayList<>(original);  // 底层数组是同一个引用
copy.add("b");
original.contains("b");  // ✅ 返回 true（同一个数组！）
```

**Cloneable 的本质**：
```java
// Cloneable 没有任何方法
public interface Cloneable {}

// Object.clone() 源码：
public native Object clone() throws CloneNotSupportedException {
    if (!(this instanceof Cloneable)) {
        throw new CloneNotSupportedException();  // 关键检查！
    }
    // 否则 native 拷贝内存
}
```
`Cloneable` 只是一个「标记」，告诉 JVM 可以执行 native 拷贝。
