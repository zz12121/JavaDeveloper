# 进程与线程

> **核心认知**：进程和线程是操作系统的基本调度单位。理解它们的本质区别、生命周期、调度算法、上下文切换机制，是理解 Java 并发编程、JVM 运行时数据区、容器化技术的底层基础。

---

## 1. 进程

### 1.1 什么是进程

**进程（Process）是操作系统分配资源的基本单位，是程序的一次执行过程。**

```
程序 vs 进程：
  程序 = 磁盘上的可执行文件（静态的）
  进程 = 运行中的程序 + 资源分配（内存、文件、CPU 时间）（动态的）

一个程序可以创建多个进程：
  java -jar app.jar     → 启动一个 JVM 进程
  java -jar app.jar     → 再启动一个 JVM 进程（与上面是独立的）
```

### 1.2 进程的组成

```
┌─────────────────────────────┐
│  PCB（进程控制块）            │  进程的"身份证"
│  - PID（进程 ID）           │
│  - 状态（运行/就绪/阻塞）     │
│  - 优先级                   │
│  - 程序计数器（PC）           │
│  - 内存管理信息              │
│  - 打开的文件描述符           │
│  - 信号处理                 │
├─────────────────────────────┤
│  代码段（Text Segment）       │  程序的机器指令（只读）
├─────────────────────────────┤
│  数据段（Data Segment）       │  全局变量、静态变量
├─────────────────────────────┤
│  堆（Heap）                  │  动态分配的内存（malloc/new）
├─────────────────────────────┤
│  栈（Stack）                 │  函数调用栈、局部变量
└─────────────────────────────┘
```

### 1.3 进程的状态

```
                    ┌──────────┐
          就绪 ────→│  运行    │
           ↑        └────┬─────┘
           │             │ 时间片用完 / 被抢占
           │             ↓
           │        ┌──────────┐
           │        │  就绪    │
           │        └────┬─────┘
           │             │
           │     I/O 完成 / 获得锁
           │             │
           │        ┌────┴─────┐
           └────────│  阻塞    │
                    └──────────┘

        运行 → 阻塞：等待 I/O、等待锁、sleep
        阻塞 → 就绪：I/O 完成、获得锁、sleep 到期
        就绪 → 运行：被调度器选中
        运行 → 就绪：时间片用完、被更高优先级进程抢占
        运行 → 结束：进程执行完毕 / 被终止
```

### 1.4 进程间通信（IPC）

| 方式 | 原理 | 特点 |
|------|------|------|
| 管道（Pipe） | 半双工，数据单向流动 | 父子进程间通信 |
| 命名管道（FIFO） | 半双工，有文件系统路径 | 无亲缘关系进程间通信 |
| 消息队列 | 内核中的消息链表 | 有格式、有优先级 |
| 共享内存 | 映射同一块物理内存 | 最快（零拷贝），需同步 |
| 信号量 | 计数器，控制对共享资源的访问 | 常与共享内存配合使用 |
| 信号（Signal） | 异步通知机制 | 如 SIGKILL、SIGTERM |
| Socket | 网络通信，也支持本地 | 跨机器通信 |

## 2. 线程

### 2.1 什么是线程

**线程（Thread）是 CPU 调度的基本单位，是进程中的一个执行单元。**

```
进程 vs 线程：
  进程 = 资源分配单位（内存空间、文件描述符）
  线程 = CPU 调度单位（执行指令）

一个进程可以包含多个线程：
  ┌───────────────────────────────┐
  │  JVM 进程                      │
  │  ┌───────┐ ┌───────┐ ┌──────┐│
  │  │ main  │ │ GC    │ │ ...  ││  多个线程共享：
  │  │ 线程   │ │ 线程   │ │ 线程  ││  - 堆内存
  │  └───────┘ └───────┘ └──────┘│  - 方法区/元空间
  │  ┌──────────────────────────┐ │  - 打开的文件描述符
  │  │ 线程私有：               │ │
  │  │ - 程序计数器（PC）        │ │  线程私有：
  │  │ - 虚拟机栈（栈帧+局部变量）│ │  - 程序计数器
  │  │ - 本地方法栈             │ │  - 虚拟机栈
  │  │ - 线程本地存储（TLS）     │ │  - 本地方法栈
  │  └──────────────────────────┘ │
  └───────────────────────────────┘
```

### 2.2 线程私有 vs 线程共享

| 区域 | 共享/私有 | 说明 |
|------|----------|------|
| 程序计数器（PC） | **私有** | 每个线程执行到不同的代码位置 |
| 虚拟机栈 | **私有** | 栈帧中的局部变量表、操作数栈等 |
| 本地方法栈 | **私有** | Native 方法调用 |
| 堆 | **共享** | 所有线程共享的对象实例 |
| 方法区/元空间 | **共享** | 类信息、常量、静态变量 |
| 直接内存 | **共享** | NIO 的 DirectByteBuffer |

### 2.3 线程的生命周期（Java Thread）

```
         new Thread()
              │
              ↓
          ┌────────┐   start()    ┌──────────┐
          │  NEW   │────────────→│ RUNNABLE │
          └────────┘             └────┬─────┘
                                       │
                         ┌─────────────┼─────────────┐
                         │             │             │
                    获得锁          sleep/join    等待 I/O
                         │           等待通知        │
                         ↓             │             │
                   ┌──────────┐       ↓             ↓
                   │BLOCKED   │  ┌──────────┐  ┌──────────┐
                   │(等锁)    │  │ WAITING  │  │TIMED    │
                   └────┬─────┘  │(无限等待) │  │WAITING  │
                        │        └────┬─────┘  │(超时等待)│
                     获得锁          │        └────┬─────┘
                        │        notify/notifyAll  │
                        │        notify/超时到期     │
                        ↓             │             │
                   ┌──────────┐       │             │
                   │ RUNNABLE │←──────┘─────────────┘
                   └────┬─────┘
                        │
                    执行完毕 / 异常
                        │
                        ↓
                   ┌──────────┐
                   │TERMINATED│
                   └──────────┘
```

**状态对应关系**（Java vs OS）：

| Java 状态 | OS 状态 | 说明 |
|-----------|---------|------|
| NEW | - | 对象创建但未 start() |
| RUNNABLE | Ready / Running | Java 统一为 RUNNABLE，不区分就绪和运行 |
| BLOCKED | Sleeping / Waiting | 等待 Monitor 锁（synchronized） |
| WAITING | Waiting | wait()/join()/park()，无超时 |
| TIMED_WAITING | Waiting | sleep(time)/wait(time)/parkNanos()，有超时 |
| TERMINATED | Terminated | 线程执行完毕 |

## 3. 调度算法

### 3.1 进程调度

| 算法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 先来先服务（FCFS） | 按到达顺序执行 | 公平 | 短作业等待长作业（护航效应） |
| 短作业优先（SJF） | 优先执行短作业 | 平均等待时间最短 | 长作业可能饿死 |
| 时间片轮转（RR） | 每个进程分配固定时间片 | 响应时间好 | 上下文切换频繁 |
| 优先级调度 | 按优先级执行 | 灵活 | 低优先级可能饿死 |
| 多级反馈队列 | 多个队列，不同时间片 | 综合以上优点 | 实现复杂 |

**Linux CFS（Completely Fair Scheduler）**：
- 红黑树维护进程的虚拟运行时间（vruntime）
- vruntime 越小的进程优先调度
- 保证所有进程公平地分享 CPU 时间

### 3.2 线程调度

Linux 中线程和进程统一对待（都叫 task），调度算法同上。Java 中的线程调度依赖操作系统调度。

## 4. 上下文切换

### 4.1 什么是上下文切换

CPU 从一个线程/进程切换到另一个线程/进程时，需要保存当前执行状态、恢复目标执行状态。

```
线程 A 的时间片用完，切换到线程 B：

  1. 保存线程 A 的上下文：
     - 程序计数器（PC）：下一条要执行的指令地址
     - 寄存器值：CPU 各寄存器的当前值
     - 栈指针（SP）：栈顶位置

  2. 恢复线程 B 的上下文（之前保存的）

  3. 跳转到线程 B 的程序计数器位置，继续执行

代价：
  - 保存/恢复寄存器：约几十纳秒
  - 切换页表（如果跨进程）：可能触发 TLB 失效
  - 刷新 CPU 缓存（L1/L2/L3）：缓存命中率下降
  - 整体耗时：微秒级（1~10μs）
```

### 4.2 上下文切换的分类

| 类型 | 场景 | 代价 |
|------|------|------|
| 进程切换 | 时间片轮转、进程调度 | **高**（需要切换地址空间、刷新 TLB） |
| 线程切换 | 同进程内的线程切换 | **低**（共享地址空间，不需要切换页表） |
| 用户态↔内核态切换 | 系统调用（read/write/malloc） | 中等（不需要切换线程，但需要保存/恢复部分寄存器） |

### 4.3 如何减少上下文切换

```java
// 1. 减少线程数量（线程池）
ExecutorService executor = new ThreadPoolExecutor(
    corePoolSize,    // CPU 密集型：N+1
    maximumPoolSize,  // IO 密集型：2N
    keepAliveTime, TimeUnit.SECONDS,
    new LinkedBlockingQueue<>(1000)
);

// 2. 使用无锁算法（CAS）减少线程阻塞
AtomicInteger count = new AtomicInteger();
count.incrementAndGet();  // CAS 操作，不会阻塞线程

// 3. 协程/虚拟线程（减少内核态线程切换）
// Java 21 虚拟线程：轻量级线程，挂起时不消耗 OS 线程
Thread.startVirtualThread(() -> {
    // 阻塞操作不会阻塞 OS 线程
});
```

### 4.4 如何监控上下文切换

```bash
# 查看系统上下文切换次数
vmstat 1
# cs 列：每秒上下文切换次数（Context Switches）
# 正常值：几千到几万。超过 10 万可能有问题

# 按进程查看上下文切换
pidstat -w 1

# 查看线程上下文切换
pidstat -wt 1
```

## 5. 进程与线程的关系

### 5.1 核心对比

| 维度 | 进程 | 线程 |
|------|------|------|
| 定义 | 资源分配的基本单位 | CPU 调度的基本单位 |
| 资源 | 拥有独立的地址空间 | 共享进程的地址空间 |
| 创建开销 | 大（需要分配独立内存空间） | 小（共享进程资源） |
| 切换开销 | 大（切换地址空间、刷新 TLB） | 小（共享地址空间） |
| 通信 | 需要IPC机制（管道、共享内存等） | 直接读写共享变量（需同步） |
| 崩溃影响 | 独立（一个崩溃不影响其他） | 一个线程崩溃可能导致整个进程崩溃 |
| 数量限制 | 受内存限制 | 受进程资源限制（栈大小、文件描述符） |

### 5.2 为什么需要多线程？

```
单线程的问题：
  - IO 等待时 CPU 空闲（利用率低）
  - 无法利用多核 CPU 的并行能力
  - GUI 程序中，一个耗时操作会冻结整个界面

多线程的优势：
  - 提高资源利用率（IO 等待时执行其他线程）
  - 提高系统吞吐量（多核并行处理）
  - 改善响应时间（异步处理耗时任务）

多线程的代价：
  - 线程安全（数据竞争、可见性、有序性）
  - 死锁
  - 上下文切换开销
  - 内存占用（每个线程需要栈空间，默认 1MB）
```

### 5.3 Java 中的进程与线程

```java
// 一个 Java 程序 = 一个 JVM 进程
// main() 方法运行在主线程中

// 查看当前 JVM 进程的 PID
// Java 9+: ProcessHandle.current().pid()
// Java 8:  ManagementFactory.getRuntimeMXBean().getName()

// 创建新线程
Thread t = new Thread(() -> System.out.println("Hello"));
t.start();

// Java 21 虚拟线程（轻量级线程）
Thread vt = Thread.startVirtualThread(() -> {
    System.out.println("Virtual Thread");
});
```
