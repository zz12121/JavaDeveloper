# 批量操作与性能优化

## 这个问题为什么存在？

> 要插入 10000 条数据，一条条 `insert` 要多久？

- 单次 insert：10000 次网络 IO + 10000 次 SQL 解析 = **慢**
- 批量操作：1 次网络 IO + 1 次 SQL 解析 = **快**

但 MyBatis 的批量有**多种方式**，用错方式反而更慢。

---

## 它是怎么解决问题的？

### 方式一：foreach 拼接 SQL（推荐）

```xml
<!-- MySQL 支持 VALUES 多行插入 -->
<insert id="batchInsert">
    INSERT INTO user (name, age) VALUES
    <foreach collection="list" item="user" separator=",">
        (#{user.name}, #{user.age})
    </foreach>
</insert>
<!-- 生成：INSERT INTO user (name, age) VALUES (?,?), (?,?), ... -->
```

**优点**：
- 只有 1 次网络 IO
- 数据库批量插入优化（MySQL 对多行 VALUES 有优化）
- 代码简单

**限制**：
- SQL 长度有限制（MySQL `max_allowed_packet`，默认 4MB）
- 超过 1000 条建议分批

### 方式二：BatchExecutor（攒批执行）

```java
try (SqlSession session = sqlSessionFactory.openSession(ExecutorType.BATCH)) {
    UserMapper mapper = session.getMapper(UserMapper.class);
    for (int i = 0; i < 10000; i++) {
        mapper.insert(user);  // ← 只 addBatch()
    }
    session.flushStatements();  // ← 真正执行
    session.commit();
}
```

**优点**：
- 内存占用低（不拼接超长 SQL）
- 适合超大批量（10万+）

**缺点**：
- 需要手动 flush/commit
- 返回影响行数需要通过 `BatchResult` 获取

### 方式三：ExecutorType.BATCH + Spring（推荐）

```java
@Autowired
private SqlSessionTemplate batchSqlSessionTemplate;  // 配置为 BATCH

public void batchInsert(List<User> users) {
    UserMapper mapper = batchSqlSessionTemplate.getMapper(UserMapper.class);
    for (User user : users) {
        mapper.insert(user);
    }
    batchSqlSessionTemplate.flushStatements();
}
```

## 深入原理

### BatchExecutor 攒批逻辑

```java
// BatchExecutor.java
@Override
public int doUpdate(MappedStatement ms, Object parameter) {
    Statement stmt = prepareStatement(handler, ms.getStatementLog());
    // 关键：只 addBatch，不 executeBatch
    stmt.addBatch();
    return BATCH_UPDATE_RETURN_VALUE;  // 返回 -1（此时还不知道影响行数）
}

// flushStatements() 时才真正执行
@Override
public List<BatchResult> doFlushStatements(boolean isRollback) {
    List<BatchResult> results = new ArrayList<>();
    for (Statement stmt : statementList) {
        int[] updateCounts = stmt.executeBatch();  // ← 真正执行
        results.add(new BatchResult(ms, sql, parameterObject, updateCounts));
    }
    return results;
}
```

### foreach 拼接方式（本质还是单条 SQL）

```
XML 中的 <foreach>
  → 解析时拼接出完整 SQL 字符串
  → INSERT INTO user (name, age) VALUES (?,?), (?,?), ...
  → 一次 PreparedStatement.executeUpdate()
  → 数据库原生支持多行插入（MySQL/PostgreSQL 都支持）
```


### 它和相似方案的本质区别是什么？

| 对比维度 | foreach 拼接 | BatchExecutor | 存储过程 | 原生 JDBC Batch |
|---------|----------------|----------------|------------|-------------------|
| 网络 IO 次数 | 1 次 | N 次（但合并发送） | 1 次（调用存储过程） | N 次（addBatch 后一起发送） |
| 内存占用 | 高（拼接 SQL） | 低 | 低（数据库内执行） | 低 |
| 数据库兼容性 | 中（部分数据库不支持多行 VALUES） | 高 | 低（数据库特定语法） | 高（JDBC 标准） |
| 代码复杂度 | 低 | 中 | 高（需写 PL/SQL） | 高 |

**本质区别**：`foreach` 拼接是**生成一条大 SQL**，BatchExecutor 是**多次 addBatch 后一次发送**。

## 正确使用方式

### foreach 分批插入（推荐）

```java
// Service 层分批
public void batchInsert(List<User> users) {
    int batchSize = 1000;  // ← 每批 1000 条
    for (int i = 0; i < users.size(); i += batchSize) {
        List<User> batch = users.subList(i, Math.min(i + batchSize, users.size()));
        userMapper.batchInsert(batch);  // ← 调用 foreach 拼接的 SQL
    }
}
```

### BatchExecutor 方式

```java
@Repository
public class UserBatchDao {
    private final SqlSessionTemplate batchTemplate;

    public UserBatchDao(@Qualifier("batchSqlSessionTemplate") SqlSessionTemplate template) {
        this.batchTemplate = template;
    }

    public void batchInsert(List<User> users) {
        UserMapper mapper = batchTemplate.getMapper(UserMapper.class);
        for (User user : users) {
            mapper.insert(user);
        }
        batchTemplate.flushStatements();  // 必须手动 flush
    }
}

// Spring 配置
@Bean("batchSqlSessionTemplate")
public SqlSessionTemplate batchSqlSessionTemplate(SqlSessionFactory factory) {
    return new SqlSessionTemplate(factory, ExecutorType.BATCH);
}
```


### 性能对比（参考值）

| 方式 | 1000 条 | 10000 条 | 说明 |
|------|----------|-----------|------|
| 逐条 insert | ~10s | ~100s | ❌ 不可用 |
| foreach 拼接（1 批） | ~0.5s | ~5s（SQL 太长） | ✅ 推荐（分批） |
| BatchExecutor | ~1s | ~8s | ✅ 适合超大批量 |
| foreach + 分批（每 1000） | ~0.5s | ~5s | ✅✅ 最推荐 |

## 边界情况和坑

### 坑1：foreach 拼接超大 SQL

```
问题：拼接出 10MB 的 SQL → MySQL 报错：PacketTooBigException
解决：分批（每批 500~1000 条，根据字段大小调整）
```

### 坑2：BatchExecutor 忘记 flush

```
mapper.insert(user);   // 只 addBatch()
// 如果这里崩溃 → 数据库中没有任何记录
// 必须：session.flushStatements()
```

### 坑3：BatchExecutor 返回的影响行数是 -1

```
BatchExecutor.doUpdate() 返回 BATCH_UPDATE_RETURN_VALUE（-1）
要获取真正的影响行数：
  List<BatchResult> results = session.flushStatements();
  for (BatchResult result : results) {
      int[] counts = result.getUpdateCounts();  // 每个 Statement 的影响行数
  }
```

### 坑4：MySQL rewriteBatchedStatements 参数

```
JDBC URL 中加上：rewriteBatchedStatements=true
  → MySQL 驱动会把多次 addBatch() 重写为一条多行 VALUES 的 SQL
  → 性能提升显著（5~10 倍）

示例 URL：
  jdbc:mysql://localhost:3306/test?rewriteBatchedStatements=true
```

### 坑5：批量更新返回自增 ID

```
foreach 批量插入时，MySQL 支持返回自增 ID：
  useGeneratedKeys="true" keyProperty="id"

但：只能返回第一批（最后一条）的 ID
  → 如果需要所有 ID，用 BatchExecutor + selectKey（麻烦）
  → 或者放弃自增 ID，用 UUID/Snowflake（应用端生成）
```
