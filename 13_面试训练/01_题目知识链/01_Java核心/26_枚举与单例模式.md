## 参考要点（盲答后再看）


**枚举的本质**：

```java
// 源码
enum Color { RED, GREEN, BLUE }

// 编译后等价于
final class Color extends Enum<Color> {
    public static final Color RED = new Color("RED", 0);
    public static final Color GREEN = new Color("GREEN", 1);
    public static final Color BLUE = new Color("BLUE", 2);
    
    private Color(String name, int ordinal) { super(name, ordinal); }
}
```

**单例模式对比**：

| 维度 | 双重检查锁 | 静态内部类 | 枚举 |
|------|-----------|-----------|------|
| 线程安全 | 需要 `volatile` | JVM 保证 | JVM 保证 |
| 序列化安全 | 需要 `readResolve()` | 需要 `readResolve()` | ✅ 天然安全 |
| 反射攻击 | ❌ 可破坏 | ❌ 可破坏 | ✅ 天然防 |
| 代码量 | ~15行 | ~8行 | ~3行 |
| 延迟加载 | ✅ | ✅ | ❌ |

**防反射原理**：
```java
// Constructor.newInstance() 源码
public T newInstance(Object... initargs) {
    if (clazz.isEnum()) {
        throw new IllegalArgumentException("Cannot reflectively create enum objects");
    }
    // ...
}
```

**序列化原理**：
```
枚举序列化：只写入 name 字符串（"RED"）
枚举反序列化：Enum.valueOf(Color.class, "RED") → 返回已有实例

普通类序列化：写入所有字段，可能创建新对象
```

**ordinal() 的坑**：
```java
// ❌ 不好：依赖 ordinal()
enum Priority { LOW, MEDIUM, HIGH }
int level = priority.ordinal();  // 如果中间插入新值，全部错乱

// ✅ 正确：自定义字段
enum Priority { LOW(1), MEDIUM(2), HIGH(3);
    private final int level;
    Priority(int level) { this.level = level; }
    public int getLevel() { return level; }
}
```

**枚举实现接口**：
```java
enum Operation implements BiFunction<Integer, Integer, Integer> {
    ADD { public Integer apply(Integer a, Integer b) { return a + b; } },
    SUB { public Integer apply(Integer a, Integer b) { return a - b; } };
}
```
