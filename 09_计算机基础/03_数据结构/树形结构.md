# 树形结构

> 树是面试中出现频率最高的数据结构，没有之一。理解红黑树是理解 HashMap/TreeMap 的前提，理解 B+ 树是理解 MySQL 索引的前提。

---

## 一、树的基本概念

### 1.1 术语

```
        A           ← 根节点（Root）
       / \
      B   C         ← B 和 C 是 A 的子节点，A 是 B 和 C 的父节点
     / \   \
    D   E   F       ← D/E 是 B 的子节点，D 和 E 互为兄弟节点
       /
      G             ← G 是叶子节点（Leaf），D 是内部节点
```

| 术语 | 定义 |
|------|------|
| 节点的度 | 子节点的个数（B 的度为 2） |
| 树的度 | 所有节点度的最大值 |
| 叶子节点 | 度为 0 的节点（无子节点） |
| 深度（Depth） | 从根到该节点的边数（G 的深度为 3） |
| 高度（Height） | 从该节点到最远叶子节点的边数（A 的高度为 3） |
| 层（Level） | 深度 + 1（根在第 1 层） |

### 1.2 二叉树的性质

| 性质 | 公式 |
|------|------|
| 第 i 层最多节点数 | 2^(i-1) |
| 深度为 k 的树最多节点数 | 2^k - 1 |
| 叶子节点数 n₀ | n₀ = n₂ + 1（度为 0 的节点数 = 度为 2 的节点数 + 1） |
| n 个节点的完全二叉树深度 | ⌊log₂(n)⌋ + 1 |

## 二、二叉树的遍历

### 2.1 四种遍历方式

```
        1
       / \
      2   3
     / \
    4   5

前序（根-左-右）：1 → 2 → 4 → 5 → 3
中序（左-根-右）：4 → 2 → 5 → 1 → 3
后序（左-右-根）：4 → 5 → 2 → 3 → 1
层序（BFS）：     1 → 2 → 3 → 4 → 5
```

### 2.2 遍历代码实现

```java
// 递归遍历（简洁，但栈深度等于树高）
void preorder(TreeNode node) {
    if (node == null) return;
    visit(node);                   // 前序：先访问
    preorder(node.left);
    preorder(node.right);
}

void inorder(TreeNode node) {
    if (node == null) return;
    inorder(node.left);
    visit(node);                   // 中序：中间访问
    inorder(node.right);
}

void postorder(TreeNode node) {
    if (node == null) return;
    postorder(node.left);
    postorder(node.right);
    visit(node);                   // 后序：最后访问
}

// 迭代遍历（用栈模拟递归）
void inorderIterative(TreeNode root) {
    Deque<TreeNode> stack = new ArrayDeque<>();
    TreeNode curr = root;
    while (curr != null || !stack.isEmpty()) {
        while (curr != null) {
            stack.push(curr);
            curr = curr.left;      // 一路向左
        }
        curr = stack.pop();
        visit(curr);               // 访问
        curr = curr.right;         // 转向右子树
    }
}

// 层序遍历（BFS）
List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> result = new ArrayList<>();
    if (root == null) return result;
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root);
    while (!queue.isEmpty()) {
        int size = queue.size();
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            TreeNode node = queue.poll();
            level.add(node.val);
            if (node.left != null) queue.offer(node.left);
            if (node.right != null) queue.offer(node.right);
        }
        result.add(level);
    }
    return result;
}
```

> **关键技巧**：已知前序 + 中序可以唯一确定一棵二叉树；已知后序 + 中序同理。前序 + 后序不行。

## 三、二叉搜索树（BST）

### 3.1 性质

左子树所有节点值 < 根节点值 < 右子树所有节点值

### 3.2 操作复杂度

| 操作 | 平均 | 最坏（退化为链表） |
|------|------|-------------------|
| 查找 | O(log n) | O(n) |
| 插入 | O(log n) | O(n) |
| 删除 | O(log n) | O(n) |

### 3.3 删除节点

```
情况1: 删除叶子节点 → 直接删除
情况2: 删除只有一个子节点 → 子节点替代
情况3: 删除有两个子节点 → 用中序后继（右子树最小值）替代
```

## 四、平衡二叉树

### 4.1 AVL 树

**严格的平衡二叉树**：任意节点的左右子树高度差不超过 1。

**平衡因子（BF）** = 左子树高度 - 右子树高度，BF ∈ {-1, 0, 1}

**四种旋转**：
```
LL（右旋）：        RR（左旋）：
    C                   A
   /                   / \
  B        →          B   C
 /
A

LR（先左旋再右旋）：  RL（先右旋再左旋）：
  A                     C
   \                   / \
    C       →         A   B
   /
  B
```

| 旋转类型 | 触发条件 | 操作 |
|---------|---------|------|
| LL | BF > 1 且左子树 BF ≥ 0 | 右旋 |
| RR | BF < -1 且右子树 BF ≤ 0 | 左旋 |
| LR | BF > 1 且左子树 BF < 0 | 先左旋再右旋 |
| RL | BF < -1 且右子树 BF > 0 | 先右旋再左旋 |

### 4.2 红黑树（Red-Black Tree）

Java 中 `TreeMap`、`HashMap`（JDK 8+）都用红黑树。它是一种**近似平衡**的二叉搜索树。

**五条性质**：
1. 每个节点是红色或黑色
2. 根节点是黑色
3. 叶子节点（NIL）是黑色
4. **红色节点的两个子节点必须是黑色**（不能有连续红节点）
5. **从任一节点到其所有叶子节点的路径上，黑色节点数相同**（黑高相同）

```
示例红黑树（B = 黑, R = 红）：

       13B
      /    \
    8R      17R
   /  \    /  \
  1B   11B 15B  25B
          \
          22B

所有路径黑高相同 = 3
```

**红黑树 vs AVL 树**：

| 维度 | 红黑树 | AVL 树 |
|------|--------|--------|
| 平衡程度 | 近似平衡（黑高相同） | 严格平衡（高度差 ≤ 1） |
| 查找效率 | O(log n) 略慢 | O(log n) 更快 |
| 插入效率 | O(log n) 最多 2 次旋转 | O(log n) 最多 O(log n) 次旋转 |
| 删除效率 | O(log n) 最多 3 次旋转 | O(log n) 最多 O(log n) 次旋转 |
| 适用场景 | 写多读少 | 读多写少 |
| Java 使用 | HashMap、TreeMap | — |

> **面试话术**："HashMap 在 JDK 8 引入红黑树是为了解决链表过长导致查找退化为 O(n) 的问题。选择红黑树而非 AVL 树是因为 HashMap 的操作以插入为主，红黑树插入时旋转次数更少，综合性能更好。"

## 五、B 树与 B+ 树

### 5.1 为什么需要 B 树/B+ 树？

内存中的查找用平衡二叉树（红黑树）就够了，但**磁盘 IO**的成本是内存访问的 10^5 倍。B 树/B+ 树通过**减少树高**来减少磁盘 IO 次数。

```
二叉树 vs B+ 树（存储 1000 万条数据）：
- 红黑树高度：约 24 层 → 24 次磁盘 IO
- B+ 树（3 层，每节点 1000 个指针）→ 3 次磁盘 IO
```

### 5.2 B 树（B-Tree）

**多路平衡搜索树**，每个节点可以存储多个 key。

```
      [30 | 60]
     /    |    \
[10|20]  [40|50]  [70|80|90]

特点：
- 每个节点最多 m 个子节点（m 阶 B 树）
- 非根非叶节点至少 ⌈m/2⌉ 个子节点
- 所有叶子在同一层
- 数据存储在所有节点中
```

### 5.3 B+ 树（B+ Tree）

B 树的变体，**MySQL InnoDB 索引的底层结构**。

```
      [30 | 60]              ← 内部节点（只存 key，不存数据）
     /    |    \
[10|20]  [40|50]  [70|80|90]  ← 叶子节点（存 key + 数据）
  |        |         |
  ↓        ↓         ↓
[10|20]→[40|50]→[70|80|90]    ← 叶子节点形成有序链表

与 B 树的区别：
- 数据只存储在叶子节点（内部节点只存 key，一个块能存更多 key）
- 叶子节点通过指针连接成有序链表（支持高效范围查询）
```

**B 树 vs B+ 树**：

| 维度 | B 树 | B+ 树 |
|------|------|-------|
| 数据存储 | 所有节点 | 仅叶子节点 |
| 范围查询 | 需要中序遍历 | 直接沿叶子链表遍历 |
| 单次查询 IO | 可能更少（数据可能在非叶节点） | 稳定（必须到叶子层） |
| 树高 | 更高 | 更矮（内部节点可存更多 key） |

### 5.4 MySQL InnoDB B+ 树索引

```
InnoDB 页大小 = 16KB

非叶子节点：每页存约 1170 个索引指针（8B 指针 + 6B key + 6B 事务ID）
叶子节点：每页存约 16 条记录（假设每行 1KB）

3 层 B+ 树可存储：1170 × 1170 × 16 ≈ 2190 万条记录
→ 查找任意记录只需 3 次磁盘 IO
```

| 索引类型 | 结构 | 特点 |
|---------|------|------|
| 聚簇索引（主键索引） | B+ 树，叶子存完整行数据 | 一棵树既是索引也是数据 |
| 二级索引（辅助索引） | B+ 树，叶子存主键值 | 回表查询（需二次查聚簇索引） |
| 联合索引 | B+ 树，按多列排序 | 最左前缀匹配 |
| 覆盖索引 | 二级索引包含查询所需列 | 无需回表 |

## 六、Trie（字典树/前缀树）

### 6.1 结构

```
插入：apple, app, apply, banana

root
├── a
│   └── p
│       └── p  ← 标记为词尾（"app"）
│           ├── l
│           │   ├── e  ← 标记为词尾（"apple"）
│           │   └── y  ← 标记为词尾（"apply"）
└── b
    └── a
        └── n
            └── a
                └── n
                    └── a  ← 标记为词尾（"banana"）
```

### 6.2 操作复杂度

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 插入 | O(m) | O(m) |
| 查找 | O(m) | O(1) |
| 前缀搜索 | O(m) | O(1) |

> m 为字符串长度，与树中总节点数无关！

### 6.3 应用场景

- **自动补全 / 搜索建议**：输入前缀，返回所有匹配词
- **拼写检查**：查找是否存在某个词
- **IP 路由**：最长前缀匹配
- **敏感词过滤**：构建敏感词 Trie，快速匹配

## 七、树的高级应用

### 7.1 二叉树的最近公共祖先（LCA）

```java
public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    if (root == null || root == p || root == q) return root;
    TreeNode left = lowestCommonAncestor(root.left, p, q);
    TreeNode right = lowestCommonAncestor(root.right, p, q);
    if (left != null && right != null) return root;  // p 和 q 在两侧
    return (left != null) ? left : right;            // 都在同一侧
}
```

### 7.2 二叉树序列化与反序列化

```java
// 前序遍历序列化（用 "#" 表示 null）
//       1
//      / \
//     2   3
//        / \
//       4   5
// 序列化结果: "1,2,#,#,3,4,#,#,5,#,#"

public String serialize(TreeNode root) {
    StringBuilder sb = new StringBuilder();
    serializeHelper(root, sb);
    return sb.toString();
}

private void serializeHelper(TreeNode node, StringBuilder sb) {
    if (node == null) {
        sb.append("#,");
        return;
    }
    sb.append(node.val).append(",");
    serializeHelper(node.left, sb);
    serializeHelper(node.right, sb);
}
```

### 7.3 最大二叉树 / 根据前序中序构造二叉树

```java
// 从前序和中序遍历构造二叉树
public TreeNode buildTree(int[] preorder, int[] inorder) {
    return build(preorder, 0, preorder.length - 1,
                 inorder, 0, inorder.length - 1);
}

private TreeNode build(int[] pre, int preStart, int preEnd,
                       int[] in, int inStart, int inEnd) {
    if (preStart > preEnd) return null;
    int rootVal = pre[preStart];
    TreeNode root = new TreeNode(rootVal);
    int rootIdx = inStart;
    while (in[rootIdx] != rootVal) rootIdx++; // 在中序中找根
    int leftSize = rootIdx - inStart;
    root.left = build(pre, preStart + 1, preStart + leftSize,
                      in, inStart, rootIdx - 1);
    root.right = build(pre, preStart + leftSize + 1, preEnd,
                       in, rootIdx + 1, inEnd);
    return root;
}
```
