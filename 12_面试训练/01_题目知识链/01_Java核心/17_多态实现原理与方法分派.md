**重载 vs 重写（面试必考）**：

| 维度 | 重载（overload） | 重写（override） |
|------|----------------|----------------|
| 决定时机 | 编译期（静态分派） | 运行期（动态分派） |
| 依据 | 参数类型（编译期类型） | 对象实际类型（运行期类型） |
| JVM 指令 | `invokevirtual`（但方法签名编译期确定） | `invokevirtual`（动态绑定） |

**JVM 方法调用指令**：

| 指令 | 用途 |
|------|------|
| `invokestatic` | 调用 static 方法 |
| `invokespecial` | 调用构造器、private 方法、super 方法（无需多态） |
| `invokevirtual` | 调用普通实例方法（需要多态，动态分派） |
| `invokeinterface` | 调用接口方法（类似 virtual，但更灵活） |
| `invokedynamic` | Java8+ Lambda / 动态语言支持 |

**动态分派的过程（invokevirtual）**：
```
1. 取操作数栈顶的对象引用（实际类型 C）
2. 在 C 的方法表中找匹配的方法
3. 找不到就沿着继承链往上找
4. 找到后，将符号引用解析为直接引用（缓存起来，下次快）
```
→ 这就是为什么多态的调用有性能开销，也是 JIT 做「内联缓存」优化的原因。

**虚方法表（vtable，HotSpot 优化）**：
- 每个类有一个 vtable，存所有可重写方法的直接地址
- `invokevirtual` 只需一次数组索引查找，不是每次都沿着继承链找
- `final`/`private` 方法不在 vtable 里 → 可以内联

**静态分派（重载的选择）示例**：
```java
void test(Object o) { print("Object"); }
void test(String s) { print("String"); }

Object o = new String("hi");
test(o);  // 输出 "Object"！因为 o 的编译期类型是 Object
```
