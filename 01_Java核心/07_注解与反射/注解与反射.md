# 注解与反射

> 注解是"给代码贴标签"，反射是"读懂标签并执行动作"。两者合在一起，构成了所有 Java 框架（Spring、MyBatis、Hibernate）的底层运行机制——**在运行时动态解读代码的意图，并据此改变程序行为**。

---

## 这个问题为什么存在？

Java 程序在编译期就确定了所有类型和方法调用。但框架需要**在运行时才知道要操作什么类、调什么方法**：

- Spring 不知道你写了哪些 Controller——它需要在启动时扫描 `@RestController`
- MyBatis 不知道哪个方法对应哪条 SQL——它需要读取 `@Select` 注解
- JUnit 不知道哪些方法是测试用例——它需要找到 `@Test` 标记

**注解解决了"如何标记意图"的问题，反射解决了"如何在运行时读取标记并执行"的问题。** 缺一不可。

---

## 核心脉络

### 注解与反射的协作关系

```
定义注解 → 编译器写入字节码 → 运行时反射读取 → 框架执行逻辑
   ↑                              ↑                    ↑
 @Target/@Retention          getAnnotation()      Spring IoC/AOP
 自定义元注解               AnnotationParser     MyBatis Mapper
```
