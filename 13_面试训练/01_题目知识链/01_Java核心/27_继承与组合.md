## 参考要点（盲答后再看）


**构造函数链**：

```java
class Parent {
    Parent() { System.out.println("parent"); }
}

class Child extends Parent {
    Child() {
        // 编译器自动插入 super();
        System.out.println("child");
    }
}

new Child();
// 输出：
// parent
// child
```

**无参构造器缺失**：

```java
class Parent {
    Parent(int x) { }  // 没有无参构造器
}

class Child extends Parent {
    Child() {
        super(10);  // 必须显式调用
    }
}
```

**构造函数中的覆写陷阱**：

```java
class Parent {
    Parent() {
        doSomething();  // ❌ 危险！如果被子类覆写，子类字段还没初始化
    }
    void doSomething() { }
}

class Child extends Parent {
    int value = 10;

    @Override
    void doSomething() {
        System.out.println(value);  // 打印 0，不是 10！
    }
}
```

**脆弱的基类问题**：

```java
// 父类后续修改，可能悄悄破坏子类
class Parent {
    void process() {
        doStep1();   // 调用子类的覆写版本！
        doStep2();
    }
    void doStep1() { /* ... */ }
    void doStep2() { /* ... */ }
}

// 子类覆写 doStep1() 抛异常，整个 process() 就炸了
// 但父类作者完全不知道这个覆写的存在
```

**组合优于继承**：

```java
// ❌ 不好：继承 ArrayList
class CountingList<E> extends ArrayList<E> {
    int addCount = 0;
    
    @Override
    public boolean addAll(Collection<? extends E> c) {
        addCount += c.size();
        return super.addAll(c);  // ❌ addAll 内部调用 add()，双计数！
    }
}

// ✅ 好：组合
class CountingList<E> {
    private final List<E> list = new ArrayList<>();
    private int addCount = 0;
    
    public boolean add(E e) {
        addCount++;
        return list.add(e);  // 完全可控
    }
}
```

**覆写规则**：

```
返回值：可以是父类返回类型的子类（协变返回类型）
访问修饰符：只能放宽，不能收紧
  ✅ 子类 public 覆写父类 protected
  ❌ 子类 protected 覆写父类 public
异常：只能抛出父类方法声明的检查型异常的子类
```

**字段隐藏**：

```java
class Animal { String name = "Animal"; }
class Dog extends Animal { String name = "Dog"; }

Animal a = new Dog();
System.out.println(a.name);          // "Animal"！看编译时类型
System.out.println(((Dog) a).name);  // "Dog"
```
