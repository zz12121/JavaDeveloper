## 盲答引导

1. CountDownLatch 和 CyclicBarrier 的根本区别是什么？（提示：从"谁等谁"和"可不可重用"两个维度）
2. CyclicBarrier 的 BrokenBarrierException 是什么意思？为什么是"传染性"的？
3. Semaphore 的 state 可以是负数吗？acquire 和 release 可以是不同线程吗？
4. Phaser 和 CyclicBarrier 相比，增强在哪里？什么场景必须用 Phaser？
5. 哪个工具类是基于 AQS 的共享模式，哪个不是？

---

## 知识链提示

这道题应该让你联想到：

- `AQS 共享模式` → CountDownLatch 和 Semaphore 的底层
- `ReentrantLock + Condition` → CyclicBarrier 的底层
- `生产者消费者模式` → 配合 Semaphore 做限流
- `多阶段任务` → Phaser 的典型场景
- `工作窃取算法` → ForkJoinPool 的双端队列

## 核心追问

1. CyclicBarrier 的 party 数如果和实际线程数不匹配，会发生什么？
2. CountDownLatch 的 count 减到负数会怎样？await() 会受影响吗？
3. Semaphore 的公平模式和非公平模式有什么区别？性能差异大吗？
4. Phaser 的 `arriveAndDeregister()` 和普通 `awaitAdvance()` 的区别是什么？
5. 如果一个线程在 CyclicBarrier.await() 中被中断，barrier 会进入什么状态？其他线程怎么办？

## 参考要点（盲答后再看）

**四大工具类全景**：

```
                    CountDownLatch      CyclicBarrier       Semaphore         Phaser
核心语义             倒计数到0           集合点同步           许可证限流         分阶段同步
可重用               ❌ 一次性           ✅ 循环              ✅                ✅
线程角色             不对等（主/从）     对等                 不对等（竞争）      对等
回调                 无                  barrierCommand      无                 onAdvance
动态参与者           ❌                  ❌                  —                  ✅
底层                 AQS 共享模式        ReentrantLock+Cond  AQS 共享模式       独立实现
```

**CountDownLatch vs CyclicBarrier**：

```
记忆口诀：Latch 是门闩（打开就没了），Barrier 是栅栏（到了继续，下轮再用）

CountDownLatch：
  - 1 个线程等 N 个线程完成
  - 一次性，count 到 0 后不能重置
  - 典型：主线程等待所有子任务

CyclicBarrier：
  - N 个线程互相等待，一起行动
  - 可循环，重置后继续用
  - 典型：多玩家游戏同时开始

场景对比：
  - "等所有服务启动完成" → CountDownLatch
  - "多线程分阶段计算，每阶段必须所有人到齐" → CyclicBarrier
```

**CyclicBarrier Broken 机制**：

```java
// 一个线程中断 → barrier broken → 其他所有人抛 BrokenBarrierException
// 这叫"传染性"：一人掉队，全队皆知

try {
    barrier.await();
} catch (BrokenBarrierException e) {
    // 记录日志、通知监控、执行降级
}
```

**Semaphore 深入**：

```java
// state 可以是负数！
Semaphore sem = new Semaphore(3);
sem.acquire();  // state = 2
sem.acquire();  // state = 1
sem.acquire();  // state = 0
sem.acquire();  // state = -1（有人进了等待队列）
// state 为负时，表示等待中的线程数

// acquire 和 release 可以是不同线程（允许！）
// 但逻辑上建议同一资源持有者负责释放
```

**Phaser vs CyclicBarrier**：

```java
// Phaser 的优势：动态注册参与者
Phaser phaser = new Phaser(3);  // 初始 3 个

phaser.register();    // 动态注册 → 4 个
phaser.arriveAndDeregister();  // 注销 → 3 个

// CyclicBarrier 做不到：party 数必须在构造时固定
```

**场景速查**：

```
Q1: 需要限流？→ Semaphore
Q2: 等 N 个任务完成（一次性）？→ CountDownLatch
Q3: 多轮同步，线程数固定？→ CyclicBarrier
Q4: 多轮同步，线程数变化？→ Phaser
```
