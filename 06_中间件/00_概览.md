# 中间件 — 全景地图

> 中间件是连接各系统的枢纽。理解它们的设计思想，比记住API重要得多。

---

## 学什么

```
06_中间件/
├── 01_Redis
│   ├── 数据结构与底层实现    — SDS/跳表/ZipList/QuickList/IntSet
│   ├── 持久化               — RDB快照/AOF日志/混合持久化
│   ├── 内存淘汰策略          — LRU/LFU/TTL/随机
│   ├── 高可用               — 主从复制、Sentinel哨兵、Cluster集群
│   ├── 应用模式             — 分布式锁、缓存、排行榜、限流、消息队列
│   └── 常见问题             — 缓存穿透/击穿/雪崩、热Key、大Key
│
├── 02_消息队列
│   ├── MQ核心概念            — 生产者/消费者/Topic/Partition/消费者组
│   ├── Kafka
│   │   ├── 架构与存储        — Broker/Controller、日志存储、零拷贝
│   │   ├── 高可靠与高吞吐    — 副本机制、ISR、acks策略
│   │   └── 消费语义          — exactly-once、事务消息、消息积压
│   └── RabbitMQ
│       ├── 架构与交换机      — Direct/Fanout/Topic/Header
│       ├── 消息可靠性         — 确认机制、持久化、死信队列
│       └── 适用场景           — 与Kafka的对比选型
│
├── 03_Elasticsearch
│   ├── 倒排索引             — 分词器、文档映射、相关性打分
│   ├── 集群架构             — 分片/副本、Master/DataNode、脑裂
│   ├── 查询优化             — DSL查询、聚合分析、深度分页问题
│   └── 与MySQL的数据同步     — Canal/Lasticsearch-connector

└── 04_其他中间件
    ├── Nginx                — 反向代理/负载均衡/限流
    ├── ZooKeeper            — ZAB协议、临时/持久节点、Watch机制
    └── MongoDB              — 文档模型、聚合管道、适用场景
```

## 怎么学

**顺序**：01_Redis（用得最多，基础最好）→ 02_消息队列（Kafka为主）→ 03_Elasticsearch（搜索场景）→ 04_其他（按需）

**关键认知**：
- Redis 数据结构的设计思想（跳表为什么比B树好、ZipList为什么省内存）比API重要
- Kafka 的高吞吐来自**顺序写+零拷贝+页缓存+批处理**，理解这些才能做正确选型
- 每种中间件都有**适用边界**——Redis不适合存大数据，Kafka不适合海量Topic，ES不适合强事务

## 和其他域的关系

```
06_中间件 ←→ 01_Java核心   （Redis客户端 → Netty/NIO；序列化 → 反射）
06_中间件 ←→ 02_并发       （分布式锁 → Redisson；消费模型 → 线程池）
06_中间件 ←→ 05_数据库      （ES → 倒排索引 vs B+树；Canal → binlog同步）
06_中间件 ←→ 07_分布式      （CAP → 最终一致性；幂等 → 消费去重）
06_中间件 ←→ 09_计算机基础  （网络 → TCP模型；OS → 零拷贝/mmap）
```

---

*中间件的价值不在API调用，在于理解「它为什么这么设计」「它解决什么问题」「它的边界在哪里」。*
